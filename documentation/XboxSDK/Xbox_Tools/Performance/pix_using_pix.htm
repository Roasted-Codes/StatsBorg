Using PIX

The process of using PIX is fairly straightforward. The following example is designed to familiarize you with the tool.

  1. To make sure that your development machine can connect to the XDK, type:

    xbdir /x <xboxname> xe:\

    This should show a directory listing for the Xbox hard disk and should also set your default Xbox.

  2. Compile the Fur Sample using either a debug or profile version of Direct3D. After you have compiled and deployed on the XDK, run the sample.

  3. From the Start menu, run PIX. It is located under Programs, Microsoft Xbox SDK, Graphics Tools, Performance Investigator for Xbox.

  4. Click Record on the File menu, and be sure to check that you want to record both push-buffer and timing data. After you click OK, a progress bar appears and PIX starts recording the data. This should take less than 30 seconds for any of the samples provided with the SDK. If you are recording a more complex program, it may take longer.

    If the capture times out, check your application to make sure that it did not set D3D__SingleStepPusher to 1.

  5. After PIX records the data, it displays the timing information for CPU and GPU events in the Timeline window. The information is also listed in columnar form in the Events window. When you left-click on the Timeline window, the cursor moves to that point in the window and the corresponding events are highlighted in the Events window. Likewise, when you select an event in the Events window, the cursor moves to the corresponding place in the Timeline window. To see greater detail, right-click in the Timeline window and click Zoom In.

  6. Now that you have captured the data, you can save it out to a .PIX file on the PC so that you can reload it later. Simply click Save As on the File menu.

  7. Now run the analysis by clicking Begin Analysis on the File menu. Your XDK will reboot and begin running the PIX analysis application. During the analysis, columns in the Events window, such as "Effective Fill Rate", will be filled in as the results get computed. While running, the Rendertarget window will redraw with the contents of the render target at whatever point in the scene you have selected.

  8. Moving the pointers in the Timeline window also moves the highlighted text in the Events window, and vice versa. Within the Events window, you can view statistics for a given event, such as GPU Duration, Back-end Time, and Effective Fill-rate. Clicking on one of the column headers in the Events window sorts the events based on that column.

  9. User-defined events will also appear in the Events window. The Fur example has many user-defined events: FrameMove, Bear Mesh 0, Bear Mesh 1, Bear Fur 0, Bear Fur 1, and so on. To expand events that contain other events, click the plus sign to the left of the event name. When an event is expanded or collapsed in the grid window, the Timeline window view is updated to reflect the change. When an event is collapsed, it displays a summary of the events that it contains.

  10. When you are done running your analysis, click End Analysis on the File menu.
