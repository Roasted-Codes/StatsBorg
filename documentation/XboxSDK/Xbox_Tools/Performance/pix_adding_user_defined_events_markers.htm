Adding User-Defined Events and Markers

By default, PIX captures many CPU events and Direct3D function calls for use in profiling. In addition, it is possible to define your own events and markers specific to your game engine.

Markers specify when a particular event occurs. They have no duration and are represented in the PIX Timeline Window by a single line. You set markers by calling D3DPERF_SetMarker.

A user-defined event occurs over a period of time. An event, such as a game's physics engine, would be bracketed by a call to D3DPERF_BeginEvent at the start of the event, and a call to D3DPERF_EndEvent at the end. Since PIX supports hierarchical events, calls to D3DPERF_BeginEvent/D3DPERF_EndEvent and D3DPERF_SetMarker may be nested inside each other, up to eight levels deep. Both D3DPERF_BeginEvent and D3DPERF_EndEvent return an integer that represents the level of the hierarchy in which they are starting or ending. These return values may be used to debug mismatched pairs of calls.

The following code snippet shows how these functions can be used:

void FrameMove()
{
   int fmBegin = D3DPERF_BeginEvent( D3DCOLOR_ARGB( 0xff, 0xff, 0x7f, 0x7f ),
                                     "FrameMove" );
   // Update the frame
   // ...
									 
   int fmEnd = D3DPERF_EndEvent();
   assert( fmBegin == fmEnd );
}
