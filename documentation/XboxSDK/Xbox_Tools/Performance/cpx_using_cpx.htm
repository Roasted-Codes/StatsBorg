Using CPX

The process of using CPX is fairly straightforward. The following example is designed to familiarize you with the tool.

  1. Using a debug configuration, compile and deploy the DolphinClassic sample. In a default installation, the sample is in: 
    C:\Program Files\Microsoft Xbox SDK\Samples\Xbox\Graphics\DolphinClassic
  2. Using the XDK Launcher, start the sample running.
  3. From the Start menu, run CPX. It is located under Programs, Microsoft Xbox SDK, CPU Profiler for Xbox.
  4. From the CPX File menu, select Record. This launches the Record dialog box.
  5. To set the path to the debug symbols and choose a sampling method, click Settings, which launches the Settings dialog box.
  6. Type the path to the symbols in the Symbols Path text box. In this example, type: 
    C:\Program Files\Microsoft Xbox SDK\Samples\Xbox\Graphics\DolphinClassic\Debug

    In general, the path should include both the location of the .pdb and .exe files.

  7. Choose the Timer event in the Event Name drop down list and make sure that the threshold is set to 2000000.
  8. Click OK to close the Settings dialog box.
  9. Click Start to begin profiling. The Status pane will update to show the elapsed sampling time, the total number of samples, and the number of overhead interrupts.
  10. Click Stop to stop the profiling session.
  11. Use the TopX View and Call Stack View to investigate the profiling of the title.
