Remote Performance Monitor Tool (XbPerfMon)

The Remote Performance Monitor tool (XbPerfMon) connects the Xbox performance counters with the widely supported Windows 2000 performance counters system. 

USAGE: xbperfmon [-x Name][options]
OptionDescription
-c Performance counter client. Specify the name of the performance counter tool that XbPerfMon should launch and wait for. If not specified, PerfMon.Msc is used. PerfMon4.Exe from the Windows 2000 Resource Kit may be used to view counters at a sampling rate below 1 second. Many Xbox performance counters are only sampled at 100 ms intervals, so higher sampling rates will not improve the data displayed.
-g Disable GPU counters. This option lowers the sampling overhead.
-x Xbox name or Xbox IP. Specify a particular target machine.
-t ms Connection timeout. Specify a connection timeout in milliseconds.
-a Authoring Mode (not recommended). Open the management console in author mode, which allows modifying the console.
-? Display a complete list of available options.

Remarks

Game developers need a simple way to poll their Xbox for metrics and display the results. The XbPerfMon tool connects the Xbox performance counter system with the Windows 2000 performance counter system. This provides Xbox game developers with a widely understood and supported set of performance measuring tools.

Details

The XbPerfMon tool has no user interface by itself. It launches the specified performance monitor (PerfMon.Msc, by default). PerfMon allows the user to select which performance counter(s) to display and, optionally, to log. Alerts can be created and then configured to take various actions (such as sending a message) when specific counters cross specified thresholds. For more information on PerfMon.Msc, including how to add counters, save or display logs, set alerts, and so on, see MPconcepts.Chm in the C:\WinNT\Help directory (by default). This is also available from the PerfMon Help menu.

The XbPerfMon tool adds the Xbox system's performance counters to the development system's performance registry and then launches the configured performance tool (usually PerfMon). It waits for that tool to exit, and then it removes the Xbox values from the registry.

Averaged Samples

The value counters are sampled (on the Xbox side) every 100 ms. These samples are averaged over the last second to generate the value that is returned to the Windows machine. This is done to reduce the number of spikes in the sample data. This is true regardless of the sampling rate set in the performance counter tool on the Windows machine. If the sampling rate is 2 seconds or 0.1 seconds, the data is still the average of the last second of samples. This is a tradeoff of reducing the sample noise.
