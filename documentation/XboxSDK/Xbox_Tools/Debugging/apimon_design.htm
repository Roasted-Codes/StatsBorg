Suite Design Tool (API Monitor)

The API Monitor suite design program does everything that the API Monitor SE program does. In addition, it enables developers to create test suites that testers can execute. The test suites can be targeted toward validating compliance with individual TCRs. They can also be focused on your specific debugging and testing needs.

Note    The purpose of the API Monitor tool is to simulate failure rather than success. As a result, the return values it provides for spoofing function calls are the values that the API functions return when they fail. Although it is possible to return success values for some simple calls, it is not recommended that you do so. Because spoofed functions are not actually invoked, information in OUT parameters is invalid when spoofed calls complete.

Creating Test Suites

Create a test suite by first running the suite design program (API Monitor) and selecting the category of API functions you wish to monitor. The available categories are Winbase, Xbox, and XOnline. The functions in the Winbase category are Xbox versions of Win32 API functions. Therefore, it includes such common functions as GetLastError and GlobalAlloc. The functions in the Xbox category are selected functions from the Xbox Title Libraries. Lastly, the functions in the XOnline category relate to Xbox Live. You can create a test suite that includes functions from all categories by setting the Category drop-down list to All.

After you set the category, you can choose individual functions using the API drop-down list. Select a function in the list and click the Apply button. It appears in the list in the Selected APIs group. You may add as many functions as you like to the test suite.

Editing Test Suites

To edit an existing suite, click the Load button and select the test suite's XML file. If you already have a test suite loaded and load another, the suite design tool merges the new test suite into the current suite.

When editing a test suite, you can add API functions or change function return values. To delete API functions from the suite, click the Remove button.

Saving Test Suites

To save the test suite, click the Save As button in the Selected APIs group. API Monitor stores test suites as XML files. You can load your test suites later into API Monitor, or you can provide them to testers who use the API Monitor SE program.

Executing Test Suites

Like API Monitor SE, the API Monitor program can run test suites. Click the Start button to begin monitoring API functions in the suite.

To reboot the Xbox on which you are running the test suite, click the Reboot button. To reset a particular API function, select it in the list and click Reset.

Because API Monitor does everything that API Monitor SE does, you can log the return value of each monitored function call by clicking the Enable log check box. To save the contents of the log into a CSV file, click the Save As button in the Log group.
