Compiler Error C2632

'type1' followed by 'type2' is illegal

Possible cause

  • Missing code between two type specifiers.

The following sample generates C2632:

int float i;   // error
Programs can also generate this error by attempting to use the keyword bool as an identifier. In versions of the Visual C++ compiler prior to Visual Studio .NET 2003, the type bool was created using a typedef. As a result, programs could create identifiers with that name. Now, however, bool is a proper type. Therefore, it cannot be used as an identifier name.

The following sample generates C2632:

// C2632b.cpp
// compile with: /LD
void f(int bool);   // C2632

To resolve this error so that the code is valid in both the Visual Studio .NET 2003 and prior versions of Visual Studio, rename any variables called bool.
