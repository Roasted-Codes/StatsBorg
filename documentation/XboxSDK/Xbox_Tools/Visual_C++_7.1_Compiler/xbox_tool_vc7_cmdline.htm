Visual C++ 7.1 Command-Line Compiler

This section provides information regarding the Microsoft® Visual C++® 7.1 (VC7.1) command-line compiler included in this version of the Xbox Development Kit (XDK). The tools described in this section are subject to change in later versions of the XDK.

Optimizations

A commonly asked question is "What are the best optimization settings for the VC++ 7.1 compiler?"

For Visual C++ 7.1 code optimization on an Xbox console, use -GB/-G6 (GB now maps to G6). For speed optimization, the recommended settings are -O2 -GL -Gr. For aliasing, you can use -Oa or -Ow. Note that if your program uses a linked list, do not use these settings. Also note that the majority of aliasing problems disappear with Link Time Code Generation.

Link Time Code Generation

Most compiler optimizations are limited to a single file being compiled. Using Link Time Code Generation (LTCG), you can apply optimizations to an entire program at once by specifying /GL on the compiler and /LTCG on the linker when doing release builds. Link Time Code Generation offers the following benefits:

Increased Frame Rate
In-house testing has shown that using LTCG results in a 5–10 percent increase in frame rate.
Custom Calling Convention
A calling convention is a protocol describing how parameters are pushed onto the stack and then later, popped off it. Link Time Code Generation enables the optimizer to utilize unused registers to pass parameters instead of using the stack like a regular calling convention.
Cross-Module Inlining
Regular optimizers limit the inlining of functions to a single module. Using LTCG, the compiler/linker will attempt to insert function code inline with the caller (even if it's in a different module) instead of making a function call. To use this, you need to use the /Ob2 compiler option (which is on by default when using /O1 or /O2 switches) to tell the compiler that it should inline as much as it can.
Thread Local Storage (TLS) Displacement
Using LTCG, the compiler and linker now pick the most frequently used TLS variables and specifically allocate them in first, instead of just assigning them in order. This means the code generator can use byte and word instructions more easily. This saves three bytes per TLS reference.
Interprocedural Register Allocations
Normally, certain registers are considered scratch. If we know that the function doesn't modify the scratch register, it can just leave it alone as opposed to reloading it.
Improved Memory Disambiguation
If no addressing operator is applied to global variables, the linker/compiler can more aggressively optimize how the global variables are handled.

F1 Help

The Visual C++ IDE now attempts to determine whether a selected keyword is an Xbox keyword (or an MSDN keyword) when F1 is pressed in the editor. If the keyword is a valid Xbox keyword, the Xbox Help file is opened and the topic corresponding to the selected keyword is displayed. If the selected keyword is not an Xbox keyword, MSDN is launched as you would normally expect.

To disable this feature, rename [XDK install path]\Doc\XboxSDK.kwd to XboxSDK.kwd.bak. Once you've disabled this feature, the Visual C++ IDE should behave as if it were unmodified.
