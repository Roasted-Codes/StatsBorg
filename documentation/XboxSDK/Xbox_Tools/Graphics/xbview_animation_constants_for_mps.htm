Animation Data and Shader Constants for Matrix Palette Skinning

The animation data is stored in a resource block in the XBR file. The Skeleton::UpdateTransformations function traverses the bone hierarchy and applies the animation data to each frame node, if there is any animation data for that node. Skeleton::UpdateTransformations also concatenates this node with all of the previous nodes to get the local transformation matrix. The animation data output by Deep Exploration requires that the animation data be concatenated in according to the following formula:

mRotation * mScale * pFrame->m_Matrix * mTranslation

where mRotation is the rotation data matrix, mScale is the scale data matrix, pFrame->m_Matrix is the local transformation matrix, and mTranslation is the translation data matrix. Because matrix multiplication is not commutative, you cannot create a simple animation matrix by multiplying the rotation, scale, and translation together and then multiplying that by the frame.

( mRotation * mScale * mTranslation ) * pFrame->m_Matrix !=
	mRotation * mScale * pFrame->m_Matrix * mTranslation)

This is the reason why the XBR::GetAnimationData function doesn't return a single matrix for the animation data and instead takes three matrices as parameters.

When Skeleton::UpdateTransformations has finished traversing the hierarchy, the result is that the bones are concatenated and have the animation data applied to them. You still need two other matrices to construct the matrix needed for the vertex shader constant used in the shader. The first is the inverse bone rest matrix. This matrix is written out, already inverted, by the XDX file export plug-in in Deep Exploration. It is the inverse of the rest matrix for the bone, already concatenated for that position in the hierarchy. The other matrix you will need (potentially) is the matrix to convert the skin vertices from object to world space, if they are not already in world space. Thus, the final formula to compute the vertex shader constants is:

mObjectToWorldSpace * mInverseBoneRest * mBone * mWorldToObjectSpace

If the skin vertices are already in world space, mObjectToWorldSpace and its inverse, mWorldToObjectSpace will be identity, and we can save two matrix multiplies per vertex shader constant. It's usually pretty easy to ensure that the skins are in world space by having the artist do a ResetXForm in MAX or the equivalent in Maya, Lightwave, or XSI.
