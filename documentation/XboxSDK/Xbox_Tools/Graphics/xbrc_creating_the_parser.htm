Creating the Parser (XbRC)

XbRC makes use of the MSXML SAX parser. This parser is created by the following:

CoInitialize( NULL ); 
ISAXXMLReader* pRdr = NULL;
HRESULT hr = CoCreateInstance( __uuidof( SAXXMLReader40 ), NULL, CLSCTX_ALL, 
                               __uuidof( ISAXXMLReader ), (void **)&pRdr );

After creating the MSXML SAX parser, we have to point it at two handlers—the content handler and the error handler. The content handler will be called as SAX finds each new start tag, each end tag, tag data, and so on. This is done by calling:

pRdr->putContentHandler( g_pXBParser );

The error handler is called if, and when, the MSXML SAX parser encounters an error in the XML file. This error can be generated internally by SAX, or we can generate the error as our content handler runs if we find problems. This handler is setup by calling:

pRdr->putErrorHandler( g_pXBParser );

Both handlers are included in the same class in XbRC, the CXBParser class. The CXBParser class defines a set of functions for each handler.
