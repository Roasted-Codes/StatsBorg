XBR::SetVertexShaderResource (XbView)

HRESULT SetVertexShaderResource( DWORD VertexShaderResourceIndex );

This function sets the vertex shader to the resource index passed in. It is somewhat complicated by the fact that there are, typically, two vertex shader resources for a given vertex shader in a Pass. The first resource is the vertex shader code itself and any global constants we need to set. The second resource is the vertex shader pass information, which includes local vertex shader constants to set (like bones) and the vertex buffer stream to use with this shader. As such, the function is split into two parts based on which of these two resources we are setting. There is an if statement which breaks the function into two parts. 

if( pVS->VertexShaderReference == 0 )

If this statement is TRUE, we have the vertex shader code and any global constants. This part of the function creates the vertex shader using IDirect3DDevice8::CreateVertexShader, sets vertex shader constant 0 to the transformation matrix set in the Direct3D device, calls XBR::ApplyVertexShaderConstants, and then sets the default stream if one was specified.

Should the if statement be false, the function uses the pVS->VertexShaderReference to get the first shader reference in the resource file. It then calls XBR::ApplyVertexShaderConstants with the local constants set for the vertex shader pass. It also sets the vertex buffer stream to the stream set in the vertex shader pass.
