XbRC Command-Line Commands

Compiles a file in the XML DirectX format (.xdx) to an Xbox resource file (.xbr) that can be loaded directly into an application, or previewed using the XbView Tool on the Xbox.

USAGE: xbrc [options] fileName [/F outputName] 
            [/H resHdrName] [/P resPrefixName] [/E errorName]
Option Description
/C Compile the resources but do not link them.
/O Turn on optimizations. No effect at the present time.
/R Force rebuild for all files. XbRC will do no dependency checking.
/F outputName Optional name of the output file. By default, the output file name is the input XDX file name with the extension changed to .XBR.
/H resHdrName Optional name of the resource header file. By default, the output header file is the input XDX file name with the extension changed to .h.
/P resPrefixName Optional string prefix to use for resource variables in the header file. By default, the prefix for each resource is the input XDX file name with no extension or path information.
/E errorName Optional file name for error logging. By default, the output log file is the name of the input XDX file name with the extension changed to .LOG.
/? Display usage message.

Remarks

The Xbox Resource Compiler (XbRC) has native support for the following image file formats:

  • .BMP
  • .DIB
  • .PPM
  • .TGA

If DirectX 8.1 is installed on the PC running the compiler, XbRC will support the following additional file formats:

  • .DDS
  • .JPG
  • .PNG

The compiler looks at one environment variable, XBRC_XBO_OUTPUT. This variable, if set, specifies a directory to place the intermediate files into instead of the directory where compilation is taking place.
