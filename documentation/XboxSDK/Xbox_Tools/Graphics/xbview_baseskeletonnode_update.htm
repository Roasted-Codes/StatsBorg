BaseSkeletonNode::Update (XbView)

BaseSkeletonNode::Update checks to see if the there is a Skeleton pointer in the class and, if there is, it calls the Skeleton::UpdateTransformations function. Remember, it is possible to have a BaseSkeletonNode without a Skeleton. This can happen when you have a piece of geometry (that is, a model) that is not attached to any skeleton. When there is no skeleton, this function essentially does nothing, because a model not attached to any skeleton needs no update.

BaseSkeletonNode::Update is called once per frame and updates the skeleton hierarchy, if one exists. The function is also instrumented to show up correctly in PIX. The function returns either S_OK if there is no Skeleton pointer or the result of the Skeleton::UpdateTransformations call.
