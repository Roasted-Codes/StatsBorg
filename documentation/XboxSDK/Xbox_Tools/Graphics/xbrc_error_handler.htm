Error Handler Functions (XbRC)

There are error handler functions defined in CXBParser:

  • error
  • fatalError
  • ignorableWarning

error

virtual HRESULT STDMETHODCALLTYPE error( ISAXLocator __RPC_FAR *pLocator, 
                                         unsigned short * pwchErrorMessage,
                                         HRESULT errCode );

This handler is called when the MSXML SAX parser finds a recoverable error. XbRC does not generate any recoverable errors, although the MSXML SAX parser might.

fatalError

virtual HRESULT STDMETHODCALLTYPE fatalError( ISAXLocator __RPC_FAR *pLocator, 
                                              unsigned short * pwchErrorMessage,
                                              HRESULT errCode);

This handler is called when the MSXML SAX parser detects a fatal error. XbRC can also generate calls to this handler if needed. Parsing ends at this point, and the compilation is unsuccessful.

ignorableWarning

virtual HRESULT STDMETHODCALLTYPE ignorableWarning( ISAXLocator __RPC_FAR *pLocator, 
                                                    unsigned short * pwchErrorMessage,
                                                    HRESULT errCode);

This handler is called when XbRC detects data in the XDX file that may not be what the user intended. It is not called by the MSXML SAX parser. This is presented as a warning to the user but parsing continues, and compilation is regarded as successful if it completes without any further errors. There is no option to make warnings abort compilation with XbRC.
