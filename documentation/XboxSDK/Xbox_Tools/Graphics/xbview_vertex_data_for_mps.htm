Vertex Buffer Data for Matrix Palette Skinning

The first three floats are always the vertex position. This position may be in world space or in object space. If it is in object space, there will be an object-to-world-space matrix that is not an identity matrix defined for the skin mesh itself. You will need this matrix and its inverse when computing the vertex shader constants, if it is not identity.

The next four floats are the bone weights. Often, only three weights are written out and the fourth is calculated, because the sum of all four weights must equal one. Because this calculation would be done in the vertex shader on a per-vertex basis, we have elected to save those cycles and use an additional float for the weight. It would be trivially easy to re-architect this to do the math in the vertex shader and save a float per vertex.

The next four shorts are the bone indices. These are vertex shader constant register indices and are used to dereference the vertex shader constants you need for each bone. You'll see how these are used in the shader implementation section below.

Note that we also assume four weights per vertex always. Often, two bones are more than enough to achieve the desired skinning effect. This would save three floats worth of data space (assuming you do the math to calculate the other weight in the shader) and two shorts worth of indices. As you will see, it also saves a bunch of vertex shader instructions since you won't need to multiply and add four bones per vertex but only two.

These two fields (the bone weights and bone indices) are the only additional data needed per vertex. All of the rest of the vertex data is the same as it would be given the material type of the vertex.
