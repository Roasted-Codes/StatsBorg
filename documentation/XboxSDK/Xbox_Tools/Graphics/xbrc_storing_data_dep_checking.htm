Storing Data for Dependency Checking (XbRC)

The next step to extend XbRC is to write the inputIntermediateFile and outputIntermediateFile functions for the class derived from XBRCRawData to store the data for dependency checking. At this point, you would write functions to output and input the data to and from an intermediate file. These data are used by the needsCompiling function below to check dependencies and by the linker to quickly assemble the large resource file. In the Direct3D cases, we also implemented the input and output intermediate file function through Bundler. The reason why is so that the intermediate files can be viewed on the Xbox with XbView (mainly for debugging). Much of the data would be useless to look at, but you could check the texture output, for example, and ensure that the textures got converted correctly. Again, you are not obligated to follow any standard here because you will control both the input and output of the intermediate file data for any extensions you create.
