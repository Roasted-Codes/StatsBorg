Pass (XbView)

The Pass structure represents a single pass of rendering. The structure specifies the vertex shader, pixel shader, vertex buffers, index buffer, texture stages, and render states needed to draw that pass. Each effect is made up of one or more passes to render it. For example, you might have a single mesh with three different texture maps on it. Each separate texture map would be stored in a separate Pass.

struct Pass 
{
    DWORD VertexShaderResource;
    DWORD PixelShaderResource;	
    DWORD RenderStateCount;
    RenderState *rRenderState;
    DWORD TextureStageCount;
    TextureStage *rTextureStage;
    DWORD DrawCount;
    DrawData *rDraw;
};

VertexShaderResource is an index into the resource list for the vertex shader that will be used during this pass. Similarily, PixelShaderResource is an index to the resource list for the pixel shader that will be used. RenderStateCount gives the number of render states that will be set in the pass by rRenderState. TextureStateStateCount describes the number of texture stage states set in the pass by rTextureStage. Both render states and texture stage sets are reset every pass by XBR::SetDefaultState to the defaults required by XbView. DrawCount gives the number of DrawData structures pointed at by rDraw. Multiple draws per pass allow different sets of indices from the same vertex buffer. You can pack all of the vertex data from multiple passes into the same vertex buffer and draw different sections of it by using different index buffers.
