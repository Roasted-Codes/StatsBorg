VertexShader (XbView)

The VertexShader structure contains the data needed to setup the vertex shader.

#define VERTEXSHADER_MAX_STREAMS 16
struct VertexShader 
{
    DWORD Handle;                // Direct3D handle to vertex shader
    DWORD DeclarationByteCount;  // size of Declaration
    DWORD *Declaration;          // the declaration maps streams to vertex 
                                 // registers, and defines constants
    DWORD FunctionByteCount;     // size of Function
    DWORD *Function;             // function microcode maps vertex registers 
                                 // to transformed vertices
    DWORD VertexShaderReference; // Index into the resource list for the 
                                 // Pass Vertex Shader resource
    DWORD m_dwNumFrameConstants; // Stores number of vertex shader constants 
                                 // pointing to frame matrices
    DWORD *FrameConstants;       // Maps frame matrix data to the correct 
                                 // vertex shader constants
    struct StreamInput 
    {
        DWORD VertexBufferReference; // Index into the resource file for 
                                     // this stream's Vertex Buffer
        DWORD Stride;                // Vertex stride
        DWORD Offset;                // Vertex Offset
    } rStreamInput[VERTEXSHADER_MAX_STREAMS];
};

The Direct3D handle is set when the resource is first used by calling XBR::SetVertexShaderResource. This handle is released during the XBR::Cleanup call for this resource. DeclarationByteCount specifies the length of Declaration. Declaration is a token stream that describes the vertex shader constants that are used. FunctionByteCount specifies the length of the microcode used by the shader, and Function is that microcode. The m_dwNumFrameConstants member specifies the number of FrameConstants needed to map the bones to the correct vertex shader constants. The StreamInput structure defines the vertex buffers this vertex shader will use. VertexBufferReference is an index back into the resource list pointing to the vertex buffer for that stream. Stride and Offset define the stride and offset for the vertex buffer stream.

VertexShaderReference is an index back into the resource list pointing to another vertex shader. The reason for this is that there are typically two vertex shader resources for a given effect to draw itself. The first vertex shader is the shader definition in the XDX file, which specifies the shader assembly code and any generic constants. This is separate from the pass version of the vertex shader because multiple passes might want to use the same vertex shader and you don't want to store it redundantly. The other vertex shader is stored in the pass context. This vertex shader refers back (hence, the VertexShaderReference variable) to the predefined vertex shader and uses its microcode and generic constants. The pass vertex shader can also specify additional constants as well as the vertex buffer streams that the pass context will use to draw the effect.
