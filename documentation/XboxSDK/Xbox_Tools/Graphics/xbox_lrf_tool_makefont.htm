MakeFont Tool (MakeFont)

The MakeFont tool converts a TrueType font into a bitmap font file (.bmf).

Usage

USAGE: MakeFont [options] infile outfile
Option Description
-d directory Directory where the TrueType font is located.
-f fontfile TrueType font file to load; using this switch overrides the -d switch.
-h height Height of the font, in pixels.
-e default Hexadecimal number of the default character in the font.
-a alpha Amount of antialiasing information to store; valid values for alpha are 0, 2 or 4.
-r bits Width of the font's RLE packet, in bits; valid values are 0 or from 2 to 8, inclusive.
-s style Style of the font; valid values are NORMAL, BOLD, ITALICS, or BOLDITALICS.
-c name Create a C structure called name in the output file.
-v Display font statistics.

The infile argument specifies the name of an .rsp file that lists the characters from the TrueType font that should be converted.

The outfile argument specifies the name of the bitmap font file (.bmf) that MakeFont creates.

RSP File Format

An .rsp file is a text file that lists the hexadecimal Unicode value for each character in the TrueType font that should be converted. The file should list one value per line, as in the following example.

67
3040
3041
3042
3043
3044
3045
3046
3047
3048
3049
304a
304b

Note    You must include the lowercase "g" character (U+0067) in the list. Without the "g" character, the font will not load on the Xbox console.

An .rsp file usually starts with five lines of configuration information that describe the file name of the TrueType font and what properties the resulting .bmf font should have. These lines are optional; their values may be specified with command-line options. Any configuration option that you do not specify in the .rsp file must be specified as a command-line switch when running MakeFont.

The configuration section has the following format.

FONT "fontfile.ttf"
HEIGHT height
DEFAULT default
ANTIALIASLEVEL alpha
RLEWIDTH bits

The FONT option specifies the file name of the TrueType font to convert. This option corresponds to the -f command-line switch.

The HEIGHT option specifies the height of the font, in pixels. This option corresponds to the -h command-line switch.

The DEFAULT option specifies the default character in the font. This option corresponds to the -e command-line switch.

The ANTIALIASLEVEL option specifies the level of antialiasing information for the font. Valid values are 0, 2, or 4. This option corresponds to the -a command-line switch.

The RLEWIDTH option specifies the width of the font's RLE packet, in bits. Valid values are 0 or from 2 to 8, inclusive. This option corresponds to the -r command-line switch.
