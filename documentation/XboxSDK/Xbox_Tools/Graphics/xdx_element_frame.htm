frame Element

The <frame> element is the container for a frame hierarchy.

<frame
   id = string
   flags = string
   skin = string
   value = string
/>

Required Attributes

id

Identifier used for referencing from game code or from the parameter attribute of a <vertexShader> element. Identifiers must be 64 characters or less. Longer identifiers will be truncated and will generate linker warnings. The id must be unique amongst <frame> elements.

If the <frame> element is the root node for a skeleton, and the first nine characters of the id are "NoUpdate_", that skeleton's transformations are not updated every frame. This is done because bone rest matrices never need updating.

Optional Attributes

flags

Specifies flags for the frame node and must be one or more of the following values ORed together:
Value Description
hidden Any geometry associated to the frame node will not be drawn by XbView. This is the only flag value used by the Xbox Resource Compiler (XbRC).
bone This flag is set by Deep Exploration and is used for reimportation of the XDX file. It is ignored by XbRC. It indicates that the frame is a bone in the heirarchy.
restmatrix This flag is set by Deep Exploration and is used for reimportation of the XDX file. It is ignored by XbRC. It indicates that the frame is an inverse rest matrix.

skin

This attribute is set by Deep Exploration and is used for reimportation of the XDX file. It is ignored by XbRC. It specifies the identifier of the frame node holding the skin geometry the bone will influence.

value

Specifies the local object-to-world-space matrix for the frame node. If specified, the attribute will consist of 16 space-delimited floating-point numbers in row-major form (_11, _12, ..., _21, _22, ..., etc.) If this is not specified, the matrix is the identity matrix.

Element Information

Parent Elements XDX
Child Elements frame, model

Remarks

The frame hierarchy is created by looking at how <frame> elements are specified in the XDX file. A given <frame> element can have up to 200 children.
