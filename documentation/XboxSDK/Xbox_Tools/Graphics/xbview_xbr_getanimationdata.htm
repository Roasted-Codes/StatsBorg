XBR::GetAnimationData (XbView)

HRESULT GetAnimationData( const CHAR *strFrameId,
                          FLOAT fTimeStamp, 
                          BOOL bInterpolate,
                          XGMATRIX &mTranslation,
                          XGMATRIX &mRotation,
                          XGMATRIX &mScale );

This function gets the animation data requested for the node ID at the time passed in and interpolates it if the flag passed in is TRUE. It also takes references to the rotation, scale, and translation matrices that are set with the animation data.

The function first scans through the resources in the XBR class until it (a) finds animation data with the node ID requested, or (b) completes its scan through the list of resources. If it doesn't find the node ID requested, it returns E_FAIL to let the caller know that there is no animation data for this node. If it finds the node with animation data, it calculates the offset based on the timestamp passed in and indexes into the animation translation, rotation, and scale data at that offset and pulls out the relevant animation data. It returns the frame of data that is at or just before the timestamp passed in. If the bInterpolate flag is set to TRUE, the code does a linear interpolation between this frame's translation and scale data and the next frame's data. It also does a spherical linear interpolation between this frame's rotation data and the next frame's rotation data. After the translation and scale vectors are created, the scale and translation matrices are created using the XGMatrixTranslation and XGMatrixScaling functions. The rotation matrix is built from the rotation quaternion using the XGMatrixRotationQuaternion function. The function then returns S_OK to let the caller know that the matrices passed in contain valid animation data.
