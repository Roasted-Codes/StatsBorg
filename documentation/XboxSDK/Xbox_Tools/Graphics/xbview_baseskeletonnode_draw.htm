BaseSkeletonNode::Draw (XbView)

BaseSkeletonNode::Draw iterates through the list of BaseFrameNode classes and calls the draw routine for each one if and only if the BaseFrameNode has geometry associated with it. It checks the pointer returned by GetNodeMesh. If it is NULL, the routine skips drawing that BaseFrameNode because there is no geometry associated with it.

The first call in BaseSkeletonNode::Draw sets the world matrix to the root matrix for the BaseFrameNode that is to be drawn. The next call is the BaseFrameNode::Draw. BaseSkeletonNode::Draw then restores the pixel states back to defaults, because these are allowed to persist only across passes. The pixel state block is originally set up in the first call to CXBView::DrawModels and stores all of the render states defined by D3DSBT_PIXELSTATE.

BaseSkeletonNode::Draw is completely instrumented for PIX. When PIX runs, the timing information for the mesh is recorded by name so that artists can determine how long the GPU and CPU took to render this particular mesh.
