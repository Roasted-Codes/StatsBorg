XACT Programming Guide

This topic discusses the Xbox Audio Creation Tool (XACT) from the programmer's perspective. In brief, when programming with the XACT API, you must follow these three basic requirements:

  1. The game title must link the appropriate XACT library file.

    Library Name Approved Library Instrumented (Supports Auditioning) Contains Debug Information Supports Link-Time Code Generation
    Xacteng.lib Yes No No No
    Xactengd.lib No Yes Yes No
    Xactengltcg.lib Yes No No Yes
    Xactengi.lib No Yes No No
    Xactengid.lib No Yes Yes No

    For release builds, only xacteng.lib and xactengltcg.lib are approved. To take advantage of Link Time Code Generation, titles must link to xactengltcg.lib.

    Xacteng.lib is a release (non-instrumented) XACT library and does not support auditioning.

    Xactengd.lib is a debug XACT library. This file is an instrumented version that supports auditioning and is functionally equivalent to xactengid.lib.

    Xactengi.lib and xactengid.lib are instrumented XACT libraries (they support auditioning).

  2. To create the engine, you must call XACTEngineCreate. This creates and initializes the XACT audio engine and needs to be called only once at the beginning of your application. 
    XACT_RUNTIME_PARAMETERS Params;
PXACTENGINE pEngine;

memset(&Params, 0, sizeof(Params));
Params.dwMax2DHwVoices = 128;
Params.dwMax3DHwVoices = 32;
Params.dwMaxConcurrentStreams = 16;

if ( SUCCEEDED( XACTEngineCreate(&Params, &pEngine) ) )
{
// Ready to do work
}
  3. Your application must continuously call XACTEngineDoWork. Call this function early and often (such as during the main game loop). For more information, see the section on this function.

For more information about programming with the XACT API, see the XActBasicSound Sample.
