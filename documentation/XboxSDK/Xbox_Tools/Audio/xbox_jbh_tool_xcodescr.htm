Code Scrambler Tool (XCodeScr)

The XCodeScr tool scrambles individual DSP program binaries before distribution.

USAGE: xcodescr key infile outfile
OptionDescription
key Private key to use for scrambling. This must be an 8-character ASCII string.
infile Input file to be scrambled. This may be any file, but is typically a GP DSP program binary image, the result of running a Motorola 56300 DSP assembler.
outfile Output file. This may be any file, but is typically a GP DSP program binary image. By convention, the output extension is ".scr".
-? Display usage.

Overview

The nature of the DSP instruction set and the generally small size of a program with GP DSP audio effects make it trivial to disassemble a straight binary image of such a program and reverse-engineer its algorithms. It is unlikely, therefore, that third-party vendors of these programs would risk porting their code over for use with Xbox without some guarantee of protection. To encourage these vendors to support the Xbox and make their DSP algorithms available to Xbox developers (and thereby increase the audio quality of Xbox titles), vendors must be able to scramble individual DSP program images.

Details of Usage

The DSPBuilder and XGPImage tools use the DSP code images scrambled by XCodeScr instead of raw DSP binary images. The output from DSPBuilder and XGPImage consist of fixed-up, re-scrambled DSP images, using the DSP program's original private key. Upon DSP program download by Microsoft® DirectSound®, the image file is unscrambled and the DSP program is executed on the GP DSP.

Note  For DSPBuilder, .ini files are required for parameter states, to generate headers, and so on.

  • DSP developer develops and debugs DSP program using Motorola 56300 DSP assembler and creates a binary output image (.bin).
  • DSP developer runs XCodeScr program to create a scrambled DSP program image (.scr), using a private key known only to the DSP developer.
  • DSP developer makes DSP program in scrambled form (.scr) available to Xbox developers.
  • Xbox developer integrates .scr into their title by using DSPBuilder or XGPImage to build a DSP image including the scrambled DSP program.
  • Output image produced by DSPBuilder or XGPImage is used by Xbox title, using the IDirectSound8::DownloadEffectsImage method.
