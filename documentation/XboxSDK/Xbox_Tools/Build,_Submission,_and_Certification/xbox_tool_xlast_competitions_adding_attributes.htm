Adding Custom Attributes

Your title can define custom attributes for the Competitions and Entrants tables in a competition's database. For example, suppose your title enables players to select a variety of terrains in which to play capture the flag. You can use a custom attribute to store the ID number of the terrain map. When a player joins the game, their copy of your title would obtain the map ID number from the database and use it to load the correct map for the competition.

XLAST provides an easy way to create any custom attributes that you might need. To do so, use the following steps.

  1. Click Custom Attributes in the project tree.
  2. Select the table you want to add the attributes to by clicking either the Competitions tab or the Events tab in the Properties window.
  3. Click the Add button.
  4. In the dialog box that appears, enter the name of the attribute. Names can consist of alphanumeric characters and spaces.
  5. Set the type of the attribute. Attributes can be Int, String, or Binary. If you specify the types String or Binary, you must also set the maximum number of characters or bytes, respectively, that the attribute can hold.

You can edit a custom attribute's value by selecting it and clicking the Edit button. In the dialog box that appears, you can edit the name, type, maximum number of characters for String attributes), or maximum number of bytes (for Binary attributes).

To delete a custom attribute, highlight it and click the Delete button.

Note    Deleting a custom attribute also deletes all filters, sort operators, and return values in the competition's queries that reference the attribute.
