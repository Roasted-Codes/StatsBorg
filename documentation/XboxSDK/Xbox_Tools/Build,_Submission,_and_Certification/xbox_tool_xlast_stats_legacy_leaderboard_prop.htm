Setting Leaderboard Properties for Legacy Versions

Clicking on an individual leaderboard in the project tree in the Document View window enables you to set the properties for that leaderboard. The indvidual leaderboards support the following properties.

Reset Type
Indicates how often the leaderboards should be reset to zero. This can be set to Never, Monthly, Bi-Monthly (every other month), Weekly, Bi-weekly (every other week), or Daily.
Maximum Number of Attachments
Specifies the maximum number of attachments any given leaderboard may have.
Maximum Attachments Size
Contains the maximum size, in bytes, of any single attachment.
Decay
Specifies the decay rate, in days, of the stats in the leaderboard. This information is used in conjunction with the Reset Type to determine when stats information is cleared from the leaderboard.
Attributes Visible on the Web
Selects which of the leaderboard's attributes are visible on the Web. These are displayed on the Web only if the leaderboard itself is visible on the Web. This property contains a list of attribute ID numbers, separated by commas.
Attributes Containing PUIDs
Contains a Passport User ID for the leaderboard. Currently, this property is not used by the Statistics Service API functions. It is provided for compatibility with future versions of the Statistics Service.
Top 100
Specifies that this leaderboard contains only the top 100 rankings.
Localized Soureboard Title
Uses a drop-down list of locales to enable you to specify a set of localized leaderboard titles. You can add locales to this list by clicking Manage Locales on the project property page.

Note    You can renumber the currently-selected leaderboard by clicking on its ID number, and then typing a new number for it. Likewise, you can rename attributes by clicking on the attribute name and typing a new name.
