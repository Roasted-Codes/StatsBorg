Build Speech Recognition Bank (BuildSpeechRecBank)

The Build Speech Recognition Bank (BuildSpeechRecBank) tool uses a section-based .txt file containing vocabulary information to generate an Xbox Speech Recognition (.xsr) file that can be copied directly to Xbox game discs and loaded into memory at runtime by the XHV API. For information on the XHV interface, see the XHV Speech Recognition Reference section.

Usage

USAGE: buildspeechrecbank outputfile.xsr vocablist.txt [options]
Option Description
-l Specifies the language file set to use. "USEnglish", "UKEnglish", "French", "German", "Japanese", or "Spanish" may be specified, with the default being "USEnglish".
-n Specifies the number of simultaneous users supported. Can be a value between 1 and 4, with the default being 1.
-h filename Specifies the name of a header (.h) file to be generated. The generated header file contains defines for each section represented in the resulting .xsr file.

Two language support files are required to be in either the tool's installation directory or the current working directory where the tool is being run. These files consist of a Pronunciation Dictionary file (.pdc) and a Packed Neural Network file (.pni). Additionally, the files are labeled language.pdc and language.pni, where language is "USEnglish", "UKEnglish", "French", "German", "Japanese", or "Spanish". BuildSpeechRecBank requires these files in order to build the .xsr file for the target language.

Creating a Vocabulary List

A vocabulary list is simply a Unicode .txt file containing word sections, word indices, and vocabulary words. For example, a vocabulary list containing a word section called [actions], followed by the vocabulary words run, jump, and fly might look like this: 

[actions]
100 Run
101 Jump
102 Fly
This will cover some basic actions for a character in a game title. You may add more groups to support voice recognition in different contexts of the game. The game may load a new context depending on the game state. For example, the following entries could be added to the same .txt file to support menu controls: 
[menu]
200 Up
201 Down
202 Left
203 Right
204 Select
Note that in each of the above sections, the numbers associated with the vocabulary words are unique. This is not a requirement of the engine, but is a recommended practice because it may alleviate confusion during development.

Generating a Header File

As specified in the command-line arguments, the -h switch will generate a header file based on the section information listed in the .txt file. To generate a header, you would use the following command line: 

buildspeechrecbank title.xsr title.txt -h title.h
For the example .txt above, the resulting header file would appear as follows: 
#include <xhv.h>

const XHV_SR_VOCAB_SECTION XHV_SR_VOCAB_ACTIONS = 0;
const XHV_SR_VOCAB_SECTION XHV_SR_VOCAB_MENU = 1;

const DWORD XHV_SR_WORD_RUN = 100;
const DWORD XHV_SR_WORD_JUMP = 101;
const DWORD XHV_SR_WORD_FLY = 102;
const DWORD XHV_SR_WORD_UP = 200;
const DWORD XHV_SR_WORD_DOWN = 201;
const DWORD XHV_SR_WORD_LEFT = 202;
const DWORD XHV_SR_WORD_RIGHT = 203;
const DWORD XHV_SR_WORD_SELECT = 204;
The first two constants in this example can be used when calling IXHVEngine::SelectVocabulary to determine which vocabulary (or vocabularies) are active. The constants that follow (elements from the [actions] and [menu] sections of the vocabulary list) may be used when calling the ITitleXHV::SpeechRecognized method to identify unique words.
