A Note on the Performance of Diffuse and Specular Interpolation

By Mike Abrash

The GPU for the Xbox® video game system from Microsoft (the Xbox GPU custom-designed by NVIDIA®) has a number of per-pixel interpolators, including diffuse, specular, and the four texcoords. Any of the four texcoords can be used as a general-purpose interpolator by selecting passthrough mode for the corresponding texture unit. However, doing so means that that texture unit cannot be used for any other purpose, such as looking up texels or doing bump-mapping or any other texture-related task. The diffuse and specular interpolators, on the other hand, are not tied to any other hardware, so they are normally the general-purpose interpolators of choice.

Despite the nomenclature of "diffuse" and "specular," these are truly general-purpose interpolators; their interpolation parameters can be set arbitrarily by the vertex shader, and their output can be used arbitrarily by the pixel shader. Complex pixel shaders are likely to need interpolated inputs of various sorts; whatever the desired interpolated inputs are, diffuse and specular interpolators are generally the preferred sources.

That's all straightforward enough, but there's one performance issue of which you should be aware. There's really only one general-purpose hardware interpolator in the Xbox GPU, which can process only one interpolant per clock, so if both diffuse and specular are in use, it takes one clock to interpolate diffuse, then another clock to interpolate specular. (There are separate hardware interpolators for texcoords.) Thus, the maximum possible fill rate if you are using both diffuse and specular interpolators is half the maximum fill rate of the GPU—400 MPix/sec in the case of the Alpha-2 XDK.

In many cases, this won't affect overall performance. If you're using more than two register combiners, for example, or if you're using more than two texture stages, the maximum fill rate is only 400 MPix/sec anyway, and since interpolation and pixel shading overlap, there will be no penalty for two interpolators in this case. Similarly, if you're using a slow texture address op or two ops that don't pair in a single clock (for example, one 3D texture and one 2D texture together), you won't run any faster than 400 MPix/sec.

On the other hand, if you have, say, one or two 2D textures and are using only two register combiners (plus the final combiner, which is free), be aware that you will find your maximum fill rate cut in half if you use both diffuse and specular interpolators.

One alternative is to use a texcoord interpolator in pass-through mode. However, if you do this, it takes up a texture unit, so you can use only one more texture unit before exceeding two texture units and thereby cutting fill rate in half, at which point you might as well have used diffuse and specular anyway.
