Xbox Live Titles Libraries and Xbox Live 3.0

This topic explains which XDK libraries titles should link with in order to use the new features that are supported in Xbox Live 3.0.

Xbox Live 3.0 Features

This XDK release provides Xbox Title Library support for the following features:

  • Voice Mail
  • Title Managed Storage
  • Family Title IDs
  • Teams
  • Competitions
  • Messaging
  • New Xbox Live Statistics Features

For brief descriptions of these features, see the What's New topic. For more in-depth information about these features, see the applicable reference pages.

These features require release 3.0 of the Xbox Live service. The Xbox Live 3.0 release has sometimes been referred to as "Tsunami." The Xbox group is releasing Xbox Live 3.0 around the beginning of April 2004. Titles that take advantage of these new features should schedule their release to occur after the release of Xbox Live 3.0. Titles that intend to release before the release of Xbox Live 3.0 should not use any these features.

You can identify Xbox Title Library functions that correspond to Xbox Live 3.0 features by looking for the following text at the top of the function reference page in the XDK documentation:

This function uses features that won't be available to Xbox Live subscribers until Xbox Live 3.0 is released. Titles that are scheduled to release before the release of Xbox Live 3.0 should not use this function. For more information, see Xbox Live Libraries.

Xbox Title Libraries for Xbox Live 3.0

This release of the XDK provides two variations of the XOnline libraries:

Standard Legacy Description
XOnline.lib XOnlinel.lib Standard XOnline lib.
XOnlined.lib XOnlineld.lib Debugging version of XOnline.lib.
XOnlines.lib XOnlinels.lib Secure version of XOnline.lib.
XOnlinesd.lib XOnlinelsd.lib Secure debugging version of XOnline.lib.

It also provides multiple versions of XNet.lib, as shown in the following table.

Library Description
XNet.lib Standard version of XNet.lib.
XNetd.lib Debugging version of XNet.lib.
XNets.lib Secure version of XNet.lib.
XNetsd.lib Secure debugging version of XNet.lib.
XNetn.lib "Neutered" version of XNet.lib. This lib is intended specifically for demos of Xbox Live games, where the demos run in a non-networked environment. The lib provides stubbed versions of all XOnline functions that run to completion, but do not actually do anything.
XNetns.lib Neutered debugging version of XNet.lib.

Titles that use the Xbox Live 3.0 features should link with the Standard XOnline libraries shown in the preceding table.

XDK sample code that uses the Xbox Live 3.0 features links with the Standard libraries.

Note    Titles that release before the release of Xbox Live 3.0 must not link to the standard XOnline libraries. A side effect of supporting the Xbox Live 3.0 features is that titles that link to these libraries will not be able to communicate with versions of the Xbox Live service prior to Xbox Live 3.0, even if the title does not use Xbox Live 3.0 features.

Titles that intend to release before the release of Xbox Live 3.0 should link with the Legacy XOnline libraries shown in the preceding table. Titles that link with these libraries will be able to communicate with the Xbox Live service both before and after the release of Xbox Live 3.0. However, these libraries do not support any of the Xbox Live 3.0 features.

XDK sample code that does not use Xbox Live 3.0 features links to the Legacy libraries. Also, Downloader.xbe and Stats.xbe both link to the Legacy libraries.

Note    Titles that use Downloader.xbe or Stats.xbe may link with the Standard libraries even though Downloader.xbe and Stats.xbe link with the Legacy libraries. Also, both Xbox High-Level Voice (XHV) and Drop-In UI for Xbox (UIX) may be used with either the Standard libraries or the Legacy libraries. However, if a title using UIX links with the Legacy libraries, that title will not be able to use Voice Mail with UIX: Friends.
