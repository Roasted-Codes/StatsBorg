The Game Loop

Here is the simple main loop that works in conjunction with the subroutines discussed in previous topics. Once initialization is complete, we enter the main loop. XACTEngineDoWork is a function defined in the XACT libraries and is responsible for driving internal XACT processing (for example, sending notifications). After that, we're free to act on notifications received from XACT and input state changes received from the user.

//
//  Main loop for XACT soundtrack sample
//
void main( void )
{
    CreateXACTEngine();
    CreateSoundBank();
    DownloadDSPEffectsImage();
    RegisterXACTNotifications();
    EnumerateSoundtracks();
    SetXACTPlayList( 0 );
    InitializeInput();

    // Main Loop
    while( TRUE )
    {
        XACTEngineDoWork();
        ProcessXACTNotifications();
        ProcessInput();
    }
}

To successfully compile and link the code in the entire Soundtrack Management Using XACT section, you'll need to link with xactengi.lib and dsound.lib (or their equivalent debug versions).
