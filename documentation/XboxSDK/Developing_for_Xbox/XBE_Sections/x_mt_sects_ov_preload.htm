Preloaded XBE Sections

By default, all sections in an XBE are loaded when the title is launched. These preloaded sections have a reference count of one. Sections can be marked as delayed load with the /NOPRELOAD flag of XBE Image Builder Tool (ImageBld). Sections marked as delayed load are not automatically loaded when the image is launched; the title must explicitly load them before they can be used. Marking sections as delayed load enables a title to forego loading of a section until it is needed, thus improving load time and decreasing the title's memory footprint.

Preloaded sections, like delayed load sections, can be freed. Since preloaded sections have a reference count of one, the first call to XFreeSection or XFreeSectionByHandle on that section will free the section. The section can later be loaded again as desired. The only difference between a preload and delayed-load section is that the preload section is loaded at image launch and has a reference count of one.

To mark the TESTSEG section (created in the example above) as delayed load, add the following switch to the ImageBld command line for the XBE: 

/NOPRELOAD:TESTSEG

Now the TESTSEG section will not be loaded when the title is launched. The title must explicitly load the section before functions and data in that section can be accessed.

While testing your section code, one caveat is that, in some cases, functions and data in a section may be accessible even though the section they are contained in has not been loaded. This is because portions of an XBE image are loaded into memory at the granularity of one page (4 KB). Especially for a small example as given above, the code and data at the beginning or end of the TESTSEG section may share a portion of a page with another section that has been loaded. When a section is loaded, all pages that contain any portion of that section are loaded into memory. Thus, even though the TESTSEG section itself has not been loaded (and has a reference count of zero), portions of that section may be in memory and appear to be accessible. Titles should never rely on this behavior; there is no guarantee of where the page boundary may fall. Always explicitly load a delayed-load section into memory before accessing it.
