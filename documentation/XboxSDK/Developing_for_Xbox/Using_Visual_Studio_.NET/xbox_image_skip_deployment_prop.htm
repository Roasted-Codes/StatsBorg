Skip Deployment

The Skip Deployment property enables the compilation of an XBox game without deploying the game to an XBox.

Remarks

Using the Skip Deployment property, developers who do not have an XBox attached to their development computer can still compile XBox programs. Therefore, they can make code modifications and verify that the code still compiles. This is extremely useful for developers working on a game that will be deployed to both Windows and the XBox.

The Skip Deployment property performs exactly the same task as the XBox Deployment tool's Exclude from Build property. However, configuration information for the XBox Deployment tool is stored in a .vcproj file, which may be checked into a source control system. Debugger properties, on the other hand, are stored in .suo files, which are usually local to each individual development computer.

If either the Skip Deployment or Exclude from Build property is set to Yes, the XBox Deployment tool is not executed when the build is performed.

To set this property in the Visual Studio environment:

  1. Open the Xbox project's Property Pages dialog box.
  2. Click Debugging.
  3. Set the Skip Deployment property to Yes.
