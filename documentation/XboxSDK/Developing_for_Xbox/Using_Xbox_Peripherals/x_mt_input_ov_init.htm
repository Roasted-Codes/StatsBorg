Device Initialization

A title cannot enumerate, open, or access any peripherals until it has successfully called the XInitDevices function. XInitDevices initializes peripheral support on the Xbox by specifying the number and type of devices that will be used. The Xbox system software allocates enough resources to handle the device limits specified and initializes the device control software.

The following code example demonstrates what the device initialization code for a game might look like. The g_chMUDrives and g_hPads global arrays will hold information about available devices and are discussed later.

#define DEBUG_KEYBOARD
#include <xtl.h>


#define NUM_PORTS   4       // Number of controller ports.
#define NUM_SLOTS   2       // Maximum number of slots per controller.

char g_chMUDrives[NUM_PORTS][NUM_SLOTS];
HANDLE g_hPads[NUM_PORTS];

void DeviceInit( void )
{
    XDEVICE_PREALLOC_TYPE xdpt[] = {
        {XDEVICE_TYPE_GAMEPAD, 4},
        {XDEVICE_TYPE_MEMORY_UNIT, 8},
        {XDEVICE_TYPE_DEBUG_KEYBOARD, 1} };

    XINPUT_DEBUG_KEYQUEUE_PARAMETERS xdkp = {
        XINPUT_DEBUG_KEYQUEUE_FLAG_KEYDOWN     |
         XINPUT_DEBUG_KEYQUEUE_FLAG_KEYREPEAT  |
         XINPUT_DEBUG_KEYQUEUE_FLAG_ASCII_ONLY,
        50,
        500,
        70 };

    int i;


    // Initialize the peripherals.
    XInitDevices( sizeof(xdpt) / sizeof(XDEVICE_PREALLOC_TYPE), xdpt );

    // Create the keyboard queue.
    XInputDebugInitKeyboardQueue( &xdkp );

    // Set device handles to invalid.
    ZeroMemory( g_hPads, sizeof(g_hPads) );

    // Set drive letters to invalid.
    ZeroMemory( g_chMUDrives, sizeof(g_chMUDrives) );
}

The XDEVICE_PREALLOC_TYPE structure passed to XInitDevices describes the types of devices the title will use as well as the number of devices of each type that will be used simultaneously.

This initialization code also calls XInputDebugInitKeyboardQueue to initialize a keyboard queue that will hold key input from the debug keyboard. A title not interested in debug keyboard input or one compiled as a retail (shipping) build should skip calling XInputDebugInitKeyboardQueue.

The XInitDevices call initializes the device control software and the peripheral bus and begins the process of querying the connected devices.
