Keyboard and Support Routines

Debug Keyboard Input

The code examples given in the previous topics already do most of the work required to read input from the debug keyboard. The DeviceInit() routine creates a keyboard input queue by calling XInputDebugInitKeyboardQueue. CheckDeviceChanges() dynamically handles keyboard insertions and removals. All that needs to be done now is to ask for keystrokes:

void CheckKeyboard( void )
{
    XINPUT_DEBUG_KEYSTROKE ks;
    CHAR szOut[2] = "X";

    if( ERROR_SUCCESS == XInputDebugGetKeystroke( &ks ) )
    {
        *szOut = ks.Ascii;
        OutputDebugString( szOut );
    }       
}

This simple example calls XInputDebugGetKeystroke to get a keystroke (if one is available), and if so displays it to the debugger.

Utility Functions

The DbgPrintf() function referenced earlier is a simple utility function that allows sending output to the debugger (using OutputDebugString) with a format string:

//
// This just does OutputDebugString using a format string.
//
int DbgPrintf( LPTSTR ptszFormat, ...)
{
    TCHAR tszBuf[MAX_PATH*2];
    va_list arglist;
    int nRes;


    va_start( arglist, ptszFormat );
    nRes = _vsntprintf( tszBuf, sizeof(tszBuf), ptszFormat, arglist );
    OutputDebugString( tszBuf );
    va_end(arglist);

    return nRes;
}
