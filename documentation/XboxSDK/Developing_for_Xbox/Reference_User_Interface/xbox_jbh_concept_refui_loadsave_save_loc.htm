Selecting the Location (Save)

Selecting the location for save game is much the same as in the load game case. The only differences are that (1) some devices will not have enough available storage for the saved game and should be unavailable, and (2) the space available/used indicators become much more valuable to users. The GUI model was preferred for selecting the location; see the section Selecting the Location under Load Game.

Devices that do not have enough free space to save the game will be shown as unavailable. In addition, a metadata pane will display a message indicating that the device does not have enough free space to save the game. The save device location screen should appear if there is one memory unit plugged into a controller regardless of whether or not the memory unit has enough space to store the saved game. The device selection screen is shown for consistency and because memory units are hot swappable.

When saving the game, a status message will be displayed to the user. When saving to an MU the message will read, "Saving game. Please don't remove memory units or disconnect any controllers." When saving to the hard disk the message will read, "Saving game. Please don't turn off your Xbox." For a current list of sanctioned error messages, please see "Standard UI Messages" in the Xbox Guide.

When a game fails to load, there should be a status message displayed to the user: "Saved game wasn't able to load." Again, the official wording for this (and every other status message) can be found in the aforementioned document.
