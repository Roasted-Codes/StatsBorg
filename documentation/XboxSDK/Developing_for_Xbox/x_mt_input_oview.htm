Using Xbox Peripherals

This section presents and explains end-to-end example code that covers the steps necessary to interact with Xbox peripherals. These peripherals include game controllers, memory units, debug keyboards, microphones, and headphones. Small code examples demonstrate each of the tasks that, together, enable device interaction. The following topics discuss each of the tasks involved:

  • Device Initialization describes the initial steps required in order to use Xbox peripherals. Before a title can enumerate, open, or access any peripherals, it must initialize the device control software.
  • Determining Connected Devices explains the initial procedure of determining the peripherals that are connected to the Xbox.
  • Detecting Device Changes describes the tracking of peripheral insertion and removal.
  • Handling Device Changes discusses the actions a title must take when it detects device changes. After detecting device insertion or removal, the title must open, close, or refresh handles to those devices.
  • Memory Units describes the steps that enable a title to access the memory unit's data after detecting a connected memory unit.
  • Save Games demonstrates the most common task a title performs on memory units, the storing of save games.
  • Querying Gamepads discusses the mechanism for retrieving information from gamepads, including button presses and thumbstick positions.
  • Keyboard and Support Routines provides a simple routine for reading debug keyboard input as well as other support routines used by this code example.
  • The Game Loop shows a simple game loop that pulls together and drives the device interaction demonstrated by the code example.
