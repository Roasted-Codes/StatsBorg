Changing the Tempo

The tempo of a performance dictates the conversion between the two types of time used in DirectMusic, which in turn controls the resolution of events to musical boundaries. (See Clock Time vs. Music Time.) The tempo track of the control segment usually manages the tempo, but it is also possible for an application to set the tempo dynamically.

To change the tempo, you will need to call the IDirectMusicPerformance8::SetGlobalParam method and change the master tempo.

The following example demonstrates how to play the music twice as fast as the default tempo.

    // ptPerf8 is a valid pointer to an IDirectMusicPeformance8 interface.    
    
    float fNewTempo = 2.0;
    ptPerf8->SetGlobalParam(GUID_PerfMasterTempo, (void *)&fNewTempo, sizeof(fNewTempo));

Note    Global tempo changes do not take effect immediately. Instead, they wait until transport time, which is approximately 500 ms.

For more information on changing the master tempo and other global parameters, see Setting and Retrieving Global Parameters.
