IDirectSoundStream::PauseEx

Sets the pause state of the sound stream.

HRESULT PauseEx(
    REFERENCE_TIME rtTimestamp,
    DWORD dwPause
);

Parameters

rtTimestamp
REFERENCE_TIME value that specifies the time in the future to resume playing the stream.
dwPause
Value that specifies the pause state. Can be one of the following values:
Value Description
DSSTREAMPAUSE_PAUSE Pause the stream.
DSSTREAMPAUSE_PAUSENOACTIVATE Pause the stream. If this flag is used on a playing stream, the behavior is identical to using the DSSTREAMPAUSE_PAUSE flag. However, if this flag is used on a stream that is not already playing, audio data packets are queued without actually activating the voice. This is useful when queueing data before making a call that requires the voice to be stopped, or when queueing data that you may not actually play.
DSSTREAMPAUSE_RESUME Resume playing the stream.
DSSTREAMPAUSE_SYNCHPLAYBACK Synchronize playback of multiple buffers and streams. See the IDirectSound8::SynchPlayback function for more information on using this flag.

Return Values

Returns DS_OK if successful; returns an error code otherwise.

Remarks

The stream will sit idle until the timestamp elapses if the stream is paused, primed with at least one packet, and then scheduled to resume.

Requirements

Header: Declared in DSound.h.

Import Library: Use DSound.lib.
