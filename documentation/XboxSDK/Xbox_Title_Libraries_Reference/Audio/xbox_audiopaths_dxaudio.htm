Audiopaths

Audiopaths manage the flow of audio content though the different objects in DirectMusic and the Xbox Media Communications Processor (MCP).

  • Related Topics

Understanding Audiopaths

The following list contains details you need to understand audiopaths:

  • All DirectMusic content plays on an audiopath.
  • Content on an audiopath can be routed to up to eight mixbins.
  • An audiopath can only play a wave whose number of channels divide evenly into the number of mixbins assigned to it (in other words, the number of mixbins must be a multiple of the number of channels in the wave).
  • Audiopaths determine how the audio content is routed. For example, audiopaths determine whether the audio content is mono, stereo, has a music reverberation, and so on.
  • Each audiopath has its own set of performance channels.
  • Audiopaths all share the same memory for wave data.
  • Playing two pieces of music that use different sets of instruments on the same audiopath will cause the first piece to play improperly (since the patch change and other MIDI controllers of the last piece loaded would override the ones loaded by the first piece). Playing these two pieces of music on different audiopaths should work just fine.
  • All DirectMusic Performance limitations still apply to all the audiopaths in the performance (that is only one primary segment can be played at a time, and there can only be one shared tempo).
  • DirectMusic Producer can be used to author more sophisticated audiopath configurations into a segment.
