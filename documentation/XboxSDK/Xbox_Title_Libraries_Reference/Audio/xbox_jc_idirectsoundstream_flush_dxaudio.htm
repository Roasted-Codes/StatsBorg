IDirectSoundStream::Flush

Flushes the stream (stops output and completes all packets).

HRESULT Flush();

Return Values

Flush should always return DS_OK if successful.

Remarks

When the stream is playing, calling Flush may cause a busy-wait state while waiting for the hardware voice to stop. To avoid busy-wait states, call IDirectSoundStream::FlushEx and specify the DSSTREAMFLUSHEX_ASYNC flag.

Additionally, calling some methods on the stream immediately after stopping the stream can generate debug warning messages. See DirectSound Warning Messages for more information.
