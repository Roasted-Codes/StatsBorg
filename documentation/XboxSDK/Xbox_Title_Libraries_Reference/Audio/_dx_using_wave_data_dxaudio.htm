Using Wave Data

DirectSound buffers and streams play waveform audio data in pulse code modulation (PCM) and Adaptive Differential Pulse Code Modulation (ADPCM) formats.

Wave data is usually stored in files or resources in Resource Interchange File Format (RIFF). The data includes a description of the wave format, including parameters such as the sampling rate and number of output channels.

The particular format of a sound can be described by a WAVEFORMATEX structure.

The wFormatTag member of this structure contains a unique identifier assigned by Microsoft Corporation. Only two tags are valid with DirectSound:

  • WAVE_FORMAT_PCM—indicates PCM, an uncompressed format.
  • WAVE_FORMAT_XBOX_ADPCM—indicates ADPCM, a compressed format.

It is important to note that ADPCM, 4-channel PCM, and 6-channel PCM streams require that stream packet data reside in physically contiguous memory (this does not apply to buffers). See the XPhysicalAlloc function for more information regarding allocation of physically contiguous memory.

More information on waves is included in Multichannel Wave Formats.
