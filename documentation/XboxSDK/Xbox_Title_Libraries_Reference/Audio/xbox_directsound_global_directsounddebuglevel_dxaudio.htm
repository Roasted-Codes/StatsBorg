g_dwDirectSoundDebugLevel

Determines the level for debug messages that will appear in the debugger.

DWORD g_dwDirectSoundDebugLevel;

Remarks

Xbox DirectSound uses two global variables to control debug output: g_dwDirectSoundDebugLevel controls the amount of output displayed in the debugger, and g_dwDirectSoundDebugBreakLevel determines whether a particular level of debug message will cause a break in the debugger.

Each time a DPF (debug printf) statement is made in DirectSound, a level is included. If g_dwDirectSoundDebugLevel is greater than or equal to this level, the statement appears in the debugger. If not, the DPF returns immediately.

If the debug level check succeeds and the statement is then sent to the debugger, the debug engine compares the level passed in with the DPF call to g_dwDirectSoundDebugBreakLevel. A debug break occurs if the following two conditions apply: g_dwDirectSoundDebugBreakLevel is greater than or equal to this level, and g_dwDirectSoundDebugBreakLevel is non-zero. This is useful for tracking down the source of a particular warning or error.

The debug levels currently used are as follows:

Debug Level Description
1 Errors
2 Resource allocation failures
3 Warnings
4 General information
5 Verbose information

The default DPF level is 3 (warnings) and the default debug break level is 2 (resource allocation failures).

Requirements

Import Library: Use DSound.lib.
