IDirectSoundBuffer8::StopEx

Causes the sound buffer to stop playing.

HRESULT StopEx(
    REFERENCE_TIME rtTimeStamp,
    DWORD dwFlags
);

Parameters

rtTimeStamp
REFERENCE_TIME value that specifies when the buffer should stop playing. If this value is zero, playback will stop immediately.

Note    Time stamps cannot be used to guarantee sample-accurate synchronization.

dwFlags
Flags that modify the behavior of the method. This parameter is a combination of one or more of the following values:
DSBSTOPEX_ENVELOPE
The buffer should enter its release phase.
DSBSTOPEX_IMMEDIATE
The buffer should stop immediately. This is equivalent to calling IDirectSoundBuffer8::Stop.
DSBSTOPEX_RELEASEWAVEFORM
The buffer should stop looping. This flag is used with the DSBSTOPEX_ENVELOPE flag so that the buffer breaks out of its loop region while in its release phase.

Note    These flags are not supported with buffers created using the DSBCAPS_FXIN2 flag.

Return Values

Returns DS_OK if successful, an error code otherwise.

Remarks

Reference time units are hundreds of nanoseconds from an arbitrary point when the Xbox is powered on.

Calling IDirectSoundBuffer8::StopEx does not guarantee that the hardware voice has actually shut off and stopped accessing memory. Titles must take steps to make sure the buffer is no longer being accessed before freeing the buffer. For more information, see Freeing Buffers on the Understanding Direct Sound page.

This command is valid for buffers created using the DSBCAPS_FXIN2, but is not valid for buffers created using the DSBCAPS_FXIN or DSBCAPS_MIXIN flags.

Requirements

  Header: Declared in Dsound.h.
