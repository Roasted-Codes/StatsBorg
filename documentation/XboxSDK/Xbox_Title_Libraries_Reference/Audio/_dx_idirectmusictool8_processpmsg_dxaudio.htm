IDirectMusicTool8::ProcessPMsg

Performs the main task of the tool. It is called from inside the performance's real-time thread for all messages that match the types specified by IDirectMusicTool8::GetMediaTypes. 

HRESULT ProcessPMsg(
    IDirectMusicPerformance *pPerf,
    DMUS_PMSG *pPMSG
);

Parameters

pPerf
Performance that is generating messages.
pPMSG
Message to process.

Return Values

Return values are determined by the implementation. If the method succeeds, the return value can be one of the following:

DMUS_S_REQUEUE
DMUS_S_FREE
S_OK

If it fails, the method can return E_POINTER.

Remarks

This method should return DMUS_S_REQUEUE when the tool wants the message to be requeued. This allows the tool to put a new time stamp and parameters on the message and requeue it, or to requeue the message with a different delivery type. If the tool wants the message to be sent on to the next tool in the chain, it calls IDirectMusicGraph8::StampPMsg to reassign the PMsg and then return DMUS_S_REQUEUE.

This method should return DMUS_S_FREE when the tool wants the message freed automatically and does not want to requeue the message.

This method should return S_OK when the tool does not want the message to be freed automatically. Perhaps the tool is holding on to the message for some reason or has freed it itself. For example, if the tool wants to send the message itself (like an echo that creates multiple copies), it calls StampPMsg on the message and IDirectMusicPerformance8::SendPMsg to send it. Then it returns S_OK, since it is now done with the PMsg.

Tools should not perform time-consuming activities because doing so can severely affect overall performance. Also be sure not to create a circular reference to the performance represented by pPerf. For more information, see DirectMusic Tools.
