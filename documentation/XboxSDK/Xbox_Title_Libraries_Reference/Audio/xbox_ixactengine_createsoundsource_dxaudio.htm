IXACTEngine::CreateSoundSource

Creates a 2D or 3D IXACTSoundSource object.

HRESULT CreateSoundSource(
    DWORD dwFlags,
    PXACTSOUNDSOURCE *ppSoundSource
);

Parameters

dwFlags
[in] The type of sound source to create. This parameter must be one of the following values:
Value Description
XACT_FLAG_SOUNDSOURCE_2D Create a 2D sound source. Not typically used; see Remarks below.
XACT_FLAG_SOUNDSOURCE_3D Create a 3D sound source. The sound source can be positioned and modified with 3D methods such as IXACTSoundSource::SetPosition.
ppSoundSource
[out] Pointer to a PXACTSOUNDSOURCE variable that receives the created IXACTSoundSource.

Return Values

Returns DS_OK if successful, an error code otherwise.

Remarks

Any multichannel wave content played onto a 2D or 3D sound source is downmixed to mono. To play multichannel waves (stereo, 4.0, 5.1, and so on) without downmixing to mono, use the default sound source—that is, pass NULL, rather than a sound source created with CreateSoundSource, as the pSoundSource parameter to IXACTSoundBank::Play.

Sounds played onto 2D and 3D sound sources mix into a MIXIN buffer, and are then played as mono. 2D sound sources send to all six speakers and the I3DL2 reverb mixbins. 2D sound sources are not typically used; the default sound source should be used for most audio playback. 2D sound sources are typically used only for software-implemented "3D" speaker panning (as demonstrated in the Pan3D Sample), and, infrequently, for certain digital signal processor (DSP) effects programs.

Manually managed sound sources can be freed when they are no longer active. See the IXACTSoundSource::GetStatus method.

For more information about sound sources, see the XACT Sound Sources section of the Programming with XACT white paper.
