IDirectSoundBuffer8::SetOutputBuffer

Specifies that the output of this buffer should go to the input of another buffer.

HRESULT SetOutputBuffer(
    LPDIRECTSOUNDBUFFER pOutputBuffer
);

Parameters

pOutputBuffer
Pointer to the destination buffer for output of this buffer. If pOutputBuffer is NULL, the buffer is treated as a normal DirectSound buffer and its output is sent to the final output.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

SetOutputBuffer will reset your mixbins assignments, but you're free to call SetMixBins after that. If you set the mixbins after calling SetOutputBuffer, they'll be preserved if you call SetOutputBuffer(NULL).

Note    The SetFormat call also resets the mixbin assignments and should be called before calling SetOutputBuffer. See the SetFormat function for more information.

pOutputBuffer must have been created with either the DSBCAPS_MIXIN or DSBCAPS_FXIN flag set. This specifies that the destination buffer cannot have any data written to it directly by the application; the data instead comes from one or more other DirectSound buffers or streams.

Buffers created with the DSBCAPS_FXIN2 flag can be passed to the SetOutputBuffer function, but the application is expected to call Play (PlayEx) and Stop (StopEx) on the destination voice for buffers of this type. Buffers created with either the DSBCAPS_MIXIN or DSBCAPS_FXIN flags automatically start playing when the first source voice starts and stop when the last source voice stops. Additionally, source buffers attached to an FXIN2 destination do not have to be mono, and do not automatically have their mixbins reset. You must point the source at the destination's input mixbin manually.

As mentioned above, you can disconnect the output buffer by calling SetOutputBuffer(NULL). However, if you stop the source voice and then immediately disconnect it before the hardware sends stop notification, you will not be able to stop or free the destination voice. See IDirectSoundBuffer8::Stop or IDirectSoundBuffer8::StopEx for more information.

This method cannot be used by buffers that were created with either the DSBCAPS_MIXIN or DSBCAPS_FXIN flag set, and cannot be called on a buffer that is already playing.
