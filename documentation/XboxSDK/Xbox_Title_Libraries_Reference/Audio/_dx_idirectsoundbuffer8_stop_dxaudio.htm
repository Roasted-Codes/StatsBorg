IDirectSoundBuffer8::Stop

Causes the sound buffer to stop playing.

HRESULT Stop();

Parameters

None.

Return Values

Returns DS_OK if successful, an error code otherwise.

Remarks

For sound buffers, IDirectSoundBuffer8::Stop sets the play cursor to the sample that follows the last sample played. This means that when the IDirectSoundBuffer8::Play method is next called on the buffer, it will continue playing where it left off.

Calling IDirectSoundBuffer8::Stop does not guarantee that the hardware voice has actually shut off and stopped accessing memory. Titles must take steps to make sure the buffer is no longer being accessed before freeing the buffer. For more information, see Freeing Buffers on the Understanding Direct Sound page.

This command is valid for buffers created using the DSBCAPS_FXIN2, but is not valid for buffers created using the DSBCAPS_FXIN or DSBCAPS_MIXIN flags.
