IDirectSoundBuffer8::SetNotificationPositions

Sets the notification positions. During playback, whenever the read or play cursor reaches an offset specified in one of the DSBPOSITIONNOTIFY structures in the caller-supplied array, the associated event is signaled.

HRESULT SetNotificationPositions(
    DWORD dwNotifyCount,
    LPCDSBPOSITIONNOTIFY paNotifies
);

Parameters

dwNotifyCount
Number of DSBPOSITIONNOTIFY structures.
paNotifies
Pointer to an array of DSBPOSITIONNOTIFY structures.

Return Values

Returns DS_OK if successful, an error code otherwise. Can be one of the following values:

DSERR_OUTOFMEMORY

Remarks

In order to use notifications, the buffer must be created with the DSBCAPS_CTRLPOSITIONNOTIFY flag set.

The value DSBPN_OFFSETSTOP can be specified in the dwOffset member to tell the Xbox System Software to signal the associated event when the IDirectSoundBuffer8::Stop method is called or when the end of the buffer has been reached and the playback is not looping. If it is used, this should be the last item in the position-notify array.

If a position-notify array has already been set, the method replaces the previous array.

The buffer must be stopped when this method is called.

Note    Notifications are only software based. SetNotificationPositions is essentially polling DirectSoundDoWork, which a title could do on its own.

Requirements

  Header: Declared in dsound.h.
