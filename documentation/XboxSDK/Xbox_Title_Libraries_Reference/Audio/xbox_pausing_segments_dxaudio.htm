Pausing Segments

Because DirectMusic does not support a pause method. Instead, titles must stop playing audio content in order to pause a segment. After the pause is over, the title would need to start playing the audio content again at the point it left off. The following sections will explain how to implement this functionality, and also discuss issues to consider when doing this.

  • Implementation Steps
  • Considerations
  • Related Topics

Implementation Steps

Complete the following steps to pause a segment:

  1. To retrieve the time the segment was started, call IDirectMusicSegmentState8::GetStartTime.
  2. To retrieve the time the segment was stopped, call IDirectMusicPerformance8::GetTime.
  3. To pause the segment, call IDirectMusicPerformance8::StopEx.
  4. To determine how much of the segment has been played, subtract the start time from the time the segment was stopped.
  5. To move the start point to the point where the segement was paused, call IDirectMusicSegment8::SetStartPoint (using the value provided in the previous step).
  6. To resume playing the segment from the point where the segment was paused, call IDirectMusicPerformance8::PlaySegmentEx.

Considerations

The following list contains some issues to consider when using the method described above to pause segments.

  • Notes inside a sequence or style track are played at specific points in time, but may sustain and/or decay for a period of time afterward. If a segment is stopped after a note has started playing, that note will not be triggered again when the segment is restarted. However, restarting a segment in the middle of a wave (in a wave track) will restart that wave from the appropriate offset.
  • IDirectMusicSegment8::SetStartPoint takes a start point for the segment in MUSIC_TIME. Clock-time segments and wave files loaded as segments do not have a MUSIC_TIME length, so IDirectMusicSegment8::SetStartPoint will fail unless the title sets a MUSIC_TIME length using IDirectMusicSegment8::SetLength.
