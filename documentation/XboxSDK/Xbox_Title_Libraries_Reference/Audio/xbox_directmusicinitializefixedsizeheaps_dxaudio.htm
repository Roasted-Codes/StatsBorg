DirectMusicInitializeFixedSizeHeaps

Initializes DirectMusic with fixed-size heaps.

HRESULT DirectMusicInitializeFixedSizeHeaps(
    DWORD cbNormalHeapSize,
    DWORD cbPhysicalHeapSize,
    LPDIRECTMUSICFACTORYFN pFactory
);

Parameters

cbNormalHeapSize
Value that specifies the maximum number of bytes that the normal heap is allowed to allocate.
cbPhysicalHeapSize
Value that specifies the maximum number of bytes that the physical heap is allowed to allocate. Can be set to zero if the title does not stream any ADPCM-compressed waves.
pFactory
Address of the factory function. Can be set to &DirectMusicDefaultFactory or to a custom factory generated by the Audio Analyzer tool.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

DirectMusicInitializeFixedSizeHeaps, DirectMusicInitializeEx, or DirectMusicInitialize must be called before making calls to DirectMusic. Titles are only allowed to call one of these initialization functions once per XBE. Titles must reboot before you are allowed to call any of them a second time.

The heap sizes are rounded up to a 4-KB (page size) boundry (except in the case that cbPhysicalHeapSize is set to zero).

Initially, titles should allocate approximately 4 MB for the normal heap and about 512 KB for the physical heap, and then use the xbperfmon tool to calculate the correct values for your title.
