Using DirectSound with Xbox

DirectSound is one of the main audio services for the Xbox. It is derived from the DirectX 8.0 version of DirectSound, and it also includes Xbox-specific interfaces, methods, and functions that expose the additional capabilities that are available on the Xbox.

Changes Specific to the Xbox

The following list contains some of the differences in DirectSound that are specific to the Xbox.

  • The IDirectSoundStream interface was added for audio streaming.
  • Titles must call the DirectSoundDoWork function when using streams, buffer position and stop notifications, and buffers that aren't assigned a hardware voice until they are played.
  • Titles must download a DSP effects image that contains the crosstalk cancellation code (such as Dsstdfx.bin, which is included in the source\dsound\bin directory of the XDK) to the global processor when using 3D sound, because 3D sound buffers get sent to the crosstalk mixbins.
  • Several methods have been added to IDirectSoundBuffer8 to expose the capabilities of the audio hardware.
  • DirectSound objects must be created using the functions provided by this library (DirectSoundCreate, DirectSoundCreateBuffer, and DirectSoundCreateStream), since there is no CoCreateInstance function on Xbox.

Additional Changes

The following list contains additional changes made to DirectSound for Xbox.

  • MMIO (Multimedia File I/O) functions are not supported.
  • The IUnknown::QueryInterface method is not supported.
  • 3D buffer methods on buffers that have not been created with the DSBCAPS_CTRL3D flag set will return E_FAIL.
  • There is no primary buffer.
