XGSpliceVertexShaders

Combines multiple vertex shader programs into a single vertex shader program.

HRESULT WINAPI XGSpliceVertexShaders(
  DWORD* pNewShader,
  DWORD* pcbNewShaderBufferSize,
  DWORD* pNewInstructionCount,
  CONST DWORD* CONST* ppShaderArray,
  DWORD NumShaders,
  BOOL bOptimizeResults
);

Parameters

pNewShader
[out] Pointer to a buffer that will be filled with the new vertex shader function token array that results from combining all the vertex shaders pointed to by ppShaderArray. pNewShader cannot be NULL.
pcbNewShaderBufferSize
[in, out] Points to a DWORD value containing the size of the buffer pointed to by pNewShader. pcbNewShaderBufferSize may be NULL, in which case it is assumed that the output buffer is large enough to hold the resulting shader. If pcbNewShaderBufferSize is not NULL, then upon the functions return, the DWORD value is set to the minimum size required for the output buffer to hold the new vertex shader.
pNewInstructionCount
[out] If pNewInstructionCount is not NULL, then on the functions return, pNewInstructionCount will point to a DWORD value containing the number of instructions in the newly-spliced shader pointed to by pNewShader.
ppShaderArray
[in] An array of pointers to the vertex shader token arrays that should be combined to create the new shader. None of the pointers can be NULL, and all of the vertex shaders must be of the same type.
NumShaders
[in] Number of vertex shaders in ppShaderArray.
bOptimizeResults
[in] TRUE to optimize the shader pointed to by pNewShader. If bOptimizeResults is TRUE, the new shader will be optimized by running the shader through a shader assembler. Furthermore, if the new shader is optimized, post-optimization, there must be at most 136 instructions.

Return Values

If the function succeeds, the return value is S_OK. If the buffer pointed to by pNewShader was too small and pcbNewShaderBufferSize wasn't NULL, the function will return S_FAIL.

Remarks

The resulting shader pointed to by pNewShader is never validated. An invalid vertex shader token array may result from the concatenation of the shaders in ppShaderArray. Shaders should be careful not to read from uninitialized registers.

The destination shader may not be used as any of the source shaders except the first one.

Setting pcbNewShaderBufferSize to NULL saves a little bit of computation time.

Requirements
