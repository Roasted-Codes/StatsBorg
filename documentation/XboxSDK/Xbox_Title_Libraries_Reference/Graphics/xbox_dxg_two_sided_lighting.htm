Two-Sided Lighting

It is sometimes advantageous to perform lighting calculations as if a triangle has two faces rather than one. An object that is thin and planar (such as a butterfly) can be easily represented by a single layer of triangles. Unfortunately, Direct3D only allows the lighting calculations to be performed for one side of the triangles and consequently, the geometry must be sent to the graphics processing unit twice: once for the front side and once for the back side.

The Xbox GPU supports two-sided lighting, so the Xbox defines an additional Direct3D render state that allows this capability to be controlled. Two-sided lighting is enabled using IDirect3DDevice8::SetRenderState(D3DRS_TWOSIDEDLIGHTING, TRUE), or is disabled using IDirect3DDevice8::SetRenderState(D3DRS_TWOSIDEDLIGHTING, FALSE).

Which side is considered front and which side is considered back is determined by the D3DRS_FRONTFACE renderstate. By default, the back-faces are automatically culled. Because this is generally not the desired behavior when using two-sided lighting, culling should be disabled by using IDirect3DDevice8::SetRenderState(D3RS_CULLMODE, D3DCULL_NONE). When you are finished using two-sided lighting, normal culling can be restored by using IDirect3DDevice8::SetRenderState(D3DRS_CULLMODE, D3DCULL_CCW).

Once two-sided lighting has been enabled, materials for the back side can be set using IDirect3DDevice8::SetBackMaterial or retrieved using IDirect3DDevice8::GetBackMaterial.

The ambient light color for the back side can be set using D3DRS_BACKAMBIENT.

The specular, diffuse, ambient and emissive material sources for the back side can be set using the render states D3DRS_BACKSPECULARMATERIALSOURCE, D3DRS_BACKDIFFUSEMATERIALSOURCE, D3DRS_BACKAMBIENTMATERIALSOURCE, and D3DRS_BACKEMISSIVEMATERIALSOURCE.
