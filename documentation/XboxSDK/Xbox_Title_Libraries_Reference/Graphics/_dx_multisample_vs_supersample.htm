Multisample Versus Supersample

Depending on the sampling size, a multisample region consists of either two, four, or nine fragments in the frame buffer. During polygon rasterization, the pixel shader inputs are computed once per sampling region. The pixel shader operates on these inputs and produces a single output color. The z-value is computed separately for each fragment in the region. The output color from the pixel shader is written to some subset of the fragments (in the sampling region) based on the results of the z comparison at each fragment.

Supersampling also involves sampling either two, four, or nine fragments in the frame buffer. The only difference from multisampling is that instead of computing a single color to be used by all the fragments, supersampling will execute the pixel shader once per fragment, using different inputs and computing different outputs for each fragment. The z-value is still computed separately for each fragment with the results of the pixel shader getting written to some subset of the fragments (in the sampling region) based on the z comparison at each fragment.

Multisampling can increase the fill rate since only one fragment per region is sent to the pixel shader. While the interior detail of rasterized polygons may be less with multisampling than supersampling (for a given filter type and number of samples per fragment), the edge detail will be the same. Supersampling may result in crisper looking pictures, but it is generally more expensive to use than multisampling.
