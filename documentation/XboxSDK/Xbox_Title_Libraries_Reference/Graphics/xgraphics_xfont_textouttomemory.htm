XFONT::TextOutToMemory

Draws text in a chunk of memory.

HRESULT __stdcall TextOutToMemory(
  LPCVOID pBits,
  unsigned Pitch,
  unsigned Width,
  unsigned Height,
  D3DFORMAT Format,
  LPCWSTR wstr,
  unsigned cch,
  long x,
  long y
);

Parameters

pBits
[in, out] Pointer to the memory where the text should be rendered to.
Pitch
[in] The pitch of the memory pointed to by pBits.
Width
[in] The width, in pixels, of the memory.
Height
[in] The height, in pixels, of the memory.
Format
[in] A member of the D3DFORMAT enumeration that gives the format of the pixels in memory.
wstr
[in] Pointer to the string that will be rendered.
cch
[in] The length of the string. cch can be -1 if wstr points to a NULL-terminated string.
x
[in] The x-coordinate of the reference point used to align the string. See the Remarks section for more details.
y
[in] The y-coordinate of the reference point used to align the string. See the Remarks section for more details.

Return Values

On success, the function returns S_OK.

Remarks

The meaning of the x and y coordinates specified in the parameters is determined by the text alignment set using XFONT::SetTextAlignment. By default, the coordinates refer to the upper-left corner of the text cell.
