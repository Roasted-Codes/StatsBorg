Allocating and Using Resources

The easiest way to allocate a resource is through the Create methods on IDirect3DDevice8, but in many scenarios, it makes sense to either pre-create the headers and data or to create the headers for the resource on demand and point the headers at existing data.

The headers for resources created outside of the device can be allocated in any memory, including on the stack or embedded inside of another structure. They need to follow these rules:

  • The reference count for the header should be initialized to one, and the title should never allow the count to go to zero so that Direct3D will not attempt to destroy the resource.
  • D3DCOMMON_D3DCREATED must not be set to prevent the driver from attempting to manipulate the resource allocations directly.
  • The header and data for the resource must live until the graphics processor has finished using them. For more information, see Resource Lifetime.

The data for a resource must be allocated from contiguous memory and accessed through a write-combined pointer. The methods D3D_AllocContiguousMemory and D3D_FreeContiguousMemory provide an easy to way to get such memory. It is legal, and even recommended due to the 4K-allocation granularity of this function, to have the data for multiple resources stored in one allocation as long as 1) that allocation lives as long as all of the resources whose data it contains, and 2) that data is aligned correctly within the allocation.

IDirect3DResource8::Register is a helper method that assists with filling in the data field of a manually allocated resource. It takes a pointer to a contiguous memory allocation, adds it to the Data field of the resource, converts the resulting address to a physical memory offset, and sets the result back into the Data field. This makes it easy to do a bulk load of many resources, because the headers can be pre-generated with the offset of their data within the contiguous memory allocation.
