XGSetFixupHeader

Manually creates an empty fix-up resource for use in creating per-call fix-ups with IDirect3DPushBuffer8::BeginFixup.

VOID WINAPI XGSetFixupHeader(
  UINT Size,
  IDirect3DFixup8 *pFixup,
  UINT Data
);

Parameters

Size
[in] Size in bytes of the fix-up object to create.
pFixup
[in, out] Address of the IDirect3DFixup8 interface, representing the fix-up resource that is to be filled.
Data
[in] Offset to the data held by this resource.

Return Values

None.

Remarks

The parameter Size can be zero in order to create a fix-up object that can be used to query the necessary fix-up size.

As with any Direct3D resource, Release should be called when finished with the fix-up resource.

All resources, including fix-ups, include a header that contains all of the information needed by the driver to process the resource and a data allocation that contains information used by the GPU. The resource headers are used only by the driver and so should be allocated from normal cached memory. The header contains all of the information needed by the driver to set up the hardware so that it can use the information stored in the data allocation.

Unless the title is manually creating a resource, it is recommended that the title use IDirect3DDevice8::CreateFixup.

Requirements

  Header: Declared in xgraphics.h.
  Import Library: Use xgraphics.lib.
