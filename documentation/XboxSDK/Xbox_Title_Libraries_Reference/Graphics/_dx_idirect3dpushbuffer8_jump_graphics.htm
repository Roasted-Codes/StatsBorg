IDirect3DPushBuffer8::Jump

Used only within a IDirect3DPushBuffer8::BeginFixup/IDirect3DPushBuffer8::EndFixup bracket to modify the corresponding call to IDirect3DDevice8::Nop as it was recorded in the push-buffer. This method jumps from one offset to another, skipping the commands recorded in the push-buffer that are between those offsets.

HRESULT Jump(
  DWORD Offset, 
  UINT DestinationOffset
);

Parameters

Offset
[in] The offset determined during the recording in the push-buffer as returned from IDirect3DDevice8::GetPushBufferOffset just before IDirect3DDevice8::Nop was recorded.
DestinationOffset
[in] The offset to jump to as determined during the recording in the push-buffer by IDirect3DDevice8::GetPushBufferOffset. If DestinationOffset is the same as Offset, then a NOP instruction will be inserted.

Return Values

If the method succeeds, the return value is S_OK.

Known Issues

The following is a known issue with IDirect3DPushBuffer8::Jump:

  • After Jump is used in a push-buffer fixup, calling Verify on that push-buffer will result in the assertion failure "ERR[D3D]: Assertion failure: instruction != 1".

Remarks

This method is callable only for non-D3DPUSHBUFFER_RUN_USING_CPU_COPY push-buffers. If the push-buffer was created with the RunUsingCpuCopy parameter of IDirect3DDevice8::CreatePushBuffer being TRUE, the title cannot call IDirect3DPushBuffer8::Jump.

IDirect3DPushBuffer8::Jump can be used to replace a Nop instruction with a GPU Jump instruction.

If calling IDirect3DPushBuffer8::Jump to modify a push-buffer on-the-fly from an IDirect3DDevice8::InsertCallback callback (for example, to implement a conditional jump in the push-buffer), the caller is responsible for flushing the CPU's write-combine cache using the x86 "sfence" instruction.
