D3DPERF_BeginEvent

D3DPERF_BeginEvent is used along with D3DPERF_EndEvent to define the start of a user-defined event that happens over a period of time. This event will then appear in the PIX Timeline Window.

INT D3DPERF_BeginEvent(
    D3DCOLOR Color,
    const char *szName,
    ... 
);

Parameters

Color
[in] A D3DCOLOR value used to specify the color that will show up when this event occurs in the PIX Timeline Window. If the alpha channel of the color is zero, the color will be inherited from the parent.
szName
[in] Pointer to a null-terminated string used to describe the event. The string may specify zero or more optional string format placeholders, similar to sprintf. If placeholders are used, the szName should be followed by a corresponding number of parameters whose type depend on the placeholders.

Return Values

Returns the level of the hierarchy this event is starting in. If an error occurs, this value will be negative.

Remarks

User defined events may be nested inside each other, up to eight levels deep. These events can be expanded or collapsed within the PIX Events Window to make it easier to see which CPU and GPU events belong to which of the user defined events.

Each call to D3DPERF_BeginEvent must be paired with a corresponding call to D3DPERF_EndEvent. These functions must bracket the event that is to be captured, and must not be separated by a call to Swap or Present.

In release builds of Direct3D, this function does nothing.

Requirements

Header: Declared in D3d8perf.h.

Import Library: Use Use d3d8i.lib or D3d8d.lib.
