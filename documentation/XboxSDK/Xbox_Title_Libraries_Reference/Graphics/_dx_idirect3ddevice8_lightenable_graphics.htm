IDirect3DDevice8::LightEnable

Enables or disables a set of lighting parameters within a device.

HRESULT LightEnable(
  DWORD LightIndex,
  BOOL bEnable
);

Parameters

LightIndex
[in] Zero-based index of the set of lighting parameters that are the target of this method. The maximum allowed LightIndex value is 4095.
bEnable
[in] Value that indicates whether the set of lighting parameters is being enabled or disabled. Set this parameter to TRUE to enable lighting with the parameters at the specified index, or FALSE to disable it.

Return Values

If the method succeeds, the return value is D3D_OK.

If the method fails, the return value can be D3DERR_INVALIDCALL.

Known Issues

The following is a known issue with LightEnable:

  • When switching from the fixed-function pipeline to a vertex shader, if a light is enabled then some negative vertex shader constants will be overwritten as the first primitive of each frame is drawn. If you use both the fixed-function pipeline and vertex shaders, you should disable all lights before using a vertex shader that makes use of negative shader constants.

Remarks

Using the fixed-function pipeline, it is possible to have more than eight lights set using IDirect3DDevice8::SetLight; however, Direct3D will use only the eight most recently enabled lights.

If a value for LightIndex is outside the range of the light property sets assigned within the device, the LightEnable method creates a light source represented by a D3DLIGHT8 structure with the following properties and sets its enabled state to the value specified in bEnable.
