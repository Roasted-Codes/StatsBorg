IDirect3DDevice8::SetTile

Assigns a range of memory to a tile, for efficient render target rendering. Tiles are GPU views that significantly improve rendering speed for render targets and depth buffers by reorganizing memory to improve bandwidth, and by allowing depth buffers to be compressed.

HRESULT WINAPI SetTile(
  DWORD Index, 
  CONST D3DTILE *pTile
);

Parameters

Index
[in] Index is the zero-based index of the tile to be used. There are a total of D3DTILE_MAXTILES (8) tiles available. Tiles 0 and 1 are reserved by Direct3D for the primary render target and primary depth buffer.
pTile
[in] Pointer to a D3DTILE structure, describing the new tile. If pTile is NULL, or if pTile->pMemory is NULL, the tile is disabled. If pTile is non-NULL and pTile->pMemory is non-NULL, the tile is enabled and configured according to the D3DTILE description. A tile must be disabled before it can be re-enabled.

Return Values

If the method succeeds, the return value is S_OK.

Remarks

This function is intended for use only with surfaces created in contiguous memory via D3D_AllocContiguousMemory and registered with Direct3D via IDirect3DResource8::Register. Tiles have strict alignment and pitch requirements. Note that multiple surfaces may share the same tile, so long as the same pitch is used for each

IDirect3DDevice8::SetTile is not synchronized to the push-buffer, and so always takes effect immediately at the time of the function execution. Consequently, it is the caller's responsibility to ensure that the GPU is no longer referencing tiled memory when the corresponding tile is either enabled or disabled. This may be done via synchronization means such as IDirect3DDevice8::BlockUntilIdle or IDirect3DDevice8::IsBusy.
