XFONT_OpenBitmapFont

Loads a bitmap font from a file.

HRESULT __stdcall XFONT_OpenBitmapFont (
  LPCWSTR wszFileName,
  unsigned uCacheSize,
  XFONT **ppFont
);

Parameters

wszFileName
[in] File name of the font.
uCacheSize
[in] Size of the font cache, in bytes. For how the cache size affects performance, see the remarks section.
ppFont
[out] Address of a pointer to the returned XFONT interface, representing the opened font.

Return Values

On success, the function returns S_OK.

Remarks

This method always allocates approximately 112 bytes for each font to store the font's state. Additional memory is allocated depending on the value of the uCacheSize parameter.

Value of uCacheSize Result
0 There is no cache. The memory allocation is just enough to hold the last drawn glyph. Glyph information is is taken directly from the file as each glyph is drawn and is very slow.
Less than font file size Allocates a cache of size uCacheSize that holds the most recently drawn glyphs. Glyph information is taken directly from the file if the glyph is not already in the cache. Once the cache is full, those glyphs that are accessed the least start getting ejected. This can result in reasonable performance if the cache is sized correctly as determined by experimentation.
Greater than, or equal to, font file size The cache is allocated to be just large enough to hold the contents of the font file which is then loaded entirely into the cache. This is results in very fast performance.

The font data in the file must follow the format defined in xfontformat.h.
