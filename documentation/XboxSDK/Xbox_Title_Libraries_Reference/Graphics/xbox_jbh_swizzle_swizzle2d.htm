XGSwizzleRect

XGSwizzleRect swizzles a subrectangle from a buffer into a larger texture. The destination rectangle must be completely contained within the destination texture (no clipping).

VOID WINAPI XGSwizzleRect(
  LPCVOID pSource, 
  DWORD Pitch,
  LPCRECT pRect,
  LPVOID pDest,
  DWORD Width,
  DWORD Height,
  CONST LPPOINT pPoint,
  DWORD BytesPerPixel
);

Parameters

pSource
The buffer that contains the source rectangle. For speed reasons, this buffer should be at least 4-byte aligned.
Pitch
The pitch of the buffer that contains the source.
pRect
The rectangle within the buffer to copy.
pDest
The destination texture. The destination needs to be 16-byte aligned.
Width
The width of the entire destination texture.
Height
The height of the entire destination texture.
pPoint
Where to put the rectangle in the texture.
BytesPerPixel
The number of bytes per pixel.

Return Values

None.

Remarks

If pRect is NULL, pPoint is NULL and Pitch == 0, this routine will assume that the source buffer is exactly the same size as the destination texture and will swizzle the whole thing. This routine will run considerably faster in that case.

For optimal performance when doing partial-texture swizzling, destination rectangles should be aligned according to the table below:

Bytes-Per-Texel of Source TextureAlignment of Destination Rectangle
1 8x8 aligned
2 8x8 aligned
4 8x4 aligned

If the destination rectangle contains an aligned block, either 8x8 or 8x4, then pDest must be 16-byte aligned.

Swizzling unaligned rectangles is several times slower than with aligned rectangles. For example, if you need to swizzle a 4x4 rectangle, with destination rectangle ((3, 3) to (6, 6)), it might be faster to instead work with the aligned 8x8 rectangle that surrounds it ((0, 0) to (7, 7)).

This method has been optimized for Write-Combined memory.

Linear textures have a minimum pitch of 64 bytes, and their pitch must be a multiple of 64. If width*bpp is not a multiple of 64, padding is added to the end of each row, in order to have a multiple-of-64 pitch. However, memory buffers (created with new[] or malloc(), for example), do not have this pitch requirement. Since most swizzling and unswizzling will be performed using these memory buffers, XG-Swizzling routines are set up to work with them. If you pass 0 for the pitch, XGUnswizzleRect will assume that the pitch of your linear buffer is width*bpp, with no padding. If you are in fact swizzling from a linear texture (D3DFMT_LIN_A8R8G8B8, for example), specify the pitch of that texture, otherwise strange graphical glitches may occur.
