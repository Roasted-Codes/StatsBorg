PSC0Mapping and PSC1Mapping

The PSC0Mapping and PSC1Mapping fields contain eight 4-bit integers that indicate which Direct3D constant is mapped to a constant in a particular combiner stage. The PS_CONSTANTMAPPING(s0,s1,s2,s3,s4,s5,s6,s7) macro is used to easily set the mapping. The macro is defined as below:

#define PS_CONSTANTMAPPING(s0,s1,s2,s3,s4,s5,s6,s7) \
  (((DWORD)(s0)&0xf)<< 0) | (((DWORD)(s1)&0xf)<< 4) | \
  (((DWORD)(s2)&0xf)<< 8) | (((DWORD)(s3)&0xf)<<12) | \
  (((DWORD)(s4)&0xf)<<16) | (((DWORD)(s5)&0xf)<<20) | \
  (((DWORD)(s6)&0xf)<<24) | (((DWORD)(s7)&0xf)<<28)

For example, if c0 in stage 0 is mapped to the constant 1 and c0 in stage 5 is mapped to the constant 5, the following code would properly set the mappings:

D3DPIXELSHADERDEF psd;
psd.PSC0Mapping = PS_CONSTANTMAPPING(1, 0xff, 0xff, 0xff, 0xff, 5, 0xff, 0xff);

These mappings are only used in IDirect3DDevice8::SetPixelShaderConstants, so if your application does not use this method, these fields do not need to be set.

For directly defined pixel shaders, it is often more convenient to set the constants using their associated render states rather than using the IDirect3DDevice8::SetPixelShaderConstants method. For example, the following code sets the c0 constant in stage 1 to a value of 5:

// Assume pDevice is a valid pointer to an IDirect3DDevice8 interface.
pDevice->SetRenderState( D3DRS_PSCONSTANT0_1, 5 );
