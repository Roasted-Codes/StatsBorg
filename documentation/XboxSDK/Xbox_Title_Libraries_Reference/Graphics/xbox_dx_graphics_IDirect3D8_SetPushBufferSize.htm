IDirect3D8::SetPushBufferSize

Sets the push-buffer size and the number of bytes to be written into the push-buffer before it is sent to the GPU.

HRESULT SetPushBufferSize(
  DWORD PushBufferSize,
  DWORD KickOffSize
);

Parameters

PushBufferSize
[in] The size of the push-buffer. This value has the following restrictions:
  • The size must be at least 64 KB.
  • The size must be a multiple of KickOffSize.
  • PushBufferSize / KickOffSize must be at least four.
KickOffSize
[in] The push-buffer is divided into logical units based on the size specified in the PushBufferSize parameter. The KickOffSize parameter indicates the number of units that are passed to the GPU for processing. The minimum kickoff size is 2048.

Return Values

If the function succeeds, it return S_OK.

Remarks

To change the push-buffer or kickoff sizes from their defaults, this method must be called before calling IDirect3D8::CreateDevice. Calling CreateDevice without calling this method will default the push-buffer size to 512 KB and the kickoff size to 32 KB. Once a device has been created, the only way to change the push-buffer or kickoff size is to release the device, call SetPushBufferSize, and then recreate the device.

Calling CreateDevice without calling this method will default to a push-buffer size of 512 KB and a kickoff size of 32 KB.

A smaller kickoff size will increase the granularity of what is pushed to the GPU and will reduce the amount of time that a resource is locked, but will result in more overhead. Title developers should experiment to see what value is the most efficient for their title.
