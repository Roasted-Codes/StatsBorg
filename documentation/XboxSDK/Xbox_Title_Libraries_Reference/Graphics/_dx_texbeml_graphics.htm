texbeml

Takes the color sampled by the preceding stage as DuDvL perturbation data with luminance information. 

texbeml   tDest, tSrc0

Registers

tDest
Destination register, holding the result of the operation.
tSrc0
Temporary source register, specifying the input argument.

Remarks

The texbeml texture mode is identical to texbem except that the output color is scaled by a luminance value from the displacement texture map as follows:

rgb = rgb * (lum_scale*t0.b + lum_bias)

The displacement map contains the u and v offset in addition to the luminance value. It can use the D3DFMT_L6V5U5 format or the D3DFMT_X8L8V8U8 format.

lum_scale is set using the D3DTSS_BUMPENVLSCALE texture stage state. lum_bias is set using the D3DTSS_BUMPENVLOFFSET. On the Xbox hardware, the luminance calculating takes place at a stage of the pipeline after the texture units. This means that if subsequent texture stages use the output of the texbeml stage, they will read the color before it has been adjusted by the luminance equation.

Like the texbem instruction, the texbeml instruction implements DirectX 6-style bump mapping. It uses a texture value read in a previous stage from a displacement map to perturb the texture coordinates at the current stage. The displacement map contains 2's complement signed values that are used to perturb the texture coordinates on the current stage. The new texture coordinates are calculated by multiplying the red and green components from the tSrc register by a 2 × 2 matrix. If tDest is t1 and tSrc is t0, then u and v are calculated as follows:

u = t1.s + mat00 * t0.r + mat10 * t0.g
v = t1.t + mat01 * t0.r + mat11 * t0.g

u and v then specify a 2D texture address that is used to do a normal texture lookup on stage 1. The 2 × 2 matrix is used to take into account movement of the eye point relative to the light source. It is specified using the D3DTSS_BUMPENVMAT00 through D3DTSS_BUMPENVMAT11 texture stage states. The texbeml texture mode is typically used immediately after a texture stage that reads the u and v displacement values as shown below:

tex t0          // Look up the bump map (2D bump map).
texbeml t1, t0  // Calculate a 2D texture address based on the 
                // displacement from stage 0 and then do a tex lookup.

Note that this instruction has an unnamed input which is the assigned texture that corresponds to this texture stage.
