IDirect3DDevice8::SetVerticalBlankCallback

Sets the function that will be called on each vertical blank.

void SetVerticalBlankCallback(
  D3DVBLANKCALLBACK pCallback
);

Parameters

pCallback
[in] Specifies the address of a callback function of type D3DVBLANKCALLBACK that will be called during each vertical blank. Set this to NULL to disable the vertical blank callback.

Remarks

The callback is defined as:

typedef void (__cdecl * D3DVBLANKCALLBACK)(D3DVBLANKDATA *pData);

The value passed as the pData parameter of the callback function is a D3DVBLANKDATA structure containing information about the current vertical blank.

The earliest that a title will be notified that the present or swap missed a frame is through SetVerticalBlankCallback.

This method is called at the time of deferred procedure calls (DPC) and should do little processing. Only a few XTL functions are safe to call at DPC level: The Interlocked functions are safe to use, as are the following XTL functions, if given a valid object handle:

  • SetEvent
  • ResetEvent
  • PulseEvent
  • ReleaseSemaphore
  • ReleaseMutex

When these functions are called at DPC with an invalid handle, their behavior is undefined.

Do not perform floating_point math, SSE instructions, and MMX instructions during DPC unless the XSaveFloatingPointStateForDpc and XRestoreFloatingPointStateForDpc functions are used.

Requirements
