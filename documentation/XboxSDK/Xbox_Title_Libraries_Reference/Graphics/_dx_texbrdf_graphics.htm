texbrdf

The texbrdf texture mode can be used to calculate a bi-directional reflectance distribution function. This function returns the surface color based on the angles of the vector from the fragment to the eye point and the angles of the vector from the fragment to the light.

texbrdf   tDest

Registers

tDest
Destination register, holding the result of the operation.

Remarks

This mode is typically used as follows:

tex t0	     // Convert eye vector to spherical coordinates.
tex t1	     // Convert light vector to spherical coordinates.
texbrdf t2   // Do 3D texture lookup to find fragment color value.

This mode uses a texture that can convert a three-component normal vector to spherical coordinates (phi, sigma). Each of these coordinates is 16 bits. This texture map is used in the first two stages to calculate the spherical coordinates for the eye and light vectors. This texture map should be set to use nearest-neighbor filtering on each stage. If linear filtering were used instead, adjacent spherical coordinates of 0 and 2*pi would be incorrectly blended together. The texbrdf stage performs the following calculation:

dPhi = eyePhi - lightPhi (dPhi ranges from -1.0 to 1.0)

It then performs a 3D texture lookup using the following texture coordinates:

(eyeSigma, lightSigma, dPhi)

The texbrdf stage requires a volume texture that holds the brdf transfer function for the current material. This texture should be set to wrap mode because angular values that are off by 2*pi are equivalent.
