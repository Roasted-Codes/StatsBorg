IDirect3DDevice8::EndVisibilityTest

Ends the specified visibility test.

HRESULT EndVisibilityTest(
  DWORD Index
);

Parameters

Index
[in] An integer that identifies which visibility test is being submitted. Index must be between 0 and D3DVISIBILITY_TEST_MAX - 1, where D3DVISIBILITY_TEST_MAX is defined as below: 
#define D3DVISIBILITY_TEST_MAX 16384

Return Values

If this method succeeds, it returns D3D_OK.

Remarks

This method ends the visibility test specified by the Index parameter. To reduce the memory automatically allocated, index values should be kept as low as possible and reused where feasible.

The use of any indices within a 256 range, aligned to multiples of 256, will cause the allocation of a 4K buffer. Subsequent use of any indices in the same 256 range will not cause any additional allocations. The memory will be resused.

For example, using indices 0, 1, 2 will cause the allocation of a single 4K buffer. These indices all fall within 0-255. If only index 300 is used, this will also cause the allocation of a single 4K buffer since it falls within 256-511 (a 256 range, aligned to a multiple of 256.) However, if IDirect3DDevice8::EndVisibilityTest is called using indices 0, 1, 2, 300, the visibility testing will allocate a total of 8K (4K for 0-255 and 4K for 256-511).

The results of the test can be queried with the IDirect3DDevice8::GetVisibilityTestResult method. Calling IDirect3DDevice8::EndVisibilityTest with an integer identifier that has been used with a previous visibility test that has not yet completed will result in an error.
