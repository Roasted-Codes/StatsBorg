XGSetVertexBufferHeader

Manually creates a vertex buffer resource.

VOID WINAPI XGSetVertexBufferHeader(
  UINT Length,
  DWORD Usage,
  DWORD FVF,
  D3DPOOL Pool,
  IDirect3DVertexBuffer8* pBuffer,
  UINT Data
);

Parameters

Length
[in] Ignored.
Usage
[in] Ignored.
FVF
[in] Ignored.
Pool
[in] Ignored.
pBuffer
[in, out] Address of the IDirect3DVertexBuffer8 interface, representing the created vertex buffer resource that is to be filled.
Data
[in] Offset to the data held by this resource.

Return Values

None.

Remarks

All resources, including vertex buffers, include a header that contains all of the information needed by the driver to process the resource and a data allocation that contains information used by the GPU. The resource headers are used only by the driver and so should be allocated from normal cached memory. The header contains all of the information needed by the driver to set up the hardware so that it can use the information stored in the data allocation.

Unless the title is manually creating a resource, it is recommended that the title use IDirect3DDevice8::CreateVertexBuffer.

After calling XGSetVertexBufferHeader, it is necessary to call IDirect3DResource8::Register in order to register the memory address of the resource.

Requirements

  Header: Declared in xgraphics.h.
  Import Library: Use xgraphics.lib.
