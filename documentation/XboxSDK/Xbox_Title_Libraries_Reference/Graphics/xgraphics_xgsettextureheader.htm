XGSetTextureHeader

Manually creates a texture resource.

DWORD WINAPI XGSetTextureHeader(
  UINT Width,
  UINT Height,
  UINT  Levels,
  DWORD Usage,
  D3DFORMAT Format,
  D3DPOOL Pool,
  IDirect3DTexture8 *pTexture
  UINT Data,
  UINT Pitch
);

Parameters

Width
[in] Width of the top-level of the texture, in pixels. The pixel dimensions of subsequent levels will be the truncated value of half of the previous level's pixel dimension (independently). Each dimension clamps at a size of 1 pixel. Thus, if the division by 2 results in 0 (zero), 1 will be taken instead.
Height
[in] Height of the top-level of the texture, in pixels. The pixel dimensions of subsequent levels will be the truncated value of half of the previous level's pixel dimension (independently). Each dimension clamps at a size of 1 pixel. Thus, if the division by 2 results in 0 (zero), 1 will be taken instead.
Levels
[in] The number of levels in the texture. If this is zero, Microsoft® Direct3D® will generate all texture sub-levels down to 1×1 pixels.
Usage
[in] Ignored.
Format
[in] Member of the D3DFORMAT enumerated type, describing the format of all levels in the texture.
Pool
[in] Ignored.
pTexture
[in, out] Address of the IDirect3DTexture8 interface, representing the created texture resource that is to be filled.
Data
[in] Offset to the data held by this resource.
Pitch
[in] Specifies the pitch of the texture.

Return Values

Returns the number of bytes required by the resource, not counting the header.

Remarks

All resources, including textures, include a header that contains all of the information needed by the driver to process the resource and a data allocation that contains information used by the GPU. The resource headers are used only by the driver and so should be allocated from normal cached memory. The header contains all of the information needed by the driver to set up the hardware so that it can use the information stored in the data allocation.

Unless the title is manually creating a resource, it is recommended that the title use IDirect3DDevice8::CreateTexture.

After calling XGSetTextureHeader, it is necessary to call IDirect3DResource8::Register in order to register the memory address of the resource.

The following code snippet demonstrates how a manually created texture resource should be registered.

IDirect3DTexture8* pTexture = new IDirect3DTexture8;
DWORD dwTextureSize = XGSetTextureHeader( dwWidth, dwHeight, 1, 0, D3DFMT_LIN_A8R8G8B8, 0, pTexture, 0, 0 );
void* pBuffer = D3D_AllocContiguousMemory( dwTextureSize, D3DTEXTURE_ALIGNMENT );
pTexture->Register( pBuffer );

In order to call IDirect3DDevice8::SetRenderTarget with a texture, you must select a level using IDirect3DTexture8::GetSurfaceLevel and pass the resulting surface to SetRenderTarget.
