Memory and Graphics

The memory containing data used by the graphics processor must be mapped as write-combine memory to ensure coherency between the graphics processor and the CPU. A write-combined mapping inhibits the CPU from caching reads to memory that could cause the CPU to miss writes to memory by the GPU but will combine writes in short-term write-combine buffers to improve write speeds. The Direct3D8 driver automatically flushes the CPU's write-combine buffers whenever any new commands are sent to the graphics processor, so the title never needs to do such a flush.

The graphics processor views memory as simple offsets into physical memory with no support for any kind of virtual addressing, so the memory for any single allocation must be composed of contiguous pages of physical memory.
