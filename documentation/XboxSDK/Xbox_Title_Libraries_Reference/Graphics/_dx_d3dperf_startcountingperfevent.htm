D3DPERF_StartCountingPerfEvent

Initializes buffers and variables to enable use of performance events, and turns on event recording.

__inline DWORD D3DPERF_StartCountingPerfEvent(
    DWORD EventCount
);

Parameters

EventCount
[in] Number of performance events that will be stored. Determines the size of the buffer that will be allocated for storing events. This buffer is circular. When the buffer is full, each new event will overwrite the oldest event in the buffer.

Return Values

Returns a DWORD describing which performance events are stored. To determine if a given event is recorded, bitwise-AND the DWORD with (1 << Event), where Event is an element of the D3DPERFEventsType enumeration.

Remarks

The title can manually specify the set of events that will be recorded by setting the RecordD3DPerfEvents field of the D3DPERF structure. If the title doesn't manually set this field, then D3DPERF_StartCountingPerfEvent will set the field to record (and display) the following event types:

  • D3DPERFEvent_Header
  • D3DPERREvent_Present
  • D3DPERREvent_BlockUntilIdle
  • D3DPERREvent_BlockOnFence
  • D3DPERREvent_PushBufferWait
  • D3DPERREvent_ObjectLockWait
  • D3DPERREvent_User0 to D3DPERREvent_User10

By setting the D3DPerfEventSpewFrameStart and D3DPerfEventSpewFrameCount members of the D3DPERF structure, the title can specify the number of frames of output that will get printed when D3DPERF_DumpPerfEvents is called. If the title does not initialize D3DPerfEventSpewFrameCount, then D3DPERF_StartCountingPerfEvent will initialize D3DPerfEventSpewFrameStart to two and D3DPerfEventSpewFrameCount to one.

D3DPERF_StartCountingPerfEvent should be called only when the GPU is idle, because any outstanding requests to the GPU will attempt to write back to the performance event buffer.

Requirements

Header: Declared in D3d8perf.h.
