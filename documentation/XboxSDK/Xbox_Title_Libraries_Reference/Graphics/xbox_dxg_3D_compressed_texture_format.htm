3D Compressed Texture Format

The Xbox uses an extension of DXTn to support 3D compressed textures. This format is block linear in nature and handles any size of texture map. Hardware limited to power of two texture sizes will find a simple implementation of the addressing possible. An optimization is done to minimize memory consumption and bandwidth for small texture depths.

Each texture map is based on width x height x depth texels in size, where all dimensions are greater than zero. Then, this texture map is divided up into blocks that are one of the following four sizes: 4 x 4 x 4, 4 x 4 x 3, 4 x 4 x 2, or 4 x 4 x 1. Specifically, if depth = 1, then the 4 x 4 x 1 size is selected. If depth = 2, then the 4 x 4 x 2 size is selected. If depth = 3, then the 4 x 4 x 3 size is selected. Otherwise, the 4 x 4 x 4 size is selected. The block sizes are not affected by width or height—only by depth.

DXTn 3D Block Formats

The 4 x 4 x 1 block contains one 4 x 4 x 1 subblock. The 4 x 4 x 2 block format contains two 4 x 4 x 1 subblocks that differ in position by depth only. The 4 x 4 x 3 block format contains three 4 x 4 x 1 subblocks that differ in position by depth only. The 4 x 4 x 4 block format contains four 4 x 4 x 1 subblocks that differ in position by depth only. The format for the 4 x 4 x 1 subblock matches the corresponding DXTn 4 x 4 block format (compressed to either 8 bytes or 16 bytes.)

The subblock ordering within the block is from smaller to larger depth values.

Block and Subblock Addressing

Given a texel coordinate (u, v, p) in the ranges u (0,1,...,width – 1), v (0, 1, ..., height – 1), and p (0, 1, ..., depth – 1), the following computes the corresponding address of the compressed block and subblock in memory containing that texel. As mentioned above, the subblock format matches the existing DXTn format.

subblock_size		=  for DXT1, 8; for DXT2..DXT5, 16.
block_size		=  MIN(p, 4) * subblock_size
horiz_stride		=  (width + 3) >> 2
planar_stride		=  ( (height + 3) >> 2) * horiz_stride
block_byte_address		=  ( (p >> 2) * planar_stride + (v >> 2) * horiz_stride + (u >> 2) ) * block_size
subblock_byte_address	=  block_byte_address + ( (p & 3) * subblock_size  )
