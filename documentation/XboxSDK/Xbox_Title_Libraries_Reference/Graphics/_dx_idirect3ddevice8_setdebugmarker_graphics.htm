IDirect3DDevice8::SetDebugMarker

In debug builds, sets a marker on the device that is used to filter geometry. In retail builds, compiles away to nothing. All geometry is rendered; however, both the vertex and pixel shader debuggers have switches that can be set so that they are only debugging a scene with a given marker—all other geometry will be ignored.

DWORD SetDebugMarker(
  DWORD Marker
);

Parameters

Marker
The DWORD value used for filtering.

Return Values

Returns the marker previously set on the device.

Remarks

The following example shows how SetDebugMarker is used.

  1. The debug marker is set to zero at the begining of the scene. It is automatically set to zero after each call to either IDirect3DDevice8::Present or IDirect3DDevice8::Swap.
  2. Draw some geometry.
  3. The debug marker can actually be set to any value; however, for this example it is set to one.
  4. Render the geometry you want to catch inside the vertex (or pixel) shader debugger.
  5. Set the debug marker back to zero.
  6. Render the rest of the scene.
  7. Start the vertex (or pixel) shader debugger and indicate which marker you want it to track.
  8. Start the title and the debugger will automatically catch the part of the scene you are interested in.
