IDirect3DDevice8::GetTileCompressionTags

Determines the compression status for a given depth buffer, assuming tiling has been enabled. Tiles are GPU views that significantly improve rendering speed for render targets and depth buffers by reorganizing memory to improve bandwidth, and by allowing depth buffers to be compressed.

DWORD WINAPI GetTileCompressionTags(
  DWORD ZStartTag, 
  DWORD ZEndTag
);

Parameters

ZStartTag
[in] Starting tag for a depth buffer. ZStartTag must be less than D3DTILE_MAXTAGS (76800).
ZEndTag
[in] Ending tag for a depth buffer. ZEndTag must be less than or equal to D3DTILE_MAXTAGS (76800).

Return Values

Returns the number of tag bits set for compression.

Remarks

This function can be used to determine how much compression is taking place in a given depth buffer for a given scene. Depth buffers are major consumers of memory bandwidth. There is a significant bandwidth benefit from compression, typically resulting in improved performance.

Z-Compressed tiles require that memory be allocated from special tag RAM, for the size of the depth buffer. Every D3DTILE_TAGSIZE (64) bytes of depth buffer require that one tag bit be allocated from the tag RAM. There are D3DTILE_MAXTAGS (76800) total tag RAM bits available, and the tag bits can only be allocated in bundles of 256.

ZStartTag and ZEndTag do not have to correspond to the beginning and end of a depth buffer. Instead, these values may be set to determine how only a portion of the depth buffer is compressed.

Assuming you have pDevice, a valid pointer to an IDirect3DDevice8 interface, the following code demonstrates how this function may be used:

D3DTILE PrimaryDepthBufferTile;
pDevice->GetTile( 1, &PrimaryDepthBufferTile );

DWORD startTag = PrimaryDepthBufferTile.ZStartTag;
DWORD endTag = D3DTILE_ZENDTAG( &PrimaryDepthBufferTile );

DWORD NumberCompressedTags = pDevice->GetTileCompressionTags( startTag, endTag );
