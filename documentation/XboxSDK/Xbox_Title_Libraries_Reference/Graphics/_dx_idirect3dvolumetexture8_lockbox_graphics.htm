IDirect3DVolumeTexture8::LockBox

Locks a box on a volume texture resource.

HRESULT LockBox(
  UINT Level,
  D3DLOCKED_BOX* pLockedVolume,
  CONST D3DBOX* pBox,
  DWORD Flags
);

Parameters

Level
[in] Specifies the level of the volume texture resource to lock.
pLockedVolume
[out] Pointer to a D3DLOCKED_BOX structure, describing the locked region.
pBox
[in] Pointer to the volume to lock. This parameter is specified by a pointer to a D3DBOX structure. Specifying NULL for this parameter locks the entire volume level.
Flags
[in] A combination of zero or more locking flags, describing the type of lock to perform.
D3DLOCK_TILED
This flag should be used when calling LockBox on surfaces that reside in tiled memory (such as the default frame buffer surfaces or any surfaces residing in a IDirect3DDevice8::SetTile range). This flag has to be specified to allow CPU reads and writes to properly respect the tile remapping setting. If it is not specified on a tiled surface, the CPU rendering results may appear "jumbled." Using this flag can result in somewhat slower CPU write access and significantly slower CPU read access, and so should only be used when needed and only with tiled surfaces.
D3DLOCK_READONLY
Indicates that application will not write to the buffer.
D3DLOCK_NOOVERWRITE
This flag keeps the lock from blocking if the texture is in use by the GPU.
D3DLOCK_DISCARD
This flag is not supported on Xbox.

Return Values

If the method succeeds, the return value is D3D_OK.

If the method fails, the return value is E_OUTOFMEMORY.

Remarks

D3DLOCK_READONLY will block if the GPU is using the texture since the texture may be a render target (in which case you may be reading dirty data). If you are reading from a texture that is not a render target, you should use ( D3DLOCK_READONLY | D3DLOCK_NOOVERWRITE ) to guarantee that the lock operation does not block.
