10x11 Pixel Aspect Ratio

This is a video mode attribute that may be specified in all video modes that are 640 pixels wide. It is not supported in any other modes. Enabling the 10 × 11 pixel aspect ratio will increase the black space on the left and right sides of the screen so that more of the rendered pixels are squeezed into the active portion of the screen.

This mode is enabled by ORing the D3DPRESENTFLAG_10X11PIXELASPECTRATIO flag in with the Flags member of D3DPRESENT_PARAMETERS.

If you choose to use this mode, you will need to adjust your projection transformation matrix to accommodate the non-square pixels in the display (the pixels are taller and thinner 1:1.1). For example:

// Setup the presentation parameters.
D3DPRESENT_PARAMETERS params;
ZeroMemory( &params, sizeof( params ) );
params.BackBufferWidth   = 640;
params.BackBufferHeight  = 480;
params.Flags             = D3DPRESENTFLAG_10X11PIXELASPECTRATIO;

// Initialize the other members of D3DPRESENT_PARAMETERS and create the device.
// ...

// Setup the projection transformation matrix.
XGMATRIX matProjection;
XGMatrixPerspectiveFovLH( &matProjection, // The projection transformation matrix.
                          XG_PI / 4,      // Field of view, in radians.
                          4.0f / 3.3f,    // The aspect ratio.
                          1.0f,           // The Z-value of the near view plane.
                          100.0f);        // The Z-value of the far view plane.

Notice that in this example, the aspect ratio is not 4:3 but is instead 4:3.3.
