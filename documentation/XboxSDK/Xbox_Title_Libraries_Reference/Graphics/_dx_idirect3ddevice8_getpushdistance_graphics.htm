IDirect3DDevice8::GetPushDistance

Allows the title to compute how close the GPU is to releasing a resource, or how close it is to idling or waiting.

DWORD GetPushDistance(
    DWORD Handle
);

Parameters

Handle
[in] Determines from which point to measure the distance. Handle must be either the handle to a fence (as returned by IDirect3DDevice8::InsertFence), or a member of the D3DDISTANCETYPE enumerated type.

Return Values

Returns the number of fences that the GPU must process before hitting the fence (or idling, or waiting) as specified in Handle.

Remarks

Querying the number of fences is a very light-weight operation, and so may be done repeatedly with little overhead for the CPU, and no overhead for the GPU. Fences are automatically inserted into the push-buffer at regular intervals (as dictated by the KickOffSize parameter of IDirect3D8::SetPushBufferSize) and whenever InsertFence is called.

Note    GetPushDistance can't return a good predictor of actual clock time, as that depends greatly on the complexity of the rendering operations that have been recorded in the push-buffer.

Requirements

Header: Declared in D3d8.h.

Import Library: Use D3d8.lib.
