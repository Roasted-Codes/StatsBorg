IDirect3DDevice8::SetTimerCallback

Sets a callback function that will execute at a certain time.

HRESULT SetTimerCallback(
    ULONGLONG Time,
    D3DCALLBACK pCallback,
    DWORD Context
);

Parameters

Time
[in] Specifies the absolute time (in rdtsc units) when the callback should execute.
pCallback
[in] Address of the callback function of type D3DCALLBACK that is called after the period of time given in Time has passed. Setting pCallback to NULL cancels pending callbacks.
Context
[in] A 32-bit value that is passed to the callback. This can be set to any value the title desires. It will be passed as the Context parameter of the callback routine.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

The callback function will get called only once; however, the callback function itself can call SetTimerCallback in order to reschedule itself.

SetTimerCallback is called at the time of deferred procedure calls (DPC) and should do little processing. Only a few XTL functions are safe to call at DPC level: The Interlocked functions are safe to use, as are the following XTL functions, if given a valid object handle:

  • SetEvent
  • ResetEvent
  • PulseEvent
  • ReleaseSemaphore
  • ReleaseMutex

When these functions are called at DPC with an invalid handle, their behavior is undefined.

Do not perform floating-point math, SSE instructions, and MMX instructions during DPC unless the XSaveFloatingPointStateForDpc and XRestoreFloatingPointStateForDpc functions are used.

Requirements

Header: Declared in D3d8.h.

Import Library: Use D3d8.lib.
