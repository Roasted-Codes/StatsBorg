XOnlineContentSetSecurityKey

Sets the security key used to decrypt the package keys of specially-signed title content received from PartnerNet.

Note    This function is implemented in the debug xonlined.lib only, should not be used by retail (shipping) titles, and is provided for testing purposes only.

HRESULT XOnlineContentSetSecurityKey(
    const BYTE *pbSecretKey
);

Parameters

pbSecretKey
[in] The 128-bit extra security key that was used to sign the content package.

Return Values

If the function succeeds, the return value is S_OK.

If the function is called in a retail build, it returns E_NOTIMPL.

Remarks

If desired, content sent to PartnerNet can be encrypted with an extra key when building the content package (otherwise, content packages on PartnerNet can potentially be downloaded and viewed by other publishers). XOnlineContentSetSecurityKey sets the key to use to decrypt that content when it is retrieved from PartnerNet using XOnlineContentInstall. To install an offering that has the extra encryption, call XOnlineContentSetSecurityKey before XOnlineContentInstall.

When building a content package, the extra security key is specified as the SecurityKey value in the content metadata file for the package. BuildOffer looks for the SecurityKey value in the metadata and signs the package with the extra key if found. The 128-bit key is a byte array specified as a big-endian hexadecimal value; the same byte array must be passed as the pbSecretKey parameter to XOnlineContentSetSecurityKey when the content is later installed.

Once XOnlineContentSetSecurityKey has been called to set the extra security key, there is no way to 'unset' the key (other than rebooting the console), and there is no 'identity' key that would produce the same result as having not signed the package with an extra key. Therefore publishers should extra-sign either all or none of the content they send to PartnerNet (or, after switching to either method, only download content that matches the new method being used). Before setting the extra key (with XOnlineContentSetSecurityKey) all extra-signed content is unreachable, and, after setting the extra key, only content signed with that extra key is reachable.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonlined.lib.
