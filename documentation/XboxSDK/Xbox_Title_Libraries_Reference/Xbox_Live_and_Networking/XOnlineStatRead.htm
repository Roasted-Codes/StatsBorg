XOnlineStatRead

Initiates a task to retrieve statistics for the specified users. 

HRESULT XOnlineStatRead(
    DWORD dwNumStatSpecs,
    const XONLINE_STAT_SPEC *pStatSpecs,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwNumStatSpecs
[in] Number of XONLINE_STAT_SPEC structures in the array pointed to by the pStatSpecs parameter.
pStatSpecs
[in] Pointer to an array of XONLINE_STAT_SPEC structures. Each structure specifies a user and the statistics to retrieve for that user. The user is specified by the xuidUser member of the structure. See the Remarks section for more information.
hWorkEvent
[in, optional] Optional. Handle to a caller-created event object. This object will be signaled when there is work to do by calling XOnlineTaskContinue. The caller may pass in NULL instead, indicating that the caller will poll XOnlineTaskContinue itself.
phTask

[out] Address of an XONLINETASK_HANDLE that receives the statistics retrieval task. This task sends the statistics request to the Statistics Service and receives the response. This handle must be pumped to completion by XOnlineTaskContinue. The task is complete when XOnlineTaskContinue returns XONLINETASK_S_SUCCESS. After the task completes, call XOnlineStatReadGetResult to retrieve the statistics. After retrieving the statistics, call XOnlineTaskClose to close the task.

Note    To throttle requests sent to the Statistics Service, XOnlineTaskContinue will return XONLINE_E_TASK_ABORTED_BY_DUPLICATE if the task is still being pumped when a second operation is started.

Return Values

The XOnlineStatRead function always returns S_OK.

The following table lists the possible values that XOnlineTaskContinue may return when called with the handle for the task created by XOnlineStatRead.

Value Description
XONLINE_E_STAT_TOO_MANY_SPECS You are trying to get too many specs in a single call. Send fewer specs per call.
XONLINE_E_STAT_TOO_MANY_STATS You are trying to read too many stats for a given user. Send fewer stats per user.
XONLINE_E_STAT_BAD_REQUEST The message could be empty, the attribute count could be zero, and so on. Check the values you are passing in. If the values are all correct, this error could indicate a problem on the server.

Remarks

The XOnlineStatRead function requests statistics from the Statistics Service.

Each XONLINE_STAT_SPEC structure in the array pointed to by the pStatSpecs parameter specifies the statistics to retrieve for a single user. The user is specified by the xuidUser member of the structure. The dwLeaderBoardID member specifies the leaderboard with which the statistics are associated. The dwNumStats member specifies the number of statistics to retrieve. Each statistic is represented by an XONLINE_STAT structure in the array pointed to by the pStat member. Therefore, dwNumStats also specifies the number of structures in this array. For each XONLINE_STAT structure, the title should provide a value for the wID member that corresponds to the statistic to retrieve. A statistic is defined either by the title or by the Statistics Service. The following table lists the wID values for service-defined statistics.

Identifier Description Type
XONLINE_STAT_RANK Rank statistic. Ranks have a type of XONLINE_STAT_LONG (32 bits).
XONLINE_STAT_RATING Rating statistic. Ratings have a type of XONLINE_STAT_LONGLONG (64 bits).
XONLINE_STAT_NICKNAME User nickname. Nicknames have a type of XONLINE_STAT_LPCWSTR (variable-length string).
XONLINE_STAT_LEADERBOARD_SIZE Number of entries on the leaderboard. The leaderboard size has a type of XONLINE_STAT_LONG (32 bits).
XONLINE_STAT_ATTACHMENT_PATH Path to a statistics attachment. Attachment paths have a type of XONLINE_STAT_LPCWSTR (variable-length string).
XONLINE_STAT_ATTACHMENT_SIZE Size of a statistics attachment. Attachment sizes have a type of XONLINE_STAT_LONG (32 bits).
XONLINE_STAT_UNIT_ACTIVITY_COUNTER Activity counter for a unit. Unit activity counters have a type of XONLINE_STAT_LONG (32 bits).
XONLINE_STAT_UNIT_LAST_ACTIVITY_DATE Last activity date for a unit. Unit last activity dates have a type of XONLINE_STAT_LONGLONG (64 bits).

Note    Leaderboards are also referred to as scoreboards.

If the statistic specified by the wID member is not found, the Statistics Service sets the type member to XONLINE_STAT_NONE.

The rank statistic is computed by the Statistics Service. Although the rank statistic can be retrieved by the XOnlineStatRead function, it cannot be set by the XOnlineStatWrite function.

The XOnlineStatRead function can retrieve statistics for users other than those that are logged in.

If the statistics for a user include a statistic of type XONLINE_STAT_ATTACHMENT_PATH, the title may download an "attachment" that contains title-specific content related to the user's statistics. Use the XOnlineStorageDownload function to download the attachment.

For maximum performance, the title should combine reads for multiple users into a single function call. XOnlineStatRead can retrieve leaderboard stats for up to 5 leaderboards per user, for up to 4 users for a total of 20 user/leaderboard combinations per call. However, to preserve server performance, titles are expected to read from no more than five leaderboards per player in a single call. If this is not sufficient for your game needs, please contact your technical game manager (TGM). To prevent the Statistics Service from overloading, stat reads are automatically throttled by the XOnline library to a maximum of one request every 300 milliseconds, no matter how many times or how often XOnlineStatRead is called.

This function must not be called more often than once a second between game sessions. During a game session, this function can be called only once at the beginning, once at the end, and every five minutes if the game session lasts longer than five minutes. Note that this function will assert if you violate the usage constraints. Contact your TGM or consult the Technical Certification Requirements for details.

This function can be called only with direct user input related to Statistics.
