XOnlineMessageEnableReceivingFamilyTitleIDs

Specifies which family title IDs are valid for messages.

HRESULT XOnlineMessageEnableReceivingFamilyTitleIDs(
    DWORD dwNumTitleIDs,
    const DWORD *pdwTitleIDs
);

Parameters

dwNumTitleIDs
[in] Number of family title identifiers to monitor. The maximum number of identifiers is four.
pdwTitleIDs
[in] An array of the title IDs.

Return Values

Returns S_OK if successful, an error code otherwise.

E_INVALIDARG is returned when the family title identifiers supplied are invalid.

Remarks

XOnlineMessageEnableReceivingFamilyTitleIDs allows the title to begin receiving messages sent using one of the family title IDs (see XOnlineMessageSetSendingFamilyTitleID) in addition to messages sent using the game's title ID. This function should be called after XOnlineStartup and before XOnlineLogon to ensure notifications and messages from family title IDs are immediately available. Titles generally do not need to call this function again.

To disable messages for family title IDs, set the dwNumTitleIDs parameter to zero and the pdwTitleIDs parameter to NULL.

Attempting to use this function to enable a family title ID without first including that ID in the title's certificate will result in an error message displayed in the debug output as well as the function not working correctly. A family title ID can be included in a title's certificate by opening the project properties dialog box in Visual Studio, and adding the family title ID to the alternate title IDs.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
