XOnlineMutelistAdd

Adds a player to the specified user's mutelist.

HRESULT XOnlineMutelistAdd(
    DWORD dwUserIndex,
    XUID xUserID
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the user whose mutelist will be modified. The user must have been successfully logged on with a previous call to XOnlineLogon.
xUserID
[in] Unique identifier of the user to be locked out.

Return Values

If XOnlineMutelistAdd succeeds, the return value is S_OK.

If XOnlineMutelistAdd fails, the return value is one of the following error codes.

Value Description
XONLINE_E_NOTIFICATION_NO_TASK The mutelist startup task is not active. Call XOnlineMutelistStartup first in order to create the mutelist startup task.
XONLINE_E_NOTIFICATION_SELF The user specified themselves as the player to add to the mutelist. Don't allow users to add themselves to their own mutelist. If you haven't caught this in advance, use an error sound to indicate that they cannot do this.
E_OUTOFMEMORY Insufficient memory. Free some memory.
Other See Online Errors for information about other possible errors.

Remarks

This function adds the user specified in xUserID to the mutelist of the user specified in dwUserIndex. To later remove the specified user from the mutelist, call XOnlineMutelistRemove.

The title must successfully call XOnlineMutelistStartup once before making any calls to XOnlineMutelistAdd or XOnlineMutelistRemove. The XOnlineMutelistStartup function creates an asynchronous task that must be pumped in order to add or remove users from the mutelist. To add a user to the mutelist, call XOnlineMutelistAdd, then pump this asynchronous task by calling XOnlineTaskContinue with the handle returned by XOnlineMutelistStartup.

The task created by XOnlineMutelistStartup never completes. That is, XOnlineTaskContinue never returns XONLINETASK_S_SUCCESS. To determine if the specified user was added to the mutelist, check whether the hEvent parameter specified in the call to XOnlineMutelistStartup is signaled. If this event is unsignaled, then no work needs to be done on the asynchronous task, and the user was added successfully. If no event object was specified in the call to XOnlineMutelistStartup (the hEvent parameter is optional), the title should check XOnlineTaskContinue for the XONLINETASK_S_RUNNING_IDLE return code. This return code indicates that the user was added successfully. Alternatively, the title can determine if the user was added by enumerating the mutelist with XOnlineMutelistGet.

A player should never receive voice communications from another player that they have added to their mutelist. It is the title's responsibility to enforce this requirement by checking a user's mutelist against the source of voice data before playing that voice data. There are further technical certification requirements related to mutelists.
