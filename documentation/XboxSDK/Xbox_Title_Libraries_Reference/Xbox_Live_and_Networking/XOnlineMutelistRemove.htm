XOnlineMutelistRemove

Adds a player to the specified user's mutelist. 

HRESULT XOnlineMutelistRemove(
    DWORD dwUserIndex,
    XUID xUserID
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the user whose mutelist will be modified. The user must have been successfully logged on with a previous call toXOnlineLogon.
xUserID
[in] Unique identifier of the user specified to be removed from the mutelist.

Return Values

If XOnlineMutelistRemove succeeds, the return value is S_OK.

If XOnlineMutelistRemove fails, the return value is one of the following error codes.

Value Description
XONLINE_E_NOTIFICATION_NO_TASK The mutelist startup task is not active. Call XOnlineMutelistStartup first in order to create the mutelist startup task.
XONLINE_E_NOTIFICATION_SELF A user should not be able to mute themselves, so they should never have the option to unmute themselves. A user should be able only to remove other muted players from their mute list. If you haven't caught this in advance, then use an error sound to indicate that they can not do this.
E_OUTOFMEMORY Insufficient memory. Free some memory.
Other See Online Errors for information about other possible errors.

Remarks

This function removes the user specified in xUserID from the permanent mutelist of the user specified in dwUserIndex.

The title must call XOnlineMutelistStartup once successfully before making any calls to XOnlineMutelistAdd or XOnlineMutelistRemove. The XOnlineMutelistStartup function creates an asynchronous task that must be pumped in order to add or remove users from the mutelist. To remove a user from the mutelist, call XOnlineMutelistRemove, then pump this asynchronous task by calling XOnlineTaskContinue with the handle returned by XOnlineMutelistStartup.

The task created by XOnlineMutelistStartup never completes. That is, XOnlineTaskContinue never returns XONLINETASK_S_SUCCESS. To determine if the specified user was removed from the mutelist, check whether the hEvent parameter specified in the call to XOnlineMutelistStartup is signaled. If this event is unsignaled, then no work needs to be done on the asynchronous task, and the user was removed successfully. If no event object was specified in the call to XOnlineMutelistStartup (the hEvent parameter is optional), the title should check XOnlineTaskContinue for the XONLINETASK_S_RUNNING_IDLE return code. This return code indicates that the user was removed successfully. Alternatively, the title can determine if the user was removed by enumerating the mutelist with XOnlineMutelistGet.

Blocking a player from making friend requests does not automatically add that player to the mutelist.

Requirements
