XOnlineStorageDownloadToMemory

Downloads a storage item that was previously uploaded using the XOnlineStorageUploadFromMemory function.

HRESULT XOnlineStorageDownloadToMemory(
    DWORD dwFacility,
    DWORD dwUserIndex,
    LPCWSTR wszStoragePath,
    PBYTE pbReceiveBuffer,
    DWORD cbReceiveBuffer,
    DWORD dwDownloadFlags,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwFacility

[in] Indicates storage facility. May be one of the following:
Value Description
XONLINESTORAGE_FACILITY_STATS Game-clips (stats with attachments) facility.
XONLINESTORAGE_FACILITY_TEAMS Teams storage facility.
XONLINESTORAGE_FACILITY_PER_TITLE Title-managed global storage.
XONLINESTORAGE_FACILITY_PER_USER_TITLE Per-title, per-user storage.

For a complete list of facility values, see the XONLINESTORAGE_FACILITY page.

dwUserIndex
[in] The controller index (zero-based) of the user for whom to download the attachment.
wszStoragePath
[in] The service's storage path for the attachment (this is typically created by XOnlineStorageCreateServerPath).
pbReceiveBuffer
[in, out] Buffer to contain the downloaded data.
cbReceiveBuffer
[in] Size of the buffer, in bytes.
dwDownloadFlags
[in] Flags that qualify the download process. This value must be zero.
hWorkEvent
[in, optional] Handle to an event that is signaled when there is work to do on the asynchronous task returned by the phTask parameter. This parameter can be NULL if no such synchronization is required.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle. Use this handle in calls to XOnlineTaskContinue to process the file download.

Return Values

If XOnlineStorageDownloadToMemory succeeds, the return value is S_OK.

If XOnlineStorageDownloadToMemory fails, the return value is one of the following error codes:

Value Description
XONLINE_E_STORAGE_INVALID_REQUEST The request is invalid.
XONLINE_E_STORAGE_ACCESS_DENIED The client doesn't have sufficient permissions to download the file.
XONLINE_E_STORAGE_FILE_NOT_FOUND The file to download was not found.
XONLINE_E_STORAGE_INVALID_ACCESS_TOKEN The access token signature is invalid.
XONLINE_E_STORAGE_CANNOT_FIND_PATH Name resolution failed.
XONLINE_E_STORAGE_FILE_IS_ELSEWHERE The client received a redirection request.
XONLINE_E_STORAGE_INVALID_STORAGE_PATH Invalid storage path.
XONLINE_E_STORAGE_INVALID_FACILITY Invalid facility code.
XONLINE_E_STORAGE_UNSUPPORTED_CONTENT_TYPE Returned when the content type of the storage item is not supported by this operation (see remarks).
ERROR_INSUFFICIENT_BUFFER Buffer isn't large enough to hold content.

Note    Titles should handle the case gracefully where the content isn't found on title managed storage (that is, should not display a dialog box to the user that says content could not be downloaded).

Remarks

This is very similar to XOnlineStorageDownload, except that it downloads storage items previously uploaded using XOnlineStorageUploadFromMemory to the specified buffer. Upon success, the downloaded data can be retrieved by a call to the XOnlineStorageDwonloadToMemoryGetResults function.

This function is intended for smaller downloads such as voice mails or audio clips that just get stored in memory and never get written to disk. This function does not have a maximum size for downloads except that it is constrained by the size of the buffer passed into the function.

The integrity of the data is verified by the function prior to being returned to the caller. However, the downloaded data is returned as-is without additional security features. Titles are not recommended to save such downloaded to disk without additional security measures.

XOnlineStorageDownloadToMemory will download all storage items that were previously uploaded using XOnlineStorageUploadFromMemory. Any attempt to download storage items uploaded with XOnlineStorageUpload or XOnlineStorageUploadByServerPath will result in the XONLINE_E_STORAGE_UNSUPPORTED_CONTENT_TYPE error.

Requirements

Import Library: Use Xonline.lib.
