Offerings, Billing, and Content Download Overview

To facilitate and standardize revenue gathering for new downloadable content, the offering functions provide a quick and easy way to initiate and utilize billing relationships with consumers in a way that is secure, friendly, intuitive, and unobtrusive. Titles use the offering functions to purchase subscriptions and content from the Xbox Live service.

The offering functions provide the necessary tools for the game developer to purchase both recurring and one-time Xbox Live offerings. They make it simple for customers to make purchases inside a game, while ensuring that the user's information is accessible only to the intended audience and that the transaction is done securely.

The offering functions provide game developers with a mechanism to make online purchases that pertain only to the Xbox Live service. These functions are not meant to be general-purpose billing instruments and are not used to make purchases outside of the context of the Xbox Live Service.

To make any type of purchase, the title must first have access to a user account that has been properly authenticated with the appropriate services. A user account must already exist on the local Xbox hard disk, or on a Xbox Memory Unit (MU) that is connected to the Xbox. In addition, the title must have already performed the authentication process (see XOnlineLogon).

Once an account has been established on an Xbox and the game title has authenticated the user, the user is permitted to purchase one or more offerings. Offerings represent something that may be purchased or acquired—access to a service, a piece of content, and so on.

Titles can call XOnlineOfferingIsNewContentAvailable as a quick check to determine if new content (content offerings only, not subscription offerings) is available. If new content is available, titles can expose or emphasize a content download option for the user. Titles using the Xbox Content Download Manager can simply launch the downloader (by calling XLaunchNewImage) to handle the remaining content enumeration and download. Otherwise, the title must perform the remaining steps itself.

To begin the purchase process, titles first call XOnlineOfferingEnumerate to enumerate the content and subscriptions currently available for purchase and download. The returned results are displayed to the user. The user then has a choice of whether to proceed or go back. The user can then select a specific item to view, at which time the title calls XOnlineOfferingDetails to get more information regarding a specific offer, including the final price (if applicable). The next step depends on whether the offering is free or not, which can be determined by testing the fOfferingIsFree member of the XONLINE_PRICE structure. If the item is not free, and the user chooses to purchase the selected offering, the title calls XOnlineOfferingPurchase to complete the purchase. If an offering is free, or once it has been purchased, the title will then have access to the resources that the offering provides, and can download and install the content using the XOnlineContentInstall function. For subscription offerings that have no associated downloadable content, calling XOnlineContentInstall is not necessary.

Titles must provide users with the option to cancel subscriptions (content offerings represent one-time charges and cannot be canceled). When the user so chooses, the title calls XOnlineOfferingCancel to cancel the offering. Note that this applies only to offers with recurring fees—it does not apply to offers that are one-time charges. If a recurring charge offering is canceled, no further charges will be applied to the user. All previous charges are non-refundable, and unused portions within the subscription period will not be prorated or refunded. The title must be aware of these restrictions and inform the user appropriately.

Note that in order to purchase offerings, the user must have already purchased a standard service, which gives him or her access to enumerate more content and offerings. The standard service is purchased when the user signs up for the first time and creates an account. Players use the Xbox Dashboard to create their user account and to complete the initial sign-up. Thus, no functions are exposed to the title developers to burden them with these tasks.

Each title, over its lifetime, is limited to a total of 512 unique offerings.

Checking for New Content

The XOnlineOfferingIsNewContentAvailable function is designed to be used by both full-function online titles and by titles that are providing simple content download with the Xbox Content Download Manager. In either case, the function provides a quick, simple method of determining whether new content offerings are available, without requiring either of the following: 1) that the title first log onto the Xbox Live service, and 2) that any valid user accounts have previously been created on the calling box console.

XOnlineOfferingIsNewContentAvailable indicates whether new content that matches specified bit flags is available. Content is considered "new" if it was not available the last time the title enumerated available offerings (or launched the downloader). As a result, a title can perform a check for new content (at startup or later during actual gameplay) without requiring user input to log onto Xbox Live. A title can also perform a check for new content without actually requiring the user to purchase an Xbox Live account. If new content is available, the title can present the user with a new or emphasized download content option that will either enumerate available offerings or launch the downloader. This frees players from repeatedly checking for new content to download, only to find none available.

Titles using the Xbox Content Download Manager, after launching the downloader, need perform no further online work or calls. Other titles will need to perform the enumeration and purchasing steps described next.

Enumerating Offerings

XOnlineOfferingEnumerate is used to enumerate offerings that are available on an online server. Results are returned in batches of offerings until all available offerings have been enumerated.

When calling XOnlineOfferingEnumerate, the title specifies the logged-on user for whom the offerings will be enumerated, along with the types of offerings to be enumerated. Only registered users (not guests) can enumerate offerings. The title can specify that only subscription or content offerings (or both) be enumerated, as well as restrict the enumeration to offerings that have certain title-specific flags set. There can be multiple descriptions for each available offering (for example, one for each supported language), and the title can specify an index into the available descriptions to determine which description should be retrieved for each offering. The number and meaning of the various descriptions is title specific and is provided when the offerings are initially submitted. Except for the user and the device type to be enumerated, these enumeration options are specified in an XONLINEOFFERING_ENUM_PARAMS structure passed to XOnlineOfferingEnumerate.

XOnlineOfferingEnumerate needs a buffer to use internally while the enumeration is being processed. The title can either allocate a sufficiently large buffer and pass it to XOnlineOfferingEnumerate, or it can allow the function to allocate a buffer internally. The XOnlineOfferingEnumerateMaxSize function is used to determine the buffer size needed to contain a specified number of enumeration records. For example, if the title wishes to receive at most ten offerings in each returned results batch, it can specify that number when calling XOnlineOfferingEnumerateMaxSize. The returned required buffer size will be large enough to hold ten offerings, and each batch of returned results will contain at most ten records.

XOnlineOfferingEnumerate does not immediately perform the enumeration work and return the results. Instead, it creates an asynchronous online task and returns a handle to the created task. The return value of XOnlineOfferingEnumerate indicates the success or failure of the task creation, not of the entire enumeration operation. As time permits, the title should call XOnlineTaskContinue to perform work on the task. XOnlineTaskContinue will continue to return XONLINETASK_S_RUNNING until the task is complete, partial results are available, or an error has occurred.

Return values of XONLINETASK_S_SUCCESS or XONLINETASK_S_RESULTS_AVAIL from the XOnlineTaskContinue call indicate that results are available. XOnlineOfferingEnumerateGetResults will return a batch of currently available results in an array of pointers to XONLINEOFFERING_INFO structures, each containing the information for a single offering. The title should not free (or otherwise manipulate) the buffers containing the returned offerings. The pfMoreResults parameter of XOnlineOfferingEnumerateGetResults is used to return a value indicating whether or not more results are available. If more results are available, the title should resume calling XOnlineTaskContinue until it returns a value other than XONLINETASK_S_RUNNING. The title should then call XOnlineOfferingEnumerateGetResults to retrieve the next batch of results, and so on.

When XOnlineOfferingEnumerateGetResults indicates that no more results are available, the title should close the asynchronous online task handle by calling XOnlineTaskClose. Additionally, if the title allocated a buffer to pass to XOnlineOfferingEnumerate, it should free that buffer when the returned data results no longer need to be accessed.

If XOnlineTaskContinue returns an error code, some problem was encountered while the enumeration was being performed. The task handle should be closed with XOnlineTaskClose. If the title allocated a buffer to pass to XOnlineOfferingEnumerate, it should free that buffer. If desired, the title can re-attempt enumeration, likely after querying the user.

Offering Details

When offerings are enumerated and displayed to the user, each offering will have only a simple description associated with it that briefly describes the content or subscription. Full details for the offering, including the purchase price and extended description, are presented only after the user selects an offering from the list. The user can then see the full offering information, and then choose whether or not to purchase the offering.

XOnlineOfferingDetails is used to retrieve the pricing and detailed description information for an offering. XOnlineOfferingDetails accepts a unique offering identifier (an XOFFERING_ID), part of the information returned for each offering that has been enumerated, and will return the details associated with that offering.

Like XOnlineOfferingEnumerate, XOnlineOfferingDetails requires a buffer for storing the details information while that information is being retrieved. The title can either allocate a sufficiently large buffer itself or allow XOnlineOfferingDetails to do so internally. XOnlineOfferingDetailsMaxSize is used to calculate the proper buffer size needed to store the details information.

Also like XOnlineOfferingEnumerate, XOnlineOfferingDetails does not immediately retrieve and return the details results. Instead, XOnlineOfferingDetails creates an asynchronous online task, returns a handle to that task, and then returns a result code indicating the success or failure of the task creation (rather than of the entire details retrieval operation). The title then periodically calls XOnlineTaskContinue until a completion or error code (that is, any value other than XONLINETASK_S_RUNNING) is returned.

A return value of XONLINETASK_S_RESULTS_AVAIL from XOnlineTaskContinue indicates that the details have been downloaded and are ready to be retrieved. XOnlineOfferingDetailsGetResults can then be called to immediately retrieve the details information. As mentioned above, this details information contains the pricing and extended description of the offering. The pricing information is returned in an XONLINE_PRICE structure that can be converted to a display string by the XOnlineOfferingPriceFormat function.

After the details information has been retrieved, further calls to XOnlineTaskContinue for the details task will return XONLINETASK_S_SUCCESS. XONLINETASK_S_SUCCESS indicates that the task has completed and can be closed with XOnlineTaskClose. If the XOnlineOfferingDetails function allocated its own details buffer internally (rather than the title passing in an allocated buffer), that internal buffer is deallocated and becomes invalid when XOnlineTaskClose is called. As a result, the details information returned by XOnlineOfferingDetailsGetResults in the pDetailsBuffer parameter becomes invalid as well. Therefore, the title should read (and copy if necessary) the returned details data before closing the details task. If the title allocated its own buffer to pass to XOnlineOfferingDetails, the returned details information becomes invalid once the title has freed its own allocated buffer (which it should do only after it has finished accessing the details data).

If XOnlineTaskContinue returns an error code for the details task, some problem was encountered while the information was being retrieved. The task handle should be closed with XOnlineTaskClose. If the title allocated a buffer to pass to XOnlineOfferingDetails, it should free that buffer. The title could attempt to retrieve the details again if so desired.

Purchasing and Canceling Offerings

After reading the detailed information for an offering, the player may choose to purchase that offering. When the player chooses the option of purchasing an offering, the title calls the XOnlineOfferingPurchase function to perform the purchase operation. The purchase function should be called only for offerings that are not free, as determined by the fOfferingIsFree member of the XONLINE_PRICE structure.

XOnlineOfferingPurchase requires the following: 1) the user index of the currently logged-on user who will be making the purchase, and 2) the identifier that uniquely specifies the offering (the same identifier returned by the offering enumeration and used to retrieve the offering details). XOnlineOfferingPurchase creates an asynchronous online task and returns a code indicating the success or failure of the task creation. To perform the purchase, the title then makes repeated calls to XOnlineTaskContinue.

A return value other than XONLINETASK_S_RUNNING from XOnlineTaskContinue indicates that the task has completed. A return value of XONLINETASK_S_SUCCESS indicates that the purchase has succeeded. An error return value indicates that the purchase has failed. There are specific error codes related to purchasing that may be returned, such as a denied purchase or an invalid offering. These error codes have a prefix of XONLINE_E_OFFERING and can be found in xonline.h.

Once a purchase operation has been initiated with XOnlineOfferingPurchase, it is not possible to halt the operation. Prior to task completion, some asynchronous online tasks can be halted by calling XOnlineTaskClose for the task handle. XOnlineOfferingPurchase, however, sends the purchase request to a server, and that request cannot be recalled. Always call XOnlineTaskContinue to completion for the purchase operation, and only then call XOnlineTaskClose to close the task handle.

If the offering purchased is cancelable (usually indicative of a subscription with a recurring charge), the user can later cancel her subscription by choosing to do so from within the game menus. If the user so chooses, the title calls XOnlineOfferingCancel to begin an asynchronous task to cancel the offering. Like XOnlineOfferingPurchase, once the cancel operation has been initiated it cannot be halted by closing the task handle. Continue to call XOnlineTaskContinue until it returns a value other than XONLINETASK_S_RUNNING. Only after a value other than XONLINETASK_S_RUNNING is returned should XOnlineTaskClose be called to close the task handle for the cancel operation.

Once an offering has been successfully purchased, and if it is associated with downloadable content, it must still be downloaded and installed to the local Xbox console in order for the player to make use of the offering. This process is described in the next section.

Installing and Removing Offerings

After a user has purchased content, that offering can be downloaded to the user's Xbox for use. The XOnlineContentInstall function downloads and installs a specified offering. The function creates an asynchronous online task and returns a handle to that task. The return value of XOnlineContentInstall indicates the success or failure of the task creation, not of the entire download and installation operation. The title then calls XOnlineTaskContinue with the installation task handle until a value other than XONLINETASK_S_RUNNING is returned.

While the download and installation is in progress, XOnlineContentInstallGetProgress can be called to determine the progress of the content download. Typically this would be used to display a progress bar, the percentage downloaded, or the time remaining.

When XOnlineTaskContinue finally returns XONLINETASK_S_SUCCESS, the task is finished and has completed successfully. The content in the specified offering has been downloaded from the online server and installed onto the user's local Xbox hard disk. Once the download and installation has completed successfully, the title should verify that the content was correctly downloaded and installed, and that it is valid, signed content. Use XLoadContentSignatures to obtain access to signature data for installed content. Additionally, the title should always perform verification before using installed content in actual gameplay (including in subsequent game sessions) to verify that the content has not been altered.

The XGetContentInstallLocationFromIDs function can be used to determine the directory location where the content was installed, so that the title can find the content for use. Additionally, installed content can be enumerated with XFindFirstContent.

To remove installed content, call XRemoveContent, which removes all data associated with the offering.
