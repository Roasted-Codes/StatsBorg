XONLINEOFFERING_DETAILS

Contains details about an online offering. This structure is returned by the XOnlineOfferingDetailsGetResults function.

typedef struct{
    PBYTE pbDetailsBuffer;  // Pointer to buffer of details blob
    DWORD dwDetailsBuffer;  // Length of details blob
    DWORD dwInstances;  // Count of currently-owned instances
    XONLINE_PRICE Price;  // price structure
    DWORD dwFreeMonthsBeforeCharge;  // free months before charge begins
    DWORD dwDuration;  // duration of the recurring charge (months)
    XONLINE_OFFERING_FREQUENCY Frequency;  // how often charges are made
} XONLINEOFFERING_DETAILS, *PXONLINEOFFERING_DETAILS;

Members

pbDetailsBuffer
Pointer to the details 'blob' buffer.
dwDetailsBuffer
Size, in bytes, of the buffer pointed to by pbDetailsBuffer. The maximum size of the details blob is 64000 bytes.
dwInstances
Number of currently-owned instances of this offering. See Remarks below for more information.
Price
XONLINE_PRICE structure that specifies the price of the offering.
dwFreeMonthsBeforeCharge
Number of months for which the offering is free before a charge begins to occur.
dwDuration
Duration of the recurring charge, in months.
Frequency
An XONLINE_OFFERING_FREQUENCY value that specifies how often charges are made for the offering.

Remarks

The dwInstances member specifies the number of instances of this offering that are currently owned either by the user or by the Xbox console. For a subscription, this member indicates the number of times the user has purchased the subscription (from any Xbox, not necessarily the current one). For content, it indicates the number of times that any user has purchased the offering while logged in from the current Xbox console. Titles can choose to use this instance count however they choose (for example, by only allowing a single purchase of a given piece of content). The XOnlineOfferingPurchase function will allow a title to make multiple purchases of the same offering, with no restrictions.

In the case of a free offer, dwInstances will always be zero.

When the details task is closed with XOnlineTaskClose, the details buffer returned in pbDetailsBuffer will be deallocated and become invalid if the buffer was allocated by XOnlineOfferingDetails rather than passed in by the title.

Use XOnlineOfferingPriceFormat to convert an XONLINE_PRICE structure to a formatted text string.
