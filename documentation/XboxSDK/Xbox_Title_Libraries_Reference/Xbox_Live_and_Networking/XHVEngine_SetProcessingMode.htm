IXHVEngine::SetProcessingMode

HRESULT SetProcessingMode(
    DWORD dwLocalPort,
    XHV_PROCESSING_MODE processingMode
);

Parameters

dwLocalPort
[in] Controller port of the local talker.
processingMode

[in] An XHV_PROCESSING_MODE value specifying the current processing mode to use for the local talker. Must be one of the following values:
Value Description
XHV_INACTIVE_MODE Inactive. No voice chat encoding or recognition processing is done for this local talker.
XHV_LOOPBACK_MODE Loopback mode. Sends the microphone data through the encoder and decoder and plays it back through the communicator headset. This mode is often used to try different voice masks and to test communication effects. It is generally not used while playing the game.
XHV_VOICECHAT_MODE Voice communication mode. While in this mode, all data coming from this local talker's voice headset is encoded. The encoding process requires CPU cycles.
XHV_SR_MODE Speech recognition mode. Start recognizing speech data for this local talker. The speech recognition (SR) engine will be active (that is, executing and using CPU cycles) for as long as this mode is selected.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Prior to setting a local talker to a specific processing mode, EnableProcessingMode must have been called to enable XHV engine processing for that specific mode. After a local talker has been registered, additional processing modes cannot be enabled in IXHVEngine.

For speech recognition mode (XHV_SR_MODE), call SetProcessingMode only after successfully calling SelectVocabulary.

Requirements

Header: Declared in Xhv.h.

Import Library: Use Xvoice.lib.
