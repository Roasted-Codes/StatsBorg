XNetGetBroadcastVersionStatus

Returns a value that indicates whether the local Xbox console has received broadcast packets from an incompatible version of the same title. 

DWORD XNetGetBroadcastVersionStatus(
    BOOL fReset
);

Parameters

fReset
Value indicating whether to reset the status after returning the current status value. If this parameter is TRUE, the status is reset. If this parameter is FALSE, the status persists until the next call to XNetGetBroadcastVersionStatus.

Return Values

If no incompatible broadcast packets have been received, the return value is zero. Otherwise, the return value is one of the following.

Value Description
XNET_BROADCAST_VERSION_OLDER The local Xbox console received broadcast packets from an older incompatible version of title.
XNET_BROADCAST_VERSION_NEWER The local Xbox console received broadcast packets from a more recent incompatible version of title

Remarks

A title can receive broadcast packets from an incompatible version of the same title if the two XBEs have the same title ID, but different LAN keys and version numbers. This situation occurs if one of the XBEs receives an auto-update, but the other XBE does not.

The code below demonstrates calling XNetGetBroadcastVersionStatus. In this example, the function resets the status after returning the current value: 


    DWORD dwStatus = XNetGetBroadcastVersionStatus( TRUE );
    if( dwStatus != 0 )
    {
        return TRUE;
    }
    return FALSE;

See the WinsockPeer sample for an example of calling XNetGetBroadcastVersionStatus in the context of a complete program.

Requirements

Header: Declared in Winsockx.h.

Import Library: Use Xonline.lib or xnets.lib.
