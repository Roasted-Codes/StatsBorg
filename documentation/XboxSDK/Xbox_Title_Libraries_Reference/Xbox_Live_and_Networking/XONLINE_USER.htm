XONLINE_USER

Contains information about an online user.

typedef struct{
    XUID xuid;
    CHAR szGamertag[XONLINE_GAMERTAG_SIZE];
    DWORD dwUserOptions;
    BYTE passcode[XONLINE_PASSCODE_LENGTH];
    BYTE reserved[XONLINE_USER_RESERVED_SIZE];
    HRESULT hr;
} XONLINE_USER, *PXONLINE_USER;

Members

xuid
XUID structure that contains information about the user.
szGamertag

Null-terminated string that contains the user's Gamertag. The format of the Gamertag must adhere to the following rules:
  • Minimum of one character in length.
  • Maximum of 15 characters in length.
  • Can include only the following characters: A–Z, a–z, 0–9, and [space].
  • Cannot begin with a [space] character.
  • Cannot have multiple contiguous [space] characters.
  • Must begin with an alphabetic character (A–Z, a–z).
  • Cannot end with space.

The Gamertag is case insensitive for comparisons, but case should be preserved when stored. For example, Ubba and UBBA are the same and hence cannot both exist as accounts in the system, but which ever one was entered into the system, the case is preserved.

dwUserOptions
Value that specifies user options. Can be a combination of the following values:
Value Description
XONLINE_USER_OPTION_CAME_FROM_MU Indicates the user account information came from an Xbox Memory Unit (MU).
XONLINE_USER_OPTION_REQUIRE_PASSCODE Indicates the user requires a pass code to log on.

For a user account that came from an Xbox MU, you can use the following values to determine the port and slot associated with that user.

Value Description
XONLINE_USER_OPTION_MU_PORT_MASK Value to use with dwUserOptions to determine the port associated with the user.
XONLINE_USER_OPTION_MU_PORT_SHIFT Value to shift right after using the XONLINE_USER_OPTION_MU_PORT_MASK value.
XONLINE_USER_OPTION_MU_SLOT_MASK Value to use with dwUserOptions to determine the slot associated with the user.
XONLINE_USER_OPTION_MU_SLOT_SHIFT Value to shift right after using the XONLINE_USER_OPTION_MU_SLOT_MASK value.
passcode
Array of BYTE values that contain the pass code used to authorize access to this user account. If the dwUserOptions member does not contain the XONLINE_USER_OPTION_REQUIRE_PASSCODE flag, this member is ignored.
reserved
Reserved. Do not use.
hr
HRESULT value that is used to report any warnings or errors specific to this user after sign-in.
Value Description
XONLINE_S_LOGON_USER_HAS_MESSAGE The sign-in succeeded, but the user has at least one unread Xbox Live message. The title must provide the option of booting into the Xbox Dashboard to allow the user to view the message. Use XLaunchNewImage with a reason code of XLD_LAUNCH_DASHBOARD_ACCOUNT_MANAGEMENT to boot to the Dashboard.
XONLINE_E_LOGON_USER_ACCOUNT_REQUIRES_MANAGEMENT The sign-in failed because of a problem with the user's account. The title must provide the option of booting into the Xbox Dashboard to allow the user to address the problem. Use XLaunchNewImage with a reason code of XLD_LAUNCH_DASHBOARD_ACCOUNT_MANAGEMENT to boot to the Dashboard.

Remarks

Use the XOnlineGetUsers function to obtain an array of XONLINE_USER structures that contain user information stored on the local Xbox console's hard disk or in memory units.

Use the XOnlineGetLogonUsers function to obtain an array of XONLINE_USER structures for users that are currently logged on.

Use the XOnlineAreUsersIdentical macro to determine if two different XONLINE_USER structures represent the same user.
