XOnlineFriendsRequestByName

The XOnlineFriendsRequestByName function enables a user to send a request to another user asking that the two users be added to each other's Friends lists. The function differs from XOnlineFriendsRequest in that XOnlineFriendsRequestByName enables the user to specify the recipient of the Friends request by Gamertag rather than by XUID.

Note    It is recommended that titles use XOnlineFriendsRequestByNameEx to allow voice attachments.

HRESULT XOnlineFriendsRequestByName(
    DWORD dwUserIndex,
    LPCSTR lpUserName,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the user making the request. The user must have been successfully logged on with a previous call to XOnlineLogon.
lpUserName
[in] Pointer to a null-terminated string that contains the Gamertag of the recipient of the Friend request.
hWorkEvent
[in, optional] Handle to an event that is signaled when there is work to do on the asynchronous task. This parameter can be NULL if no such event is required.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle. Use this handle in calls to XOnlineTaskContinue to perform work on the Friend request.

Return Values

If XOnlineFriendsRequestByName succeeds, it returns S_OK.

If XOnlineFriendsRequestByName fails, it returns one of the following error codes.

Value Description
XONLINE_E_NOTIFICATION_SELF The user requested to add themselves to their own Friends List. Don't allow a user to add themselves to their own Friends List. If you receive this error, display an error message, or generate a sound to indicate that this is not a valid action.
XONLINE_E_NOTIFICATION_FRIEND_PENDING The current user previously sent a Friend request to the specified user, but has not yet received a response.
XONLINE_E_NOTIFICATION_USER_NOT_FOUND The user specified by the lpUserName parameter was not found on the Xbox Live service.
XONLINE_E_NOTIFICATION_LIST_FULL The user's Friends List is full. An appropriate way to handle this return code is to display the message, "Unable to add the specified friend because your Friends List is full. A - continue" This error should occur only if the title is currently enumerating the Friends List. You can avoid this error by verifying that the user has fewer than 100 friends before requesting a new one. There are situations in which you will not receive this error even though the list is full.
XONLINE_E_NOTIFICATION_OTHER_LIST_FULL The recipient's Friends List is full. An appropriate way to handle this return code is to display the message, "Unable to add the specified friend because his or her Friends List is full. A - continue"
XONLINE_E_NOTIFICATION_BLOCKED The recipient has responded with XONLINE_REQUEST_BLOCK.
XONLINE_E_NOTIFICATION_USER_ALREADY_EXISTS The user is already on the list. An appropriate way to handle this return code is to display the message, "This player is already on your Friends List. A - continue" Or, alternatively you can do nothing. If the user forgot that a particular friend was on his list, and invites that user again, it will simply seem as though the invited user accepted immediately. This error should occur only if the title is currently enumerating the Friends List.
Other See Online Errors for information about other possible errors.

Remarks

The return values listed above may be returned by either XOnlineFriendsRequestByName itself, or by XOnlineTaskContinue when called with the task handle returned by XOnlineFriendsRequestByName.

Because XOnlineFriendsRequestByName is not as efficient as XOnlineFriendsRequest, titles should, if possible, use XOnlineFriendsRequest instead.

Requirements

Import Library: Use Xonline.lib.
