IXHVEngine::SetPlaybackPriority

Sets the destination priority of voice data from a specified remote talker.

HRESULT SetPlaybackPriority(
    XUID xuidRemoteTalker,
    DWORD dwLocalPort,
    XHV_PLAYBACK_PRIORITY playbackPriority
);

Parameters

xuidRemoteTalker
[in] The XUID of the registered remote talker.
dwLocalPort
[in] Controller port of the local talker. To assign priorities for speaker playback, specify XHV_PLAYBACK_TO_SPEAKERS.
playbackPriority

[in] The priority to assign to voice data coming from the remote talker specified in xuidRemoteTalker. Can be one of the following values, or any value between XHV_PLAYBACK_PRIORITY_MAX and XHV_PLAYBACK_PRIORITY_MIN:
Value Description
XHV_PLAYBACK_PRIORITY_MAX Highest possible priority.
XHV_PLAYBACK_PRIORITY_MIN Lowest possible priority.
XHV_PLAYBACK_PRIORITY_NEVER The destination (local talker or speaker output) will not be sent voice data from the specified remote talker.

Values between XHV_PLAYBACK_PRIORITY_MAX (0) and XHV_PLAYBACK_PRIORITY_MIN (0xFFFF) are used to specify relative priorities.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Each local talker, as well speaker playback, can assign priorities in decoding remote talker data. The XHV engine will decode data with the highest priority first, up to the engine's maximum number of playback streams (see SetMaxPlaybackStreamsCount).

SetMixBinMapping can be used to alter the default mixbins used when sending remote talker voice data to destination local talkers or speaker playback.

Requirements

Header: Declared in Xhv.h.

Import Library: Use Xvoice.lib.
