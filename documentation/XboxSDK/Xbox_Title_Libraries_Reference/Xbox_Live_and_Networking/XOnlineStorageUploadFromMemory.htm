XOnlineStorageUploadFromMemory

Uploads a file attachment to title storage from memory.

HRESULT XOnlineStorageUploadFromMemory(
    DWORD dwFacility,
    DWORD dwUserIndex,
    LPCWSTR wszStorageFileName,
    FILETIME ftServerExpirationDate,
    PBYTE pbDataToUpload,
    DWORD cbDataToUpload,
    DWORD dwUploadFlags,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwFacility

[in] Indicates storage facility. May be one of the following:
Value Description
XONLINESTORAGE_FACILITY_TEAMS Teams storage facility.
XONLINESTORAGE_FACILITY_PER_USER_TITLE Per title, per user storage.

For a complete list of facility values, see the XONLINESTORAGE_FACILITY page.

dwUserIndex
[in] The controller index (zero-based) of the user for whom to download the attachment.
wszStorageFileName
[in] The service's storage path for uploaded file. This can be created using the XOnlineStorageCreateServerPath function.
ftServerExpirationDate
[in] The date of expiration of this item on the server. A value of zero means that the item will never expire.

Note    This must be in the UTC format.

pbDataToUpload
[in] Buffer of data to upload.
cbDataToUpload
[in] Size, in bytes, of the data buffer to upload.
dwUploadFlags
[in] Flags that qualify the upload process. This value must be set to zero.
hWorkEvent
[in] Handle to an event that is signaled when there is work to do on the asynchronous task returned by the phTask parameter. This parameter can be NULL if such synchronization is not required.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle. Use this handle in calls to XOnlineTaskContinue to process the file upload.

Return Values

If XOnlineStorageUploadFileFromMemory succeeds, the return value is S_OK.

If XOnlineStorageUploadFileFromMemory fails, the return value is one of the following error codes:

Value Description
XONLINE_E_STORAGE_INVALID_REQUEST The request is invalid.
XONLINE_E_STORAGE_ACCESS_DENIED The client doesn't have sufficient permissions to upload the file.
XONLINE_E_STORAGE_INVALID_ACCESS_TOKEN The access token signature is invalid.
XONLINE_E_STORAGE_CANNOT_FIND_PATH Name resolution failed.
XONLINE_E_STORAGE_INVALID_STORAGE_PATH Invalid storage path.
XONLINE_E_STORAGE_INVALID_FACILITY Invalid facility code.

Remarks

The primary use of this function is to perform in-memory operations, such as taking input directly from a CODEC (similar to voice mail operations).

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
