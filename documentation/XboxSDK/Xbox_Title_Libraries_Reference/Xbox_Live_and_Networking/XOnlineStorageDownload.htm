XOnlineStorageDownload

Downloads a storage item from the specified storage facility.

HRESULT XOnlineStorageDownload(
    DWORD dwFacility,
    DWORD dwUserIndex,
    LPCWSTR wszStoragePath,
    LPCSTR szInstallDirectory,
    DWORD dwDownloadFlags,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwFacility

[in] Indicates storage facility. May be one of the following:
Value Description
XONLINESTORAGE_FACILITY_STATS Game-clips (stats with attachments) facility.
XONLINESTORAGE_FACILITY_TEAMS Teams storage facility.
XONLINESTORAGE_FACILITY_PER_TITLE Title-managed global storage.
XONLINESTORAGE_FACILITY_PER_USER_TITLE Per title, per user storage.

For a complete list of facility values, see the XONLINESTORAGE_FACILITY page.

dwUserIndex
[in] The controller index (zero-based) of the user for whom to download the attachment.
wszStoragePath

[in] The service's storage path for the storage item. For game clips, this path can be obtained by calling XOnlineStatRead when specifying the XONLINESTORAGE_FACILITY_STATS facility. This path is stored in the lpString member of the statistic attribute (XONLINE_STAT structure) that has an ID equal to XONLINE_STAT_ATTACHMENT_PATH. For all other facilities, you can use XOnlineStorageCreateServerPath to create the storage path for desired storage item.

szInstallDirectory
[in] String indicating the destination to which to save the file. See the Remarks section below for details.
dwDownloadFlags
[in] Flags that qualify the download process. The title should specify zero for this parameter.
hWorkEvent
[in, optional] Handle to an event that is signaled when there is work to do on the asynchronous task returned by the phTask parameter. This parameter can be NULL if no such synchronization is required.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle. Use this handle in calls to XOnlineTaskContinue to process the file download.

Return Values

If XOnlineStorageDownload succeeds, the return value is S_OK.

If XOnlineStorageDownload fails, the return value is one of the following error codes:

Value Description
XONLINE_E_STORAGE_INVALID_REQUEST The request is invalid.
XONLINE_E_STORAGE_INVALID_ACCESS_TOKEN The access token signature is invalid.
XONLINE_E_STORAGE_CANNOT_FIND_PATH Name resolution failed.
XONLINE_E_STORAGE_FILE_IS_ELSEWHERE The client received a redirection request.
XONLINE_E_STORAGE_INVALID_STORAGE_PATH Invalid storage path.
XONLINE_E_STORAGE_INVALID_FACILITY Invalid facility code.
XONLINE_E_STORAGE_UNSUPPORTED_CONTENT_TYPE Returned when the content type of the storage item is not supported by this operation (see remarks).
Other See Online Errors for information about other possible errors.

Remarks

Access to the storage service is automatically granted by default from XOnlineLogon (it is not required that you specifically ask for the storage service when signing into Live.

For game clips, the client API enforces a limit of 20 cached attachments per title across all leaderboards.

The szInstallDirectory must be NULL for game clips. Specifying NULL for other facilities will result in the content being downloaded to a default location. The default install locations for each storage item can be obtained via a call to the XOnlineStorageGetInstallLocation function. If an explicit install path is specified (except for game clips), the content will be installed at the specified location instead.

XOnlineStorageDownload will download all storage items that were previously uploaded using either XOnlineStorageUpload or XOnlineStorageUploadByServerPath. Any attempt to download storage items uploaded with XOnlineStorageUploadFromMemory will result in the XONLINE_E_STORAGE_UNSUPPORTED_CONTENT_TYPE error.
