XOnlineMessageCreate

Creates a new message to send.

HRESULT XOnlineMessageCreate(
    BYTE bMsgType,
    WORD wNumProperties,
    WORD wExpectedValuesSize,
    ULONGLONG qwMessageContext,
    DWORD dwMessageFlags,
    WORD wExpireMinutes,
    XONLINE_MSG_HANDLE *phMsg
);

Parameters

bMsgType
[in] Type of message to send. Valid values for this parameter are shown in the following table.
Message Type Required Context Required Properties Required Flags Description
XONLINE_MSG_TYPE_TITLE_CUSTOM Defined by title. Defined by title. Defined by title. Creates a custom message whose format is defined by the title.
XONLINE_MSG_TYPE_FRIEND_REQUEST 0 N/A N/A Creates an invitation that will be sent to a member of the player's Friends List.
XONLINE_MSG_TYPE_GAME_INVITE Competition or Team ID XONLINE_MSG_PROP_SESSION_ID XONLINE_MSG_FLAG_TEAM_CONTEXT or XONLINE_MSG_FLAG_COMP_CONTEXT Creates an invitation to participate in a game.
XONLINE_MSG_TYPE_TEAM_RECRUIT Team ID N/A XONLINE_MSG_FLAG_TEAM_CONTEXT Creates an invitation to join the player's team.
XONLINE_MSG_TYPE_COMP_REMINDER Competition Event ID XONLINE_MSG_PROP_COMP_NAME, XONLINE_MSG_PROP_COMP_EVENT_START XONLINE_MSG_FLAG_TEAM_CONTEXT Creates a reminder that a player is scheduled to participate in a competition.
XONLINE_MSG_TYPE_COMP_REQUEST Competition ID XONLINE_MSG_PROP_COMP_NAME, XONLINE_MSG_PROP_COMP_START, XONLINE_MSG_PROP_COMP_REG_CLOSE XONLINE_MSG_FLAG_TEAM_CONTEXT Creates a request to participate in a competition.
XONLINE_MSG_TYPE_LIVE_MESSAGE 0 XONLINE_MSG_PROP_SYSTEM_TEXT N/A Creates a message sent by the Xbox Live service or the publisher.
wNumProperties
[in] Number of properties that will be assigned to the message using XOnlineMessageSetProperty.
wExpectedValuesSize
[in] Expected size of the message buffer. Must be less than XONLINE_MAX_MESSAGE_DETAILS. If the expected size is not known in advance, specify a size of 0. In this case, the default message buffer size is used.
qwMessageContext

[in] Information about the message context. Message context is required for most message types. See the Required Context column of the table above for more information.

dwMessageFlags
[in] One or more message flags from the following table.
Message Flag Description
XONLINE_MSG_FLAG_REQUIRED The receiver is required to read this message.
XONLINE_MSG_FLAG_RECOMMENDED It is recommended that the receiver read this message.
XONLINE_MSG_FLAG_HAS_VOICE The message contains a voice attachment.
XONLINE_MSG_FLAG_HAS_TEXT The message contains text.
XONLINE_MSG_FLAG_READ The receiver read this message.
XONLINE_MSG_FLAG_NON_EXPORTABLE This message should be displayed only on Xbox consoles. It should not be visible on Windows-based versions of the game.
XONLINE_MSG_FLAG_NOTIFY This message should cause a new message icon to be displayed to the user.
XONLINE_MSG_FLAG_TEAM_CONTEXT This message's context refers to a team ID.
XONLINE_MSG_FLAG_COMP_CONTEXT This message's context refers to a competition event ID.
XONLINE_MSG_FLAGS_TITLE_RESERVED This is a mask for title-specific flags associated with the message.
wExpireMinutes
[in] Number of minutes that the message is valid. After this time limit passes, the message expires. For message types other than XONLINE_MSG_TYPE_TITLE_CUSTOM, titles should set this parameter to 0 to use an appropriate default expiration.
phMsg
[out] Pointer to an XONLINE_MSG_HANDLE structure. Upon return, this structure contains the message created by this function.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

You can use the constant XONLINE_MSG_FLAGS_TITLE_RESERVED to set or clear the most significant byte of a DWORD containing a flag. You can then store your own custom value there. Title-specific flags should not be used for message types other than XONLINE_MSG_TYPE_TITLE_CUSTOM.

It is important to note that the XONLINE_MSG_HANDLE received through the phMsg parameter can be used in only one send operation at a time. Also, memory for the handle should not be freed until the send operation has completed.

The message should be passed to XOnlineMessageSetProperty to add the desired properties. Next, it should be passed to the sending function appropriate to the message type, such as XOnlineGameInviteSend for XONLINE_MSG_TYPE_GAME_INVITE or XOnlineMessageSend for XONLINE_MSG_TYPE_TITLE_CUSTOM.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
