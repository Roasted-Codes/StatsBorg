System Pool Allocations

Here are the network function calls that may result in data being allocated from the system pool:

FunctionSystem pool memory allocation estimate
XNetStartup Varies depending on what was set as the pool size in the call. In addition to the pool memory, a nominal amount of additional memory is required to initialize various subsystems, such as DHCP. This is freed when XNetCleanup is called.
send, sendto, WSASend, WSASendTo The number of bytes in the buffer provided to the call. This is necessary so that the call can return even if it can't immediately put the data on the wire. The maximum amount of memory allocated to send calls on a socket is controlled via the cfgSockDefaultSendBufsizeInK member of the XNetStartupParams structure passed to XNetStartup. This is freed when the data in the buffer has been transmitted (in the case of TCP, this means that it was also successfully acknowledged by the receiver).
socket SOCK_DGRAM sockets are roughly 100 bytes. SOCK_STREAM sockets are a little over 200 bytes. This is freed when closesocket is called on the socket.
