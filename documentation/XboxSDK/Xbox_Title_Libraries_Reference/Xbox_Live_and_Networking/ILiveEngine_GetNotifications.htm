ILiveEngine::GetNotifications

Determines pending notifications for a specified user.

HRESULT GetNotifications(
    DWORD Port,
    UIX_NOTIFICATION_PURPOSE Purpose,
    DWORD *pNotifications
);

Parameters

Port
[in] The controller port index (zero-based) of the user.
Purpose
[in] UIX_NOTIFICATION_PURPOSE value specifying the reason that notifications are being queried.
pNotifications
[out] Pointer to a DWORD variable that receives a set of flags indicating current notifications.
Value Description
UIX_NOTIFY_FRIEND_REQUEST A friends request has been received.
UIX_NOTIFY_GAME_INVITE A game invite has been received.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

GetNotifications returns the pending notifications for the specified user. Titles use this information to determine the notification icons that must be displayed either 1) next to the Friends item in the main start menu, or 2) in the lower right corner of the screen for three seconds during game play.

UIX_NOTIFICATION_MENU notifications are cleared when the title calls StartFeature for the Friends feature and the user views the friends list.

UIX_NOTIFICATION_IN_GAME_FLASH notifications are cleared when the title retrieves them with GetNotifications. In-game notifications are not returned if the engine is handling them (which it does by default).
