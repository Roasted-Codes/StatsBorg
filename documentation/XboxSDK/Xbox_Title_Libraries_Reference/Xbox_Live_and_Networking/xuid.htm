XUID

Contains the user identification and user-specific settings.

typedef struct{
    ULONGLONG qwUserID;
    DWORD dwUserFlags;
} XUID;

Members

qwUserID
Value that contains the user identification.
dwUserFlags

Value that contains settings associated with this user. Can be a combination of the following flags:
Value Description
XONLINE_USER_VOICE_NOT_ALLOWED This user is not allowed to use the voice chat feature.
XONLINE_USER_PURCHASE_NOT_ALLOWED This user is not allowed to make any purchases.
XONLINE_USER_NICKNAME_NOT_ALLOWED This user is not allowed to choose custom nicknames.

Additionally, the following masks indicate the bit offsets of additional data stored in the dwUserFlags member:

Value Description
XONLINE_USER_GUEST_MASK Mask for the bits that contain the user guest number. If this user is not a guest, the value is zero. If this user is a guest, the sponsoring user is specified in qwUserID. Up to three guest users (one for each additional controller) can be associated with a single logged on user.
XONLINE_USER_COUNTRY_MASK Mask for the bits that contain the user country identifier, representing the country in which the user is registered. This value will be one of the XONLINE_COUNTRY_ values in xonline.h.

Remarks

For titles using Arbitration, the service aggregates information about playing patterns. Specifically, Arbitration service keeps a tally of the number of Arbitrated sessions a user has engaged in, as well as the number of times a user disconnected from an arbitrated session prematurely. All users are divided into eight buckets based on the ratio of disconnects to the total number of sessions played.

Likewise, the service keeps track of the total number of tournament matches each user has registered for as well as the number of matches each user failed to show for. All users are divided into eight buckets based on the ratio of no-shows to the total number of tournament matches played.

Titles can use this information for player profiles, matchmaking, unlocking content, trophies, and so on. Because the Disconnect and No-Show ratings are updated every time a user participates in a tournament or an arbitrated game session, they must not be persisted beyond the duration of a user session.

The NoShowRating and Disconnect Rating are returned by XOnlineLogon in the dwUserFlags field of the XUID structure.

Use the XOnlineUserDisconnectRating(dwUserFlags) macro to access the DisconnectRating and the XOnlineUserNoShowRating(dwUserFlags) macro to access the NoShowRating.

Use XOnlineAreUsersIdentical, XOnlineIsUserGuest, XOnlineIsUserNicknameAllowed, XOnlineIsUserPurchaseAllowed, XOnlineIsUserVoiceAllowed, XOnlineUserGuestNumber, and XOnlineUserCountryId to extract user information from an XUID structure.
