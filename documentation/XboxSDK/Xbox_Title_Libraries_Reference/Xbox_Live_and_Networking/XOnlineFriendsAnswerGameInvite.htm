XOnlineFriendsAnswerGameInvite

Responds to a game invitation received from another friend.

Note    It is recommended that titles support voice message attachments to game invitations responses by using the XOnlineGameInviteAnswer method.

HRESULT XOnlineFriendsAnswerGameInvite(
    DWORD dwUserIndex,
    const XONLINE_FRIEND *pToFriend,
    XONLINE_GAMEINVITE_ANSWER_TYPE Answer
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the user answering the game invitation. The user must have been successfully logged on with a previous call to XOnlineLogon.
pToFriend
[in] Pointer to an XONLINE_FRIEND structure specifying the friend to whom the response is being made.
Answer

[in] An XONLINE_GAMEINVITE_ANSWER_TYPE value that specifies the answer to the invitation.

Return Values

If XOnlineFriendsAnswerGameInvite succeeds, it returns S_OK.

If XOnlineFriendsAnswerGameInvite fails, it returns one of the following error codes.

Value Description
XONLINE_E_NOTIFICATION_NO_TASK The Friends startup task is not active. You must first call XOnlineFriendsStartup to create the startup task.
XONLINE_E_NOTIFICATION_USER_NOT_FOUND The user specified by the pToFriend parameter is not on the current user's Friends List. This error should occur only if the title is currently enumerating the Friends List. The current user should not be accepting game invites from users that are not on their Friends List.
Other See Online Errors for information about other possible errors.

Remarks

The task returned by the XOnlineFriendsEnumerate function must be active at the time the title calls the XOnlineFriendsAnswerGameInvite function.

Pump the task returned by XOnlineFriendsStartup in order to send the answer.

XOnlineFriendsAnswerGameInvite writes invitation information to the hard disk if the title ID of the title that calls the function is different from the title ID of the title the user is being invited to play. The new title reads the invitation information from the hard disk by calling XOnlineFriendsGetAcceptedGameInvite.

Answering yes to an incoming game invitation implicitly cancels any outgoing invitations. Functionally, it is the same as first calling XOnlineFriendsRevokeGameInvite with an empty list.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
