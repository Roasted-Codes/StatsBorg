XNetUnregisterInAddr

Frees the security association slot assigned to the secure address created by XNetXnAddrToInAddr.

INT XNetUnregisterInAddr(
    const IN_ADDR ina
);

Parameters

ina
IN_ADDR structure that contains the secure address to unregister.

Return Values

Returns zero if successful; an error code otherwise.

Remarks

Security associations are created for each unique XNADDR/XNKID pair passed to XNetXnAddrToInAddr. Security associations are also implicitly created for each secure host that establishes an incoming connection with this host on a given registered XNKID. There will be only one security association between a pair of hosts on a given XNKID, no matter how many sockets are actively communicating on that secure connection.

The cfgSecRegMax member of XNetStartupParams determines the maximum number of XNADDR/XNKID security associations that can be registered at one time (the default is 32 key pairs). Security associations should be unregistered when a specific XNADDR will no longer be used.

Requirements

Header: Declared in Winsockx.h.

Import Library: Use Xnet.lib for titles that support only system link play. Use XOnline.lib for titles that support Xbox Live play. Use the secure versions of the library file (Xnets.lib or XOnlines.lib) for final testing.
