XOnlineGetLogonUsers

Retrieves the status of each user participating in a logon session, including the success or failure of user logons performed by XOnlineLogon.

PXONLINE_USER XOnlineGetLogonUsers();

Parameters

None.

Return Values

If the function fails, the return value is NULL.

If the function succeeds, the return value is an array of XONLINE_USER structures that represent the status of each of the currently logged on users.

Remarks

Use the XOnlineGetLogonUsers function to determine the logon status of users specified in a call to XOnlineLogon. Call XOnlineGetLogonUsers only after the logon task has returned XONLINE_S_LOGON_CONNECTION_ESTABLISHED.

The hr member of each XONLINE_USER structure indicates the success or failure of that individual user's logon. The hr member can be one of the following values:

  • S_OK: The user was successfully logged on, and has no messages.
  • XONLINE_S_LOGON_USER_HAS_MESSAGE: The user was successfully logged on, and has a dynamic message available for viewing. The user should be given the option of rebooting into the accounts management section of the Xbox Dashboard in order to view the message.
  • XONLINE_E_LOGON_USER_ACCOUNT_REQUIRES_MANAGEMENT: The user cannot be successfully logged on. The title will need to reboot the user into the accounts management section of the Xbox Dashboard, where the user can determine the reason for this failure and resolve the issue so they may participate in online game play. Note that the failure may simply be caused by a required message that must be read.

Note    When rebooting into the accounts management section of the Xbox Dashboard, the title should not close the logon task handle (by calling XOnlineTaskClose) before calling XLaunchNewImage to launch the Dashboard. The title should have a valid, active Xbox Live session when rebooting into the dashboard, so that the Dashboard can use the logon state to determine user messages or other problems.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
