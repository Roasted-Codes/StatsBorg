XOnlineMutelistStartup

Creates an asynchronous task that processes additions to, as well as removals from, the mutelist. 

HRESULT XOnlineMutelistStartup(
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

hWorkEvent
[in, optional] Handle to an event object, created by the title. This event is signaled when there is work to do on the asynchronous task that adds and removes users to/from the mutelist. Perform work on the task by calling XOnlineTaskContinue with the handle returned in the phTask parameter. This parameter is optional and may be NULL. If this parameter is NULL, the title should call XOnlineTaskContinue periodically to perform work on the task.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives the mutelist task. This task must be pumped by XOnlineTaskContinue in order to process mutelist additions and removals. When mutelist functionality is no longer needed, call XOnlineTaskClose to close this task.

Return Values

If XOnlineMutelistStartup succeeds, the return value is S_OK.

If XOnlineMutelistStartup fails, the return value is one of the following error codes.

Value Description
E_OUTOFMEMORY Insufficient memory. Free some memory.
Other See Online Errors for information about other possible errors.

Remarks

The title must successfully call XOnlineMutelistStartup once before making any calls to XOnlineMutelistAdd or XOnlineMutelistRemove. The XOnlineMutelistStartup function creates an asynchronous task that must be pumped in order to add or remove users from the mutelist.

The task created by this function never completes on its own, and will occasionally have work which must be done in order to process mutelist tasks. The title should continue to perform work on this task by calling XOnlineTaskContinue (typically once per frame), until either the title no longer needs any mutelist functionality or XOnlineTaskContinue returns an error code indicating a task error.

When mutelist functionality is no longer needed, pump the task handle until XOnlineTaskContinue returns XONLINETASK_S_RUNNING_IDLE, then close the asynchronous task by calling XOnlineTaskClose.

Players in the mutelist represent users who have been permanently voice muted. The mutelist is maintained by the Xbox Live service and persists across games and titles. The title must check the mutelist before sending or receiving voice to another player.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
