Adding Xbox Drop-in UI Support

Adding Xbox Drop-in UI support to a title requires initializing the engine, calling engine functions in the title's render loop, and starting features as needed. These steps are described below. Complete example code demonstrating these steps is provided in Using Xbox Drop-in UI.

Initialization

Initializing the Xbox Drop-in UI is a simple process of creating the Xbox Drop-in UI engine and the font renderer and default plug-ins, registering the plug-ins, and enabling features the title will use.

UIXCreateLiveEngine creates the Xbox Drop-in UI engine. The skin file and language to use are specified when the engine is created. The returned ILiveEngine interface pointer can then be used to make calls to the engine. Create a Direct3D device (with IDirect3D8::CreateDevice) before calling UIXCreateLiveEngine; otherwise, an assert will fire.

The plug-ins that the engine will use must be registered with the engine. These plug-ins can be the default plug-ins obtained with UIXCreateAudioPlugin and UIXCreateUIPlugin, or they can be custom plug-ins implemented by the title. ILiveEngine::SetAudioPlugin and ILiveEngine::SetUIPlugin register the audio and UI plug-ins, respectively. The font renderer plug-in ITitleFontRenderer, which must be implemented by the title, is passed as an argument to UIXCreateUIPlugin when the UI plug-in is created.

The default audio plug-in requires XACT engine and XACT sound bank pointers, and uses XACT to provide sound for the user interface. The XACT engine is created by calling XACTEngineCreate. An XACT project included with the Xbox SDK (in source\uix\UIXDefault.xap) can be used to create default sound and wave banks. Choose Build from the File menu after opening the project. Alternatively, pre-built sound and wave banks are available in redist\uix\media. The wave bank is registered by calling IXACTEngine::RegisterWaveBank, and the sound bank is created by calling IXACTEngine::CreateSoundBank. The resulting pXactEngine and pXactSoundBank pointers are passed in the UIXCreateAudioPlugin call to create the default audio plug-in. Titles can modify the default sound and wave banks, use their own banks, or provide a custom implementation of the audio plug-in to provide audio support through some other means.

Any features that will be used by the title must be enabled with ILiveEngine::EnableFeature prior to use. EnableFeature links the appropriate code in with the title but doesn't allocate any memory, so it is typically called at initialization time.

After initialization, Drop-in UI features can be started as needed.

Render Loop

The title's render loop needs to make calls to ILiveEngine::SetInput and ILiveEngine::DoWork. ILiveEngine::SetInput forwards controller input (in the form of an XINPUT_STATE structure) to the Xbox Drop-in UI engine so that the UI can reflect user input. ILiveEngine::DoWork gives the engine an opportunity to perform required processing (such as calling Xbox Live functions and pumping asynchronous online task handles).

ILiveEngine::DoWork can return flags indicating that additional calls need to be made into the engine in order to perform subsequent tasks. The UIX_DOWORK_NEED_TO_RENDER flag indicates that render work must be done by the engine, and the title should call ILiveEngine::Render to render the UI to a Direct3D surface. The UIX_DOWORK_NEED_TO_REBOOT flag indicates that the engine needs to reboot, so the title should call ILiveEngine::Reboot. A reboot is typically required when the user needs to access the Xbox Dashboard (for account management, as an example) or when an auto-update must be performed. The UIX_DOWORK_FEATURE_EXIT flag indicates that an active feature has exited, and the title should call ILiveEngine::GetExitInfo to determine which feature exited, and why. The title can use the exit information to determine the next course of action. For example, after a successful logon by the logon feature, the title can continue with online game play.

ILiveEngine::DoWork also sets the UIX_DOWORK_FEATURE_EXIT when the network connection is lost or fails, even if the logon feature is not currently active and did not exit. In this case, the exit code returned by ILiveEngine::GetExitInfo will be UIX_EXIT_LOGON_FAILED.

If a title has requested that the Xbox Drop-in UI not automatically handle notifications (by setting the UIX_PROPERTY_DISPLAY_NOTIFICATIONS property to FALSE), the UIX_DOWORK_NOTIFICATIONS flag may be set. This flag indicates that the title must call ILiveEngine::GetNotifications to retrieve in-game user notifications and then must display those notifications appropriately. Normally, the engine handles detection and display of notifications automatically.

ILiveEngine::DoWork sets the UIX_DOWORK_PROCESSING_INPUT flag to indicate that the engine is processing player input. If this flag is set, controller input is going to an active feature and should not be processed by the title. Otherwise, the title should process controller input normally.

The title should perform its normal game rendering, and then perform Drop-in UI rendering (if indicated) to the same surface, and then display the resulting surface by updating the swap chain (using IDirect3DDevice8::Present or IDirect3DDevice8::Swap).

Starting Features

To start a feature, call ILiveEngine::StartFeature. ILiveEngine::StartFeature requires a startup parameters structure that varies depending on the feature being activated.

Starting the logon feature displays the logon screen and begins the logon process. The UIX_LOGON_PARAMS structure specifies the type of logon (normal, silent, using a saved logon state, or in response to an invite) to perform, the maximum number of users to logon (1–4), the Xbox Live services that need to be accessed, and additional optional values for some logon types.

Titles using Xbox High-Level Voice (XHV) should initialize the voice engine (with XHVEngineCreate) only after the logon feature reports a successful logon.

Starting the friends feature displays the user's friends list. The UIX_FRIENDS_PARAMS structure specifies the user (by port number) whose friends list is displayed, and the friend to initially select in the list (if any). Before starting the friends feature, a successful logon must have been performed.

When a feature exits (whether successfully or unsuccessfully), or if the current network connection fails, ILiveEngine::DoWork sets the UIX_DOWORK_FEATURE_EXIT flag. Calling ILiveEngine::GetExitInfo will indicate which feature exited and the exit status and return code of the feature.

After a successful logon (UIX_EXIT_LOGON_SUCCESSFUL), the title must call XOnlineGetServiceInfo for each service requested at logon (in the LogonServiceIDs member of UIX_LOGON_PARAMS) and check the returned HRESULTs to verify which services were successfully connected to. If a service critical to the title failed, the title should log off the Xbox Live service.

For example code demonstrating starting features, see Using Xbox Drop-in UI.
