Private Pool Allocations

Here are the asynchronous network events that result in data being allocated from the Xbox network subsystem's private pool:

Asynchronous eventPrivate pool memory allocation estimate
Data copied from Ethernet DMA buffers to socket receive buffer in network subsystem pool memory. The number of bytes of data present in the packet. This is limited to a maximum of 1500 bytes of data per received Ethernet frame but will typically be much smaller. This is freed when the game calls recv, recvfrom, WSARecv, or WSARecvFrom to copy the packet into a game allocated buffer.
Incoming TCP connection accepted. Child socket created asynchronously. Unlike when a socket is created via the socket call, sockets created from incoming connections on listening sockets can happen at any time. So the Xbox network subsystem allocates these sockets from the private Xbox network subsystem pool instead of the system pool. A little over 200 bytes are allocated. This is freed when closesocket is called on the socket.
ARP packets sent/received. A nominal amount of memory is used to handle the sending and receiving of ARP packets. The memory is freed after the incoming packet has been processed or after the outgoing packet is on the wire.
DHCP packets sent/received. A nominal amount of memory is used to handle the sending and receiving of DHCP packets. The memory is freed after the incoming packet has been processed or after the outgoing packet is on the wire. These allocations are typically only encountered around XNetStartup time.
ICMP packets sent/received. A nominal amount of memory is used to handle the sending and receiving of ICMP packets. The memory is freed after the incoming packet has been processed or after the outgoing packet is on the wire.
TCP SYN-ACK, ACK, or RST sent in response to a network event. Certain incoming TCP events require immediate action by the Xbox network subsystem. A nominal amount of memory is used to handle the sending of packets that the network subsystem generates to deal with these events. The memory is freed after the outgoing packet is on the wire.
Routing table entry added (for example, when a DHCP address changes). The Xbox network subsystem's routing table is allocated from the Xbox network subsystem private pool memory.
