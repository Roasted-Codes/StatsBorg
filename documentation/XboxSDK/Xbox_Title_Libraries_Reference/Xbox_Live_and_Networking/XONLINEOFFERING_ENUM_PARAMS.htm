XONLINEOFFERING_ENUM_PARAMS

Specifies parameters and options for an offering enumeration initiated with XOnlineOfferingEnumerate.

typedef struct{
    DWORD dwOfferingType;
    DWORD dwBitFilter;
    DWORD dwDescriptionIndex;
    WORD wStartingIndex;
    WORD wMaxResults;
} XONLINEOFFERING_ENUM_PARAMS, *PXONLINEOFFERING_ENUM_PARAMS;

Members

dwOfferingType

General type of offerings that should be enumerated. This parameter can be a combination of the following values.
Value Description
XONLINE_OFFERING_SUBSCRIPTION Subscription services.
XONLINE_OFFERING_CONTENT Content offerings.

dwBitFilter
Return offerings that have one or more of these title-specific bits set.
dwDescriptionIndex
Title-specific value used as a selector between alternate descriptions of a single offering. It is intended primarily to distinguish between supported languages, but the specific use of this value is up to the title.
wStartingIndex
Starting index at which to begin offering enumeration.
wMaxResults
Maximum number of results to return per enumeration request. If more records are available, only the first wMaxResults will be returned with each enumeration request. Setting this member to zero indicates that as many records as possible (that will fit into the allocated results buffer) should be returned.

Remarks

The dwBitFilter member is used by the Xbox Live service to determine specific content to enumerate. dwBitFilter is bitwise AND'ed with the dwBitFlags specified in the content package (see Offering Packages); if the result is zero, the package is not returned in the enumeration. Thus, 0x00000000 should never be used for dwBitFlags in the content package or dwBitFilter in the enumeration parameters. The intended use of dwBitFilter is for publishers to define a bit scheme that separates types of content for enumeration (for example, new cars and new levels, so that an enumeration can enumerate only cars, only levels, or both).
