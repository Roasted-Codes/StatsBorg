Using Xbox Drop-in UI

Titles can easily add Xbox Drop-in UI support because it is simple to initialize, uses reasonable defaults, and smoothly integrates with the title's existing rendering code and render loop. Customization options allow changes to the default behavior and media as required by the title.

Adding Xbox Drop-in UI support to a title involves three steps:

  • Initialize the Xbox Drop-in UI engine.
  • Call engine functions in the title's render loop.
  • Start features as needed.

Initialization

Initializing the Xbox Drop-in UI is a simple process of creating the Xbox Drop-in UI engine, the font renderer and default plug-ins, registering the plug-ins, and enabling features the title will use. The following example code shows the steps involved. Note that example D3D Initialization, XACT Initialization, and CTitleFontRenderer code is given later in this topic.

#include <xtl.h>
#include <xonline.h>
#include <xact.h>
#include <uix.h>

// Xbox Drop-in UI interface pointers.
CTitleFontRenderer ctfr;
ITitleAudioPlugin *pAudioPlugin;
ITitleUIPlugin *pUIPlugin;
ILiveEngine *pLiveEngine;

// Simple title initialization
XOnlineStartup( NULL );
XInitDevices( 0, NULL );

// Create the Xbox Drop-in UI engine, the font plug-in, and the default audio and UI plug-ins.
ctfr.OpenFont( L"d:\\media\\arialuni.ttf" );
UIXCreateAudioPlugin( pXactEngine, pXactSoundBank, &pAudioPlugin );
UIXCreateUIPlugin( &ctfr, &pUIPlugin );
UIXCreateLiveEngine( "d:\\media\\default.uix", XC_LANGUAGE_ENGLISH, &pLiveEngine );

// Register the plug-ins and enable features the title will use.
pLiveEngine->SetUIPlugin( pUIPlugin );
pLiveEngine->SetAudioPlugin( pAudioPlugin );
pLiveEngine->EnableFeature( UIX_LOGON_FEATURE );
pLiveEngine->EnableFeature( UIX_FRIENDS_FEATURE );

This example uses the default font and skin files that are included with the Xbox SDK (redist\stats\media\ArialUni.Ttf and source\uix\default.inx). Use SkinBld to build the default.uix skin file from default.inx, the text skin description file.

After the above code executes, the engine is initialized and features can be started as needed.

Note    In this example, and the code examples below, error checking has been removed to streamline flow. Most of the functions return an HRESULT value that should be checked with the SUCCEEDED (or similar) macro.

Render Loop

A title's render loop likely already checks for input device changes, retrieves controller input, renders to the current render surface, and presents that surface. Two additional steps need to be added to the render loop:

  1. Call ILiveEngine::SetInput to forward controller input to the Xbox Drop-in UI engine.
  2. Call ILiveEngine::DoWork so the engine can perform necessary processing. The flags returned by ILiveEngine::DoWork indicate whether further engine calls need to be made to render, reboot, or retrieve exit information for a feature that has exited, and also whether the engine is processing input or the title should do so.

An example render loop, with the additional Xbox Drop-in UI calls, is shown here:

DWORD dwFlags, i;

// ... existing title code that renders to g_pSurface
// ... existing title code that detects input device changes

// Forward controller input
XINPUT_STATE InputState;
for( i = 0; i < XGetPortCount(); i++ ) {
    if( g_hInputDevice[i] ) {
        XInputGetState( g_hInputDevice[i], &InputState );
        pLiveEngine->SetInput( i, &InputState );
    } else {
        pLiveEngine->SetInput( i, NULL );
    }
}

// Call DoWork (and additional engine functions as indicated).
pLiveEngine->DoWork( &dwFlags );
if( dwFlags & UIX_DOWORK_NEED_TO_RENDER ) {
    pLiveEngine->Render( g_pSurface );
}
if( dwFlags & UIX_DOWORK_NEED_TO_REBOOT ) {
    pLiveEngine->Reboot();
}
if( dwFlags & UIX_DOWORK_FEATURE_EXIT ) {
    UIX_EXIT_INFO ExitInfo;
    pLiveEngine->GetExitInfo( &ExitInfo );
}

if( !(dwFlags & UIX_DOWORK_PROCESSING_INPUT) ) {
    // Process controller input normally.
}

// Present (or Swap)
g_pDevice->Present( NULL, NULL, NULL, NULL );

For code demonstrating the detection and handling of input device changes (and device handles), see Using Xbox Peripherals.

Starting Features

Features are partitioned into discrete functional sets so that titles can choose to use the Xbox Drop-in UI for some features while implementing other functionality directly. Features are started with ILiveEngine::StartFeature, which takes a startup parameters structure that is dependent on the feature being activated.

Logon Feature

Starting the logon feature displays the logon screen and begins the logon process. The UIX_LOGON_PARAMS structure specifies the type of logon (normal, silent, using a saved logon state, or in response to an invite) to perform, the maximum number of users to logon (1–4), the Xbox Live services that need to be accessed, and additional optional values for some logon types.

UIX_LOGON_PARAMS ulp;
ZeroMemory( &ulp, sizeof(ulp) );
ulp.StructSize = sizeof(ulp);
ulp.LogonType = UIX_LOGON_TYPE_NORMAL;
ulp.LogonUserCount = 4;
ulp.LogonServiceIDs[0] = XONLINE_BILLING_OFFERING_SERVICE;
ulp.LogonServiceIDs[1] = XONLINE_SIGNATURE_SERVICE;
ulp.LogonServiceIDs[2] = XONLINE_MATCHMAKING_SERVICE;
pLiveEngine->StartFeature( UIX_LOGON_FEATURE, &ulp );

Friends Feature

Starting the friends feature displays the user's friends list. The UIX_FRIENDS_PARAMS structure specifies the user (by port number) whose friends list is displayed, and the friend to initially select in the list (if any). The friends feature requires that XONLINE_MATCHMAKING_SERVICE was specified when the logon feature was started.

UIX_FRIENDS_PARAMS ufp;
ZeroMemory( &ufp, sizeof(ufp) );
ufp.StructSize = sizeof(ufp);
ufp.UserPort = 0;
ufp.SelectedFriendXUID.qwUserID = 0;
pLiveEngine->StartFeature( UIX_FRIENDS_FEATURE, &ufp );

Over time, additional features will be added to handle more of the areas involved in implementing Xbox Live support.

CTitleFontRenderer

When creating the default UI plugin, UIXCreateUIPlugin requires an ITitleFontRenderer interface that is used to render text. Here is a simple example implementation of ITitleFontRenderer using XFONT. The font renderer plug-in need not be implemented with XFONT; it can be implemented using the title's existing font support.

class CTitleFontRenderer : public ITitleFontRenderer
{
public:
    CTitleFontRenderer() { m_pXFont = NULL; };
    ~CTitleFontRenderer() { if( m_pXFont ) { m_pXFont->Release(); } };
    STDMETHODIMP_(ULONG)Release() { delete this; return 0; };
    STDMETHODIMP OpenFont( LPCWSTR pszFont ) {
        return XFONT_OpenTrueTypeFont( pszFont, 0x10000, &m_pXFont );
    };
    STDMETHODIMP DrawText( LPCWSTR pText, DWORD X, DWORD Y, DWORD MaxWidth ) {
        if( MaxWidth > 0 ) {
            D3DRECT r = { X, Y, X + MaxWidth, Y + 200 };
            m_pXFont->SetClippingRectangle( &r );
        }
        return m_pXFont->TextOut( g_pSurface, pText, -1, X, Y );
    };
    STDMETHODIMP GetTextSize( LPCWSTR pText, DWORD *pWidth, DWORD *pHeight ) {
        *pHeight = m_pXFont->GetTextHeight();
        return m_pXFont->GetTextExtent( pText, -1, (UINT *)pWidth );
    };
    STDMETHODIMP SetColor( D3DCOLOR Color ) {
        m_pXFont->SetTextColor( Color );
        return S_OK;
    };
    STDMETHODIMP SetHeight( DWORD Height ) {
        return m_pXFont->SetTextHeight( Height );
    };

private:
    XFONT *m_pXFont;
};

XACT Initialization

The example code in Initialization above did not contain XACT initialization code, shown here. Initializing XACT requires creating the XACT engine with XACTEngineCreate, registering a wave bank with IXACTEngine::RegisterWaveBank, and creating the sound bank with IXACTEngine::CreateSoundBank. The resulting pXactEngine and pXactSoundBank pointers are used in the UIXCreateAudioPlugin call (above) to create the default audio plug-in. This example uses the default XACT sound and wave banks that are included with the Xbox SDK (in redist\uix\media).

PXACTWAVEBANK pXactWaveBank;
PXACTSOUNDBANK pXactSoundBank;
PXACTENGINE pXactEngine;

XACT_RUNTIME_PARAMETERS Params;
PVOID SoundBankData;
PVOID WaveBankData;
DWORD SoundBankSize;
DWORD WaveBankSize;

LoadFile( "d:\\media\\UIXDefault.xwb", &WaveBankData, &WaveBankSize );
LoadFile( "d:\\media\\UIXDefault.xsb", &SoundBankData, &SoundBankSize);

ZeroMemory( &Params, sizeof(Params) );
Params.dwMaxConcurrentStreams = 16;
Params.dwMax2DHwVoices = 40;
Params.dwMax3DHwVoices = 10;
XACTEngineCreate( &Params, &pXactEngine );

pXactEngine->RegisterWaveBank( WaveBankData, WaveBankSize, &pXactWaveBank );
pXactEngine->CreateSoundBank( SoundBankData, SoundBankSize, &pXactSoundBank );

The LoadFile() support function simply loads a file into memory:

BOOL LoadFile( LPSTR pszFile, PVOID *pvData, DWORD *pdwSize )
{
    BOOL fRes = FALSE;
    DWORD dwRead;
    HANDLE hFile = CreateFile( pszFile, GENERIC_READ, 0, NULL, OPEN_EXISTING, 0, NULL );
    if( INVALID_HANDLE_VALUE != hFile ) {
        *pdwSize = GetFileSize( hFile, NULL );	
        if( *pvData = HeapAlloc( GetProcessHeap(), 0, *pdwSize ) ) {
            fRes = ReadFile( hFile, *pvData, *pdwSize, &dwRead, NULL );		
        }	
        CloseHandle( hFile );
    }
    return fRes;
}

D3D Initialization

The example code given in Initialization above did not contain D3D initialization code. A simple example is shown here:

IDirect3DSurface8 *g_pSurface;
IDirect3DDevice8 *g_pDevice;
IDirect3D8 *pD3D;

pD3D = Direct3DCreate8( D3D_SDK_VERSION );

// Set up the presentation parameters.
D3DPRESENT_PARAMETERS d3dPP;
ZeroMemory( &d3dPP, sizeof( D3DPRESENT_PARAMETERS ) );
d3dPP.BackBufferWidth                 = 640;
d3dPP.BackBufferHeight                = 480;
d3dPP.BackBufferFormat                = D3DFMT_A8R8G8B8;
d3dPP.BackBufferCount                 = 1;
d3dPP.SwapEffect                      = D3DSWAPEFFECT_DISCARD;
d3dPP.FullScreen_RefreshRateInHz      = 60;
d3dPP.FullScreen_PresentationInterval = D3DPRESENT_INTERVAL_ONE;
d3dPP.EnableAutoDepthStencil          = TRUE;
d3dPP.AutoDepthStencilFormat          = D3DFMT_D24S8;

pD3D->CreateDevice( D3DADAPTER_DEFAULT, 
                    D3DDEVTYPE_HAL,
                    0,
                    D3DCREATE_HARDWARE_VERTEXPROCESSING,
                    &d3dPP,
                    &g_pDevice );

g_pDevice->GetRenderTarget( &g_pSurface );
