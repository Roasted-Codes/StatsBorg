Customizing Xbox Drop-in UI

The easiest method for customizing UI look and feel is through the use of a skin file. A skin file contains strings, images, and layout information about each Xbox Drop-in UI screen. Content developers can customize the skin file by changing images, layout information, icons, and strings for any language setting. To change the skin file, the content developer provides art assets and edits the values in the text skin description file (.inx). The SkinBld tool creates a skin file (.uix) from the text description file. The resulting .uix file is included with the title's game disc. The path to the skin file is specified in UIXCreateLiveEngine when the title launches the Xbox Drop-in UI engine. The skin file can be modified and tested without rebuilding the title executable. See Skinning Xbox Drop-in UI.

The default ITitleAudioPlugin uses an XACT sound bank to play sounds in response to UI events. The sounds in the bank can be customized by the content developer, and the associated UI event/sound mappings can be changed in the skin file.

The default font used with Xbox Drop-in UI is a full Unicode font. However, the font file is supplied by the title on the hard disk, so the title can choose to use a smaller or customized character set. Additionally, by implementing ITitleFontRenderer, the title has complete control over fonts and rendering.

For more extensive UI customization, the title can provide its own implementation of the ITitleUIPlugin interface. Source code with a sample ITitleUIPlugin implementation is provided in the Xbox SDK (in Samples\Xbox\Networking\UIXPlugin).

Because the engine and the default plug-ins use XMemAlloc (and related functions) for memory management, titles can override Drop-in UI memory management by providing custom memory allocation routines. See XMemAlloc for more information.

Titles still have the option of not using Xbox Drop-in UI at all, providing a complete, custom Xbox Live implementation using the Xbox Live Library.

The following table compares the various components that are available for customization, the difficulty of customizing that component, the approximate risk level of violating TCRs when customizing that component, the cost of the default implementation provided by Xbox Drop-in UI, and other notes.

Component Difficulty TCR Risks Default Implementation Cost and Notes
Skinning (SkinBld tool) Very Easy Very Low The default implementation uses reasonably small textures. Titles can save memory by using even smaller textures or by using greater compression (but doing so may degrade appearance).
Replacing Strings (SkinBld tool) Very Easy Low TCR risk is low, but make sure not to use banned words (such as "reboot").
Audio (ITitleAudioPlugin) Easy Low Uses XACT. If a title does not already use XACT, adding support requires about 48 KB of code and some run-time overhead.
Font File Very Easy Low ArialUni.TTF is 22 MB. A smaller font file can be used (but if support for a given character set is removed, be sure to remove strings using those characters from the skin file so that a default, supported language will be used).
Font Renderer (ITitleFontRenderer) Easy Low Uses XFont. A title can provide its own font renderer that uses pre-rasterized glyphs or some other implementation.
UI (ITitleUIPlugin) Medium to Difficult Medium The default implementation has minimal memory impact, but does not support animations or 3D rendering.
UI Flow, Adding or Removing Buttons Medium to Difficult High Xbox Drop-in UI does not currently provide options for customizing the general UI flow or adding or removing screen elements. Support may be added in the future, but these type of customizations represent the greatest risk in terms of TCR compliance.

The default Xbox Drop-in UI implementation currently uses about 2 MB of memory, and, at maximum, slightly less than 4 MB (for a large friends list). Title customizations may increase or decrease this amount.

The Drop-in UI Customization reference contains information about the interfaces, structures, and enumerations used for plug-in customization.

Skinning Xbox Drop-in UI describes the skin file and the skinning process.

The UI Test Feature

    Illustration  UI Test Feature Displaying a List Box.

The UI test feature (UIX_UITEST_FEATURE) can be used to test a title's custom ITitleUIPlugin implementation. The feature is enabled and started just as other features are. It provides controls and an interface to enable testing screen elements and interaction with the custom plug-in. Items can be added to a list box in order to test scrolling and greying behavior. Using the feature does not require that the title log on or be connected to Xbox Live. The feature uses the one-player logon skin.

The UI test feature is available only in debug builds.
