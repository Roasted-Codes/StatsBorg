Drop-in UI and Xbox Live Aware Titles

Xbox Drop-in UI provides the easiest path to implementing a "Live Aware" title. Xbox Live Aware allows non–Xbox Live games to take advantage of the cross-title invitation and notification features of the Xbox Live service. Live Aware titles expose presence and notification functionality so that, even when playing an offline game, a player can be notified when friends send game invitations.

Live Aware titles are not full-featured online titles. They are offline titles with the following Xbox Live capabilities only:

  • Report the user's presence to the Xbox Live service.
  • Receive and reply to game invitations.
  • Receive and reply to friend requests.
  • Display a friends list.

An Xbox Live Aware Scenario

This scenario describes a possible player experience with a Live Aware title:

User Murphistopheles sits down to play "Murph Assault", an Xbox Live title, wanting to play one of her friends. However, she discovers that none of them are currently online, and she isn't in the mood to play with strangers tonight. So she decides to play "Murph or Burn 3", an offline, Live Aware title. She puts the "Murph or Burn 3" game disc into her Xbox and begins to play. The Live Aware title silently signs in to the Xbox Live service and registers Murphistopheles' online presence with the service.

Later that evening, her friend metaMurphosis signs in to Xbox Live, popping a copy of "Murph Assault" into his Xbox. He checks his friends list, sees that Murphistopheles is currently online, and decides to send her a cross-game invite.

Murphistopheles sees an in-game notification that she has received a cross-game invite, and hits the Start button. This brings up the game's normal offline title menu—but with an additional option, Live. Selecting Live invokes the Xbox Drop-in UI Friends feature, and she can see her list of friends and the invite from metaMurphosis. Delighted that she can play with her friend, she puts in "Murph Assault" and promptly proceeds to wipe metaMurphosis from the face of the (virtual) planet.

Since "Murph or Burn 3" is a Live Aware title, Murphistopheles is not afraid to play it for a while even though she is waiting for an online invitation. Live Aware titles can give players that "connected feeling" without the effort involved in a full online implementation.

Using Drop-In UI

To use Xbox Drop-in UI to implement Xbox Live Aware, the title needs to perform the normal Xbox Drop-in UI initialization, and needs to add the Drop-In UI engine calls to its render loop. For details, see Adding Xbox Drop-in UI Support and the code examples in Using Xbox Drop-in UI.

The title calls ILiveEngine::StartFeature to start the Logon and Friends features as appropriate.

Live Aware titles should map all input that comes from any controllers as input into Port 0 when calling ILiveEngine::SetInput. When starting the Friends feature, Port 0 should always be specified. Thus, any controller will be able to interact with the friends list. Live Aware titles should not call ILiveEngine::SetInput for controllers that are not connected or are removed. For Live Aware, the Xbox Drop-in UI does not display an error message when a controller is unplugged.

Silent Logon

When an Xbox Live Aware title starts, it performs a silent logon to log the user onto the Xbox Live service, without the user controlling or requesting the logon. Silent logon must be performed when the player first presses a button after the initial interactive state in the game is reached. Games must not silently logon a player prior to the initial interactive state. Silent logon is performed by calling ILiveEngine::StartFeature for UIX_LOGON_FEATURE, with UIX_LOGON_TYPE_SILENT specified in the LogonType member of the UIX_LOGON_PARAMS structure.

Silent logon automatically logs on the most recently signed-in account. There must be no pass code on the account, and automatic logon must be enabled for the Xbox console (through a setting in the Xbox Dashboard). No users are specified, and the most recently logged on account is automatically chosen. If logon using the most recently logged on account fails, no attempt will be made to log on any other accounts that may be on the console.

The Live Menu Item

The Live option must be added to the game's start and main menus. When selected, the Live option performs one of two actions:

  • If silent logon was successful, the Live option should activate the Xbox Drop-in UI Friends list.
  • If a logon error or connection error has occurred, the Live option should display the Xbox Drop-in UI one-player logon screen, allowing the user to attempt to log on again or troubleshoot their connection. Note that this screen allows entering a pass code, whereas silent logon does not.

The Status Area

A successful logon is indicated in a status area in the game's main menu screen (for example, the text "Signed In: Murphistopheles"). Thereafter, selecting Live should bring up the friends screen.

If logon fails (for any reason, such as connectivity issues or required pass code), this is also indicated in the status on the main menu (for example, "Sign In Failed", or "Not Signed In: Pass Code Needed"). Thereafter, selecting Live should bring up the logon screen, to either enter a pass code or access the network troubleshooter.

The status area in the main menu indicates status such as "Signing In Murphistopheles", "Signed In: Murphistopheles", "Not Signed In: Pass Code Required", "Not Signed In" or "Sign in Failed". This gives feedback to the user on their current status. If there is no user account on the Xbox console, then nothing should be displayed. "Not Signed In" indicates that the user has disabled automatic logon (by changing the Xbox Dashboard setting).

Friends List

When an in-game notification is received, the game invite notification icon is displayed in-game. This is handled automatically by the Xbox Drop-in UI. The player, seeing this, can bring up the game menu and select Live in order to see the friends list. After the initial silent logon, check for batched invitations by calling ILiveEngine::GetNotifications with UIX_NOTIFICATION_MENU. If there are pending/batched invites, the Live menu can blink or indicate in some other fashion that there are pending invites. Follow the applicable TCRs in this regard.

The Xbox Drop-in UI can be asked to display a Sign Out option on the Friends list. This allows the user to sign out if he or she wishes, bringing up the logon screen. Once users are signed in, they can appear offline, if they wish, by signing out with this option. The Sign Out option is enabled by setting the SignOutEnabled member of UIX_FRIENDS_PARAMS to TRUE when calling ILiveEngine::StartFeature for UIX_FRIENDS_FEATURE.
