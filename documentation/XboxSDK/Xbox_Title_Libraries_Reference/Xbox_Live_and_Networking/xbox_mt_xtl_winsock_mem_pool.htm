Xbox Network Subsystem Pool and the System Pool

The Xbox network subsystem must deal with events that can happen at any time that may require the allocation of memory (for example, an incoming TCP packet that requires an immediate response). In order to make its memory footprint completely deterministic, the Xbox network subsystem allocates a separate pool of memory to satisfy these asynchronous allocations. This pool is allocated at Xbox network subsystem initialization time, and the parameters of that allocation can be controlled via the XNetStartup function. So as a game runs and network events occur, the game never sees the amount of available system memory decrease beyond what was originally allocated during XNetStartup. These private pool allocations are described in Private Pool Allocations.

However, when a memory allocation needs to be made by the Xbox network subsystem that is not the result of some asynchronous network event, the network subsystem makes the allocation from the system pool instead. This is the case with Xbox Winsock functions such as send, in which the data provided to the function may need to be buffered before it can be dealt with by the network subsystem. These system pool allocations are described in System Pool Allocations.
