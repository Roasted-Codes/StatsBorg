XOnlineMatchSessionCreate

Initializes a session creation request.

HRESULT XOnlineMatchSessionCreate(
    DWORD dwPublicFilled,
    DWORD dwPublicOpen,
    DWORD dwPrivateFilled,
    DWORD dwPrivateOpen,
    DWORD dwNumAttributes,
    PXONLINE_ATTRIBUTE pAttributes,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwPublicFilled
[in] Number of public slots currently occupied in the game session.
dwPublicOpen
[in] Number of unoccupied public slots in the game session.
dwPrivateFilled
[in] Number of private slots currently occupied in the game session.
dwPrivateOpen
[in] Number of unoccupied private slots in the game session.
dwNumAttributes
[in] Number of attributes that will be advertised for this session. This number is equal to the number of XONLINE_ATTRIBUTE structures pointed to by the pAttributes parameter.
pAttributes
[in, out] Pointer to an array of XONLINE_ATTRIBUTE structures for the attributes to advertise for this session. The XOnlineMatchSessionCreate function ignores the value of the fChanged member in these structures. All attributes are set to the specified values regardless of whether fChanged is TRUE or FALSE. The fChanged member is used only by XOnlineMatchSessionUpdate.
hWorkEvent
[in, optional] Handle to a caller-created event object. This object will be signaled when there is work to do by calling XOnlineTaskContinue. The caller may pass in NULL instead, indicating that the caller will poll XOnlineTaskContinue itself.
phTask
[out] Address of an XONLINETASK_HANDLE variable that receives a valid handle for this session. After the task has been pumped to completion, call XOnlineMatchSessionGetInfo to obtain the ID of the new session.

Return Values

If XOnlineMatchSessionCreate succeeds, it returns S_OK.

If XOnlineMatchSessionCreate fails, it returns one of the following error codes.

Value Description
E_OUTOFMEMORY Insufficient memory. Free some memory.
WSAEADDRNOTAVAIL See Winsock Error Codes for more information.
WSACONNRESET See Winsock Error Codes for more information.
Other See Online Errors for information about other possible errors.

The following table lists the possible values that XOnlineTaskContinue may return when called with the handle for the task created by XOnlineMatchSessionCreate.

Value Description
XONLINE_E_MATCH_INVALID_SEARCH_REQ The request is invalid. The wrong number of attributes was passed to a stored procedure. More detailed error information is returned by XOnlineTaskContinue when the search completes. For debugging purposes, call XOnlineTaskContinue to find out what went wrong.
XONLINE_E_MATCH_STRING_TOO_LONG The attribute string for the create/update/search is longer then 400 characters, or the string wasn't terminated with a null character. Limit string length to 400 wide characters, and ensure that the string is properly terminated.
XONLINE_E_MATCH_BLOB_TOO_LONG The blob for the create/update/search is longer then 800 bytes. Limit blob length to 800 bytes.
XONLINE_E_MATCH_INVALID_ATTR_TYPE The attribute type for the create/update/search is something other than user or session.
XONLINE_E_MATCH_INVALID_SESSION_ID The specified session ID is not a valid session ID returned by XOnlineMatchSessionGetInfo or XOnlineMatchSessionFindFromID.
XONLINE_E_MATCH_INVALID_OFFSET One of the attribute/parameter offsets in the request was invalid. The attribute ID was OR'd with an invalid data type. Ensure that the attribute ID is OR'd with a valid attribute data type mask (X_ATTRIBUTE_DATATYPE_INTEGER, X_ATTRIBUTE_DATATYPE_STRING, or X_ATTRIBUTE_DATATYPE_BLOB)
XONLINE_E_MATCH_INVALID_PARAM The data type that was passed to the stored procedure is not valid. Ensure that the data type sent to the stored procedure is the expected type.
XONLINE_E_MATCH_INVALID_TITLE_ID The title is not properly registered on the Matchmaking server. The attribute ID is invalid. Check the XMS file generated by MatchSim to ensure that the correct attribute ID is OR'd with the correct data type mask.
XONLINE_E_MATCH_INVALID_SESSION_ID The specified session ID is invalid.

Remarks

The XOnlineMatchSessionCreate function advertises a gaming session so that other players can participate. The XOnlineMatchSessionCreate function is called from the Xbox that is hosting the online-game session.

After XOnlineMatchSessionCreate has completed, call XOnlineMatchSessionGetInfo to obtain the session ID for the online-game session.

A private player is a player who is specifically invited to join the online-game session or joined the session directly from their Friends List. A public player is a player who finds the session through a matchmaking search. Matchmaking searches only return sessions that have openings for public players—that is, sessions in which dwPublicOpen is nonzero.

The dwPublicFilled parameter should be set to the number of people playing on the current console. It should never be zero unless the player advertising the session will not be playing in the session (very unlikely).

Set the value of dwPublicFilled to the number of people playing on the local Xbox console. As new players join the session, update the number of players using the XOnlineMatchSessionUpdate function.

Some online-game sessions are private—that is, invitation only. For these games, it is still necessary to call XOnlineMatchSessionCreate in order to obtain a session ID with which to invite other players. When calling XOnlineMatchSessionCreate, specify zero for the dwPublicOpen parameter.

Attributes can be either global or title specific. Usually, attributes are title specific—that is, they pertain to a particular title only.

Attributes are per-session, not per-user. Therefore, you cannot have multiple instances of the same attribute in a given online-game session.

Attributes are represented by the XONLINE_ATTRIBUTE structure. The type of an attribute is either integer, string, or binary object. An integer occupies eight bytes (ULONGLONG). A string has a maximum size of 400 wide characters (800 bytes). Strings are NULL terminated. A binary object has maximum size of 800 bytes, and is not NULL terminated.
