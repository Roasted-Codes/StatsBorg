XOnlineMessageSend

Sends a message to one or more players.

HRESULT XOnlineMessageSend(
    DWORD dwUserIndexSender,
    XONLINE_MSG_HANDLE hMsg,
    DWORD dwRecipientCount,
    const XUID *pxuidRecipients,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndexSender
[in] Index number of the sender.
hMsg
[in] Handle of the message to send.
dwRecipientCount
[in] Number of recipients for the message.
pxuidRecipients
[in] Array of player IDs. The number of IDs in this array must match the value in dwRecipientCount.
hWorkEvent
[in, optional] Handle of the event you want triggered every time there is work to be performed on this task. If you do not want to assign an event, set this parameter to NULL.
phTask
[out] Pointer to an XONLINETASK_HANDLE type. The handle is associated with the task started by this function.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

Titles send most types of messages using other functions, such as XOnlineGameInviteSend. Generally, only messages of type XONLINE_MSG_TYPE_TITLE_CUSTOM are sent using XOnlineMessageSend.

The handle your title receives through the phTask parameter should not be released until the send operation completes. Also, your title should not destroy the message object it receives through the hMsg parameter until the send operation is done.

When sending a message using a family title ID (for more information, see XOnlineMessageSetSendingFamilyTitleID), titles must also specify that family title ID to the XOnlineMessageEnableReceivingFamilyTitleIDs. This step is necessary in order to receive notifications of the incoming message via XOnlineGetNotificationEx or to have the message returned by XOnlineMessageEnumerate.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
