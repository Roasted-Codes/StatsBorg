XOnlineSilentLogon

Enables titles to automatically sign into the Xbox Live service using the most recently signed-in account.

HRESULT XOnlineSilentLogon(
    const DWORD *pdwServiceIDs,
    DWORD dwServices,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE pHandle
);

Parameters

pdwServiceIDs
[in] Array of DWORD values specifying the services for which tickets will be obtained to enable future secure communication with the Xbox Live services. Can be one or more of the following values:
Value Description
XONLINE_BILLING_OFFERING_SERVICE Accesses the Billing service.
XONLINE_CONTENT_AVAILABLE_SERVICE Accesses the content availability checking service. Allows use of the XOnlineOfferingIsNewContentAvailable function.
XONLINE_FEEDBACK_SERVICE Accesses the Feedback service.
XONLINE_MATCHMAKING_SERVICE Accesses the Matchmaking service.
XONLINE_SIGNATURE_SERVICE Accesses the Signature service. XOnlineLogon contacts the signature service and obtains a session signature key. The key is obtained before the logon task returns XONLINE_S_LOGON_CONNECTION_ESTABLISHED.
XONLINE_STATISTICS_SERVICE Accesses the Statistics service.
XONLINE_STORAGE_SERVICE Accesses the Storage service. You must specify this service in order to use the XOnlineStorageUpload and XOnlineStorageDownload functions.
dwServices
[in] Number of services in the pdwServiceIDs array.
hWorkEvent
[in, optional] Handle to an event that will be signaled when there is work to be done on the logon task. The event will be left unsignaled if the task is currently blocked by an asynchronous operation. This parameter can be NULL if no such event is required.
pHandle
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle representing this logon task. The handle can then be used in calls to XOnlineTaskContinue to perform work associated with this task. This task must remain active while the players are signed in to Xbox Live. This task processes the periodic online "heartbeat" that maintains the console's connection to Xbox Live.

Return Values

The return value indicates whether the asynchronous task was successfully created:

Value Description
S_OK The asynchronous logon task was successfully created.
XONLINE_E_LOGON_NO_NETWORK_CONNECTION No network connection was detected.
XONLINE_E_OUT_OF_MEMORY There was not enough memory to create the asynchronous logon task.
XONLINE_E_TASK_BUSY Another logon task is currently being performed, or the previous logon task has not been closed. (In debug builds, an assert will occur when this value is returned.)
XONLINE_E_SILENT_LOGON_DISABLED Silent Sign-in is disabled in the Xbox Dashboard.
XONLINE_E_SILENT_LOGON_NO_ACCOUNTS No Xbox Live accounts were found on this Xbox console.
XONLINE_E_SILENT_LOGON_PASSCODE_REQUIRED Sign-in failed because the account requires a pass code.

The return value does not represent the success or failure of the logon task. Once the logon task has been successfully created, XOnlineTaskContinue should be called until it returns a value other than XONLINETASK_S_RUNNING. At that point, XOnlineLogonTaskGetResults is called to determine the results of the logon. See XOnlineLogonTaskGetResults for a list of the possible return values.

Remarks

The XOnlineSilentLogon function is similar to XOnlineLogon except that XOnlineSilentLogon does not require that the caller specify the users to sign in.

XOnlineSilentLogon requires that the user enable Silent Sign-in in the Xbox Dashboard.

XOnlineSilentLogon fails if a pass code has been set for the most recently signed-in account.

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
