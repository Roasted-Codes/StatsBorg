XOnlineFriendsRequestEx

Sends a request to another user asking that the two users be placed on each other's Friends Lists. Use this function rather than XOnlineFriendsRequest when a voice attachment to the request is desired.

HRESULT XOnlineFriendsRequestEx(
    DWORD dwUserIndex,
    XUID xuidToUser,
    XONLINE_MSG_HANDLE hMsg,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwUserIndex
[in] Controller index (zero-based) of the user making the request. The user must have been successfully logged on with a previous call to XOnlineLogon.
xuidToUser
[in] XUID that uniquely identifies the recipient of the Friends request.
hMsg
[in] An XONLINE_MSG_HANDLE of an XONLINE_MSG_TYPE_FRIEND_REQUEST message to be sent with the request. If this parameter is NULL, a default message with no voice attachment will be generated.
hWorkEvent
[in, optional] Handle to a caller-created event object. This object will be signaled when there is work to do by calling XOnlineTaskContinue. The caller may pass in NULL instead, indicating that the caller will poll XOnlineTaskContinue.
phTask
[out] Pointer to a variable of type XONLINETASK_HANDLE that receives a task handle representing the verify task. Pump this task to completion using XOnlineTaskContinue. When the task is complete, call XOnlineTaskClose to close the task.

Return Values

If XOnlineFriendsRequest succeeds, the return value is S_OK.

If XOnlineFriendsRequest fails, the return value is one of the following error codes:

Value Description
E_OUTOFMEMORY Insufficient memory. Free some memory.
XONLINE_E_NOTIFICATION_NO_TASK The Friends startup task is not active. You must first call XOnlineFriendsStartup to create the startup task.
XONLINE_E_NOTIFICATION_SELF The user requested to add himself/herself to his/her own Friends List. Do not allow this. If you receive this error, display an error, or generate a sound to indicate that this is not a valid action.
XONLINE_E_NOTIFICATION_TOO_MANY_REQUESTS Too many pending requests to the service. To avoid this error, pump the handle returned by XOnlineFriendsStartup more frequently. If you get this error during a game, pump the handle returned by XOnlineFriendsStartup, and then call XOnlineFriendsRequest again.
XONLINE_E_NOTIFICATION_LIST_FULL The user's Friends List is full. An appropriate way to handle this return code is to display the message "Unable to add the specified friend because your Friend's List is full. A - continue". This error should occur only if the title is currently enumerating the Friends List. You can avoid this error by verifying that the user has fewer than 100 friends before requesting a new one. There are situations in which you will not receive this error even though the list is full.
XONLINE_E_NOTIFICATION_USER_ALREADY_EXISTS The user is already in the list. An appropriate way to handle this return code is to display the message "This player is already on your Friend's List. A - continue". Alternatively, you can do nothing. If the user forgot that a particular friend was on his list, and invites that user again, it will simply seem as though the invited user accepted immediately. This error should occur only if the title is currently enumerating the Friends List.
Other See Online Errors for information about other possible errors.

Remarks

Use XOnlineTaskContinue to pump the task returned by XOnlineFriendsStartup in order to send the request. Use XOnlineTaskContinue the task handle returned by XOnlineFriendsEnumerate to update the Friends List with the state of the request.

The title should not destroy the message object it specified through the hMsg parameter until the XOnlineFriendsRequestEx operation is complete.

The user receiving the request indicates his or her response to the receiving title, which then calls XOnlineFriendsAnswerRequest to respond to the request.
