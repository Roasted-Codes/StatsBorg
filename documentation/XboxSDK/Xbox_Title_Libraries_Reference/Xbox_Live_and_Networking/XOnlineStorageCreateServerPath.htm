XOnlineStorageCreateServerPath

Creates a server path from constituent information.

HRESULT XOnlineStorageCreateServerPath(
    DWORD dwFacility,
    ULONGLONG qwUserID,
    ULONGLONG qwTeamID,
    LPCWSTR wszStorageFileName,
    LPWSTR wszStorageServerPath,
    DWORD *pcchStorageServerPath
);

Parameters

dwFacility
[in] Storage service facility for which this path is targeted.
qwUserID
[in] Target user ID (if applicable for this facility).
qwTeamID
[in] target team ID (if applicable for this facility).
wszStorageFileName
[in] File name portion of the path (e.g. "roster.dat", or "roster*.dat").
wszStorageServerPath
[out] Receives the storage path.
pcchStorageServerPath
[in, out] Receives the storage path length (must be set to the max buffer size).

Return Values

If XOnlineStorageCreateServerPath succeeds, the return value is S_OK.

If XOnlineStorageCreateServerPath fails, the return value is one of the following error codes:

Value Description
XONLINE_E_STORAGE_INVALID_FACILITY The facility is invalid.
ERROR_INSUFFICIENT_BUFFER The download buffer is too small.

Remarks

This function creates a server path from the constituent information that can be used in the following functions:

  • XOnlineStorageUploadByServerPath
  • XOnlineStorageDownload
  • XOnlineStorageUploadFromMemory
  • XOnlineStorageDownloadToMemory
  • XOnlineStorageDeleteFile
  • XOnlineStorageEnumerate (the wszStorageFileName may contain wildcard (*) characters)

Requirements

Header: Declared in Xonline.h.

Import Library: Use Xonline.lib.
