XOnlineCompetitionManageEntrant

Updates the entrant information in a competition.

HRESULT XOnlineCompetitionManageEntrant(
    DWORD dwAction,
    DWORD dwUserIndex,
    DWORD dwTemplate,
    ULONGLONG qwTeamID,
    ULONGLONG qwCompetitionID,
    DWORD dwNumAttributes,
    const XONLINE_ATTRIBUTE *pAttributes,
    HANDLE hWorkEvent,
    PXONLINETASK_HANDLE phTask
);

Parameters

dwAction
[in] Type of update to perform on the competition's dataset. Must be one of the following values.
Action Description
XONLINE_COMP_ACTION_JOIN Join the competition as a public entrant.
XONLINE_COMP_ACTION_JOIN_PRIVATE Join the competition as a private entrant. If no private slots are available, join as a public entrant.
XONLINE_COMP_ACTION_WITHDRAW Withdraw from the competition.
XONLINE_COMP_ACTION_EJECT Eject a player from the competition. This action can be performed only by the competition owner.
XONLINE_COMP_ACTION_FORFEIT Cause a forfeit.
dwUserIndex
[in] Controller index (zero-based) of the player. The player must have been successfully logged on with a previous call to XOnlineLogon.
dwTemplate
[in] ID of a template describing the competition. Obtain this ID from XLAST when you create the Competition Template.
qwTeamID
[in] Team ID. The value in this parameter is used only if this is a team competition rather than a competition between individuals. If this is not a team competition, set this value to zero.
qwCompetitionID
[in] Competition in which the player is participating.
dwNumAttributes
[in] Number of additional attributes associated with the entrant.
pAttributes
[in] Array of title-specific attributes describing the entrant.
hWorkEvent
[in, optional] Handle of an event you want to associate with this task. If you do not want to assign an event, set this parameter to NULL.
phTask
[out] Pointer to an XONLINETASK_HANDLE type. The handle is associated with the task started by XOnlineCompetitionManageEntrant.

Return Values

Returns S_OK if successful, an error code otherwise.

Remarks

When joining a tournament, a user is required to check in for all scheduled matches in that tournament. Tournament match no-shows are reflected in the user's NoShowRating. The NoShowRating is a ratio of tournament match no-shows to tournament match sessions played. The NoShowRating is returned by XOnlineLogon in the XUID structure.
