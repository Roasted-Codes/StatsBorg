XINPUT_DEBUG_KEYSTROKE

The XINPUT_DEBUG_KEYSTROKE structure is used in the XInputDebugGetKeystroke function to retrieve a keyboard input event.

typedef struct _XINPUT_DEBUG_KEYSTROKE {
  BYTE VirtualKey;
  CHAR Ascii;
  BYTE Flags;
} XINPUT_DEBUG_KEYSTROKE, *PXINPUT_DEBUG_KEYSTROKE;

Members

VirtualKey
The virtual-key code of the key causing the event. See xkbd.h for a list of the valid virtual-key (VK_xxx) codes.
Ascii
The ASCII character that is the result of the input event. If there is no ASCII character corresponding to the event, Ascii is zero.
Flags
Flags that indicate the type of keyboard event being reported, as well as additional information about the keyboard state at the time of this event. This member can be any combination of the following values:
Flag Meaning
XINPUT_DEBUG_KEYSTROKE_FLAG_KEYUP The key was released. If not set, the key was pressed.
XINPUT_DEBUG_KEYSTROKE_FLAG_REPEAT A repeat of key that is held.
XINPUT_DEBUG_KEYSTROKE_FLAG_CTRL The left or right CTRL key is down.
XINPUT_DEBUG_KEYSTROKE_FLAG_SHIFT The left or right SHIFT key is down.
XINPUT_DEBUG_KEYSTROKE_FLAG_ALT The left or right ALT key is down.
XINPUT_DEBUG_KEYSTROKE_FLAG_CAPSLOCK CAPS LOCK is set.
XINPUT_DEBUG_KEYSTROKE_FLAG_NUMLOCK NUM LOCK is set.
XINPUT_DEBUG_KEYSTROKE_FLAG_SCROLLLOCK SCROLL LOCK is set.

Remarks

A virtual-key code is a byte value that represents a particular physical key on the keyboard, not the character or characters (possibly none) that the key can be mapped to based on keyboard state. The keyboard state at the time a virtual key is pressed modifies the character reported. For example, VK_4 might represent a "4" or a "$", depending on the state of the SHIFT key.

A reported keyboard event includes the virtual key that caused the event, whether the key was pressed or released (or is repeating), and the state of the keyboard at the time of the event. The keyboard state includes information about whether any CTRL, ALT, or SHIFT keys are down, as well as whether any of the keyboard toggle states (CAPS LOCK, SCROLL LOCK, and NUM LOCK) are currently set.

If the keyboard event represents an ASCII character (for example, pressing the "A" key), the Ascii member will contain that ASCII character. Otherwise, Ascii will contain the value zero.

Requirements

Header: Declared in xkbd.h, for include use xtl.h.
