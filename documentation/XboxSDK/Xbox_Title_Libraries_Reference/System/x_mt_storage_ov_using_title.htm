Using the Title Data Region

Titles use persistent storage to maintain data across game sessions without the danger of the data being deleted (as in the utility region), and without necessarily associating the data with a specific save game (as in user data). Titles will use this region for a variety of purposes, and in general, there are few constraints on how the space should be used and managed.

When a game is launched, it is provided a drive letter (T:\) mapped to its specific directory within the persistent data region. Within this region, titles are essentially free to use the space as they see fit—creating additional subdirectories, adding content, and so on.

To free data in the title's persistent data region, the user must delete all of the storage in that region. This is done from within the Xbox Dashboard, and there is no option to delete individual pieces of the data in this region. This user is simply presented with the option of deleting all storage used by a specific title.
