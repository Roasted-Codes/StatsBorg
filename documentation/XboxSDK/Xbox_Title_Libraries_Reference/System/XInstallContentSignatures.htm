XInstallContentSignatures

Installs and resigns a content signature metadata file.

BOOL XInstallContentSignatures(
    DWORD dwTitleID,
    LPCSTR lpSourceMetadataFileName,
    LPCSTR lpDestinationDirectory
);

Parameters

dwTitleID
[in] Title identifier used to calculate the signature keys. If zero is specified for TitleID, the title identifier of the calling title is used.
lpSourceMetadataFileName
[in] Pointer to a null-terminated string that specifies the full path and file name of the source metadata file.
lpDestinationDirectory
[in] Pointer to a null-terminated string that specifies the directory where the content resides and into which the metadata file will be installed. Use XGetContentInstallLocation to determine the appropriate content installation directory.

Return Values

If the function succeeds, the return value is TRUE.

If the function fails, the return value is FALSE. To get extended error information, call GetLastError.

Remarks

Installation of packaged contents is typically handled by either a content download installer or a DVD-based package installer. However, a title that is installing content by copying individual files may use XInstallContentSignatures to install and re-sign the content signature metadata file.

XInstallContentSignatures will copy the metadata, recalculate the signature of the complete metadata file, and recalculate all non-literal signatures. Since computation of non-literal signatures relies on the data described by the signatures, all other files must be copied into the destination directory prior to calling XInstallContentSignatures.

Requirements
