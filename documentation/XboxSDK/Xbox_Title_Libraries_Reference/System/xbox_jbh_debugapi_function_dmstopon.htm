DmStopOn

Some debugging events such as DM_BREAK require title execution to be suspended. But there are some debugging events where thread suspension is an option. For these events, you can elect to have the debugging subsystem suspend title execution by calling the DmStopOn function.

DMHRAPI DmStopOn(
  DWORD dwStopFlags, 
  BOOL fStop
  );

Parameters

dwStopFlags
The first parameter, dwStopFlags, contains some combination of the following values:
Value Meaning
DMSTOP_CREATETHREAD Suspend title execution when a thread is created.
DMSTOP_FCE Suspend title execution when a first-chance exception occurs.
DMSTOP_DEBUGSTR Suspend title execution when a string is sent to debug out.
fStop
If the fStop parameter is TRUE, the debugging subsystem will suspend title execution when the specified event occurs and notify any debug event handlers that are registered to receive DM_EXEC events. When the handler is called, the DM_STOPTHREAD flag of the dwNotification parameter will be set. If fStop is FALSE, the title will not be suspended.

Return Values

Returns XBDM_NOERR if successful, an error value otherwise.

Requirements

Header: Declared in XbDm.h and XboxDbg.h
Import Library: Use XbDm.lib for target system code; and XboxDbg.lib for development system code.
