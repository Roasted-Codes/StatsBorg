Using the Publisher Common Data Region

To access the publisher common data region, a title must mount the region with the XMountAlternateTitle function, specifying the publisher common data region alternate title ID as the alternate title ID to mount.

Publisher Common Data Region Alternate Title ID

The publisher common data region alternate title ID (PCDRATI) is an alternate title identifier that provides access to the common data region. A unique PCDRATI exists for each publisher. The PCDRATI is a 32-bit value formed by combining the publisher ID in the upper 16 bits with the value 0xFFFF in the lower 16 bits. For example, a publisher with ID 0xABCD would have a PCDRATI of 0xABCDFFFF. Publishers must not create primary title IDs that contain 0xFFFF in the lower 16 bits, because this would conflict with the publisher's PCDRATI.

To be permitted access to the common data region, a title must include the PCDRATI in the title's list of alternate IDs when the title is submitted for certification. The PCDRATI is associated with the title in addition to the title's primary ID and any other alternate title IDs that the publisher desires.

Mounting and Using the Publisher Common Data Region

To access the common data region, the title must mount a logical drive letter that can be used to access the region. To mount the publisher common data region, call XMountAlternateTitle with "T:\" in the lpRootPath parameter, and with the PCDRATI in the dwAltTitleId parameter. If the call is successful, the function returns ERROR_SUCCESS and writes the drive letter of the newly mounted common data region to the variable pointed to by the pchDrive parameter.

The returned drive letter is a mounted drive that can be used with the standard file I/O functions (such as CreateFile, ReadFile, and WriteFile) to read and write data in the region.

If the function fails, it does so for one of the following reasons:

  • If the alternate title ID is not valid or was not included in the title's list of alternate title IDs when the title was signed.
  • If the title's publisher does not match the publisher portion of the PCDRATI.
  • If a drive other than "T:\" is specified for lpRootPath.

As with all data written to the hard disk, titles should sign data written to the common data area by using XCalculateSignatureBeginEx, XCalculateSignatureUpdate, and XCalculateSignatureEnd. When calling XCalculateSignatureBeginEx, specify the PCDRATI in the dwAltTitleId parameter, thus securely signing the data while also permitting multiple titles access to the data.

When the title no longer needs to access the common data region, it should unmount the region by calling XUnmountAlternateTitle.

For general information about the common data region, see Publisher Common Data Storage Region.
