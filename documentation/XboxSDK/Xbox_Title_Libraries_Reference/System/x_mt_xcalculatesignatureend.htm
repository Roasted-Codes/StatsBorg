XCalculateSignatureEnd

The XCalculateSignatureEnd function completes the calculation of a signature over a save game or other data set, returns the calculated signature, and frees the resources associated with the signature context.

DWORD XCalculateSignatureEnd(
  HANDLE hCalcSig,
  PVOID pSignature
);

Parameters

hCalcSig
[in] Handle to the signature being calculated. The handle must have been returned by a previous call to XCalculateSignatureBegin.
pSignature
[out] Pointer to a buffer that receives the signature of the data. Use XCalculateSignatureGetSize to determine the necessary buffer size to allocate and pass in pSignature. For save game signatures only, pSignature can be a pointer to an XCALCSIG_SIGNATURE structure.

This parameter can be NULL if the signature is not needed; in this case, the signature handle is closed and the associated resources are freed, but no signature is calculated.

Return Values

If the function succeeds, the return value is ERROR_SUCCESS.

If the function fails, the return value is an error code defined in winerror.h. The function does not use SetLastError to set the calling thread's last-error code.

Remarks

XCalculateSignatureEnd will calculate a signature over all of the data that has been added to the signature context with XCalculateSignatureUpdate since the context was initially opened using XCalculateSignatureBegin.

Prior to calling XCalculateSignatureEnd, call XCalculateSignatureGetSize to determine the buffer size necessary to contain the returned signature. Allocate a buffer of the required size, and pass a pointer to that buffer in the pSignature parameter.

The signature returned in pSignature can be written out to storage along with the data that it is the signature of, or it can be compared (with a simple memory comparison) against a signature read in from a save game or other data set that needs to be verified.

For each call to XCalculateSignatureBegin, call XCalculateSignatureEnd in order to free the allocated resources. The handle passed in the hCalcSig parameter is automatically closed by XCalculateSignatureEnd, and the resources associated with the signature context are freed. If the calculated signature is not needed (for example, when backing out of an operation for which there was a failure), call XCalculateSignatureEnd with NULL for the pSignature parameter in order to free up the associated resources without actually calculating the signature.

If XCALCSIG_FLAG_SAVE_GAME was specified when the signature context was created, the hash is computed over the specified data set and is signed with a key that is specific to, and only accessible by, the calling title. If XCALCSIG_FLAG_NON_ROAMABLE was specified when the signature context was created, the hash is additionally signed with a key unique to the current Xbox console, so the signature will only match a signature calculated over the same data, by the same title, on the same console.

For save game signatures (XCALCSIG_FLAG_SAVE_GAME or XCALCSIG_FLAG_NON_ROAMABLE was specified when the signature context was created), the signature returned in the pSignature buffer is an XCALCSIG_SIGNATURE structure.

Requirements
