Thread Handles and Identifiers

When a new thread is created by the CreateThread function, a handle to the thread is returned. The handle is valid until closed, even after the thread it represents has been terminated.

The CreateThread function also returns an identifier that uniquely identifies the thread throughout the system. A thread can use the GetCurrentThreadId function to get its own thread identifier. The identifiers are valid from the time the thread is created until the thread has been terminated.

A thread can use the GetCurrentThread function to retrieve a pseudo handle to its own thread object. This pseudo handle represents the currently running thread. To get the real handle to the thread, given the pseudo handle, use the DuplicateHandle function.
