Thread Local Storage

All threads share the virtual address space and the global variables of a title. The local variables used in a function being executed by a thread are local to each thread that runs the function because they are allocated on each thread's own stack. However, the static or global variables used by that function have the same value for all threads. With thread local storage (TLS), you can create a unique copy of a variable for each thread. Using TLS, one thread allocates an index that can be used by any thread to retrieve that thread's unique copy of the data.

Use the following steps to implement TLS:

  1. Use the TlsAlloc function (usually during title initialization) to allocate a TLS index.
  2. For each thread that needs to use the TLS index, allocate dynamic storage, then use the TlsSetValue function to associate the index with a pointer to the dynamic storage.
  3. When you need a thread to access its storage, specify the TLS index in a call to the TlsGetValue function to retrieve the pointer.
  4. When each thread no longer needs the dynamic storage that it has associated with a TLS index, it must free that storage. When all threads have finished using a TLS index, use the TlsFree function to free the index.

The constant TLS_MINIMUM_AVAILABLE defines the minimum number of TLS indexes available.

Thread local storage is also often used by libraries in conjunction with thread notify routines (see XRegisterThreadNotifyRoutine). Perform the initial TLS index allocation when the library is initializing, usually when XRegisterThreadNotifyRoutine is used to register the library for thread notifications. When a new thread is created, the notification routine is notified. In the notification routine (and also for the initial thread), dynamically allocate the needed memory, and use TlsSetValue with the TLS index (from the initial TlsAlloc call) to store the memory pointer at that TLS index. Then, within other functions in the library, use the TLS index in a call to TlsGetValue to access the per-thread private data for the calling thread. When a thread detaches from the library, free the dynamically allocated memory. When the library is unloaded (if it eventually is) call TlsFree to free the allocated TLS index.

For an example illustrating the use of thread local storage, see Using Thread Local Storage.
