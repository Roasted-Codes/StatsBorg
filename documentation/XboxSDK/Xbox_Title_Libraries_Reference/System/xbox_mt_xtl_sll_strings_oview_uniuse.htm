Using Unicode and ANSI Functions

When calling an XTL function that has text arguments, you must know whether the function accepts ANSI or Unicode characters, and you must pass in a string type that correctly matches the type expected by the function.

Some functions are implemented as both ANSI and Unicode versions, some as ANSI or Unicode only, and some as functions that require both ANSI and Unicode parameters. In general, Unicode versions are provided for functions that deal with strings that may eventually be displayed to the user. For optimal performance, ANSI versions are provided for functions that deal with strings used internally by the title, such as filenames.

This means that in your code you will likely have calls to both ANSI and Unicode functions and be working with both ANSI and Unicode strings.

XTL String Functions lists the versions supported by the various string functions.


As an example, the DeleteFile function accepts only ANSI strings:

char szName[] = "ai_parms.txt";
DeleteFile( szName );

While the XSetNickname function accepts only Unicode strings:

WCHAR wszName[] = L"Joey";
XSetNickName( wszName, TRUE );

And the XDeleteSaveGame function requires one ANSI parameter and one Unicode parameter:

CHAR szPath[50];
WCHAR wszGameName[50];

lstrcpy( szPath, "t:\\saves" );
lstrcpyW( wszGameName, L"AtTheBeach" );
XDeleteSaveGame( szPath, wszGameName );

Some functions have both ANSI and Unicode versions, and you can choose which version you want to use by adding, or omitting, a W (which stands for wide character) to the end of the function name. An example of this is the lstrcpy function above; the lstrcpy version is ANSI, and the lstrcpyW version is Unicode. You can also refer to the ANSI version by using lstrcpyA as the function name (the A stands for ANSI). So in this example, lstrcpy and lstrcpyA refer to the same function, the ANSI version of lstrcpy.

By default, for functions that have both ANSI and Unicode versions, omitting the final A or W results in the ANSI version being used. This default can be changed using the UNICODE compiler define, discussed in XTL String Data Types.

A string literal can be specified as Unicode by prepending an L to the beginning of the string literal. An example of this is seen above:

lstrcpyW( wszGameName, L"AtTheBeach" );

This line of code uses the Unicode string L"AtTheBeach" as the second argument to the Unicode function lstrcpyW. String literals without the prepended L are ANSI.
