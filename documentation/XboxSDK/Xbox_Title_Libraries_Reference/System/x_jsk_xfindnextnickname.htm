XFindNextNickname

The XFindNextNickname function continues a nickname cache enumeration begun with a call to XFindFirstNickname.

BOOL XFindNextNickname(
    HANDLE hFindNickname,
    LPWSTR lpNickname,
    UINT uSize
);

Parameters

hFindNickname
[in] A nickname search handle returned by a previous call to the XFindFirstNickname function.
lpNickname
[out] Pointer to a buffer that receives the next enumerated nickname from the local cache. Nicknames are limited to MAX_NICKNAME characters (including the terminating null character).
uSize
[in] The size, in characters, of the lpNickname buffer.

Return Values

If the function succeeds, the return value is TRUE and lpNickname will contain the next enumerated nickname.

If the function fails or there are no more nicknames to enumerate, the return value is FALSE.

If the function fails, it does not call SetLastError. A title cannot call GetLastError for extended error information.

Remarks

XFindNextNickname continues a nickname enumeration on the specified search handle. When the enumeration is complete, the function returns FALSE. When the search handle is no longer needed, close it using the XFindClose function.

Nicknames that will not fit in a buffer of uSize characters will be skipped. This allows a title to use the uSize parameter to filter the results of the enumeration to only those nicknames with a length that the title is capable of displaying. The calculation includes the terminating null character. So, for example, to request only nicknames with three or fewer characters, specify four for uSize.

The nicknames are returned in order from the newest in the cache to the oldest.

Requirements

Header: Declared in Xbox.h, for include use Xtl.h.
Import Library: Use Xapilib.lib
Unicode: Implemented as Unicode only.
