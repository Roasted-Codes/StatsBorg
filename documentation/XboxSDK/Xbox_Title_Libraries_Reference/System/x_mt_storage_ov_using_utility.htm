Using the Utility Region

The title's utility region, which comprises a single disk partition, is the simplest region to use and manage. The region is exposed to the title through the Z:\ drive. The Xbox system needs to understand nothing about how the data is used by the title. Titles have complete use of an entire partition, thus removing the possibility of corrupting other title's data.

There are many uses for this utility space besides the caching of DVD data. Examples include storing environment deformations, implementing virtual memory, building up sound or other dynamic data, and so on.

Mounting the Utility Region

You can set flags at build time that determine the initialization options for the utility region. These flags are passed to the XBE Image Builder (ImageBld) using the /INITFLAGS switch when the .xbe file is built for the title. The XINIT_MOUNT_UTILITY_DRIVE flag specifies that a utility region is automatically mounted for the title at initialization time. This flag is the default if no flags are set. The XINIT_FORMAT_UTILITY_DRIVE flag, in combination with XINIT_MOUNT_UTILITY_DRIVE, specifies that the partition is to be formatted by the system at title initialization time before the partition is given to the title to use.

These options allow titles that do not need utility space not to incur the overhead of mounting the partition. Similarly. if the title needs to use the partition as scratch space only, it can ensure that it always gets the partition in a clean, known state.

The XMountUtilityDrive function allows a title to mount the title's utility storage region at run time (if it wasn't already mounted during initialization).

If a title does not mount a utility partition at all, then no partition is marked as having been used by the title. If a title specifies at build time that the utility space should be formatted at title load, the utility partition it uses is not marked as being in use, because the title does not need to recover the data the next time the game runs.

When a utility partition needs to be assigned to a title, the system assigns a partition based on the following scheme:

  • If there is a utility partition already in use by that title, that partition is assigned.
  • If there is a utility partition that is not in use by any title, that partition is assigned.
  • The least-recently-used partition is assigned.
