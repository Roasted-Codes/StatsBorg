DmStartProfiling

Begins collecting profiling data. This data may later be analyzed by using the Call-Attributed Profiler (XbPerf) or XbPerfView.

HRESULT DmStartProfiling(
  LPCSTR szFileName, 
  DWORD  dwDataBufferSize
  );

Parameters

szFileName
Pointer to a null-terminated string that specifies the path and file name where the profiling log data is written.
dwDataBufferSize
Size, in bytes, of the data buffer that is allocated to hold profiling data. Passing a value of 0 for dwDataBufferSize causes DmStartProfiling to use its default buffer size of 32 MB.

Return Values

If DmStartProfiling begins profiling successfully, XBDM_NOERR is returned. If profiling is already in progress, the function returns XBDM_PROFILEALREADYSTARTED. Otherwise, an XBDM error value is returned (see Debugger API Error Codes for possible values).

Remarks

If profiling is already in progress, a second call to DmStartProfiling will restart profiling. Opening the file specified in szFileName is deferred until DmStopProfiling is called, making it faster to call DmStartProfiling every frame. This feature allows the title to call DmStartProfiling every frame (with different or same parameters) and call DmStopProfling to write out the data for slow frames. For frames where DmStopProfling is not called, the profiling data is discarded automatically.

Do not confuse the DmStartProfiling function with DmStartProfile. DmStartProfiling is used to begin gathering profiling information; DmStartProfile is used to register a profiler interrupt handler.

Requirements

Header: Declared in XbDm.h
Import Library: XbDm.lib
