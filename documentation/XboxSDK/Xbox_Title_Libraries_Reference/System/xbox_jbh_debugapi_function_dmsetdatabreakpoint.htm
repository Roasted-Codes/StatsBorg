DmSetDataBreakpoint

Sets a hardware breakpoint that suspends title execution when a particular section of memory is referenced.

DMHRAPI DmSetDataBreakpoint(
  PVOID addr, 
  DWORD dwType, 
  DWORD dwSize
  );

Parameters

addr
Address of the memory to watch.
dwType
Type of access to watch for.
Value Meaning
DMBREAK_NONE Clear this data breakpoint.
DMBREAK_WRITE Break if the address is written to.
DMBREAK_READWRITE Break if the address is read from or written to.
DMBREAK_EXECUTE Break if the address is executed.
dwSize
The size, in bytes, of the memory to be watched. The only allowable values for this parameter are 1, 2, and 4.

Return Values

Returns XBDM_NOERR if successful, an error value otherwise.

Remarks

A data breakpoint occurs when particular address has been referenced by the processor for reading, writing, or execution. These are also known as "hardware breakpoints". They are most commonly used to break when the contents of memory have changed.

Requirements

Header: Declared in XbDm.h and XboxDbg.h
Import Library: Use XbDm.lib for target system code; and XboxDbg.lib for development system code.
