Terminating a Thread

A thread executes until one of the following events occurs:

  • The thread calls the ExitThread function.
  • The thread function returns.

The GetExitCodeThread function returns the termination status of a thread. While a thread is executing, its termination status is STILL_ACTIVE. When a thread terminates, its termination status changes from STILL_ACTIVE to the exit code of the thread. The exit code is either the value specified in the call to ExitThread or the value returned by the thread function.

When a thread terminates, the state of the thread object changes to signaled, releasing any other threads that had been waiting for the thread to terminate. For more about synchronization, see Synchronizing Execution of Multiple Threads.

If a thread is terminated by ExitThread or returns from its thread function, the system calls the notification function of each library registered with XRegisterThreadNotifyRoutine.

There is no way for one thread to terminate another. The common method to signal a looping thread to terminate is as follows:

  • Use a global variable or create an event object using CreateEvent.
  • Create the thread. It begins execution.
  • At some point in the loop (or occassionally as the thread executes), the thread monitors the variable or the event state (by calling the WaitForSingleObject function with a wait time-out interval of zero).
  • The thread returns from its thread function or exits when the global variable is set to a specific value or when the event is set to the signaled state (WaitForSingleObject returns WAIT_OBJECT_0).
