SwitchToFiber

The SwitchToFiber function schedules a fiber. The caller must be a fiber. 

VOID SwitchToFiber(
  LPVOID lpFiber  // fiber to schedule
);

Parameters

lpFiber
[in] Specifies the address of the fiber to schedule.

Return Values

This function does not return a value.

Remarks

Fibers are created with the CreateFiber function. Before fibers associated with a particular thread can be scheduled, that thread must call ConvertThreadToFiber to become a fiber and to initialize the fiber state information.

The SwitchToFiber function saves the state information of the current fiber, restores the state of the specified fiber, and resumes execution of the specified fiber.

Fibers created with CreateFiber can be scheduled by fibers created on other threads. To do so, call SwitchToFiber with the fiber address returned when the other thread called CreateFiber. Fibers created by ConvertThreadToFiber can only be scheduled (with SwitchToFiber) by the same thread (and its fibers) that called ConvertThreadToFiber. When scheduling fibers on other threads, use proper synchronization between the two threads to avoid unpredictable results.

Requirements

Header: Declared in Winbase.h, for include use Xtl.h.
Import Library: Use Xapilib.lib
