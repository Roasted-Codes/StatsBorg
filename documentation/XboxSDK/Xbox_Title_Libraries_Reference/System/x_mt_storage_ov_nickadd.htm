Adding Nicknames to the Cache

The XSetNickname function adds a nickname to the cache. When a new nickname is added, it is placed at the top of the list of nicknames or moved to the top if it already existed in the cache. If the cache is full when a new nickname is added, the oldest nickname will be removed from the cache.

Since the nickname cache is local to the console, a player's nickname will not be available on other Xbox consoles unless the user also adds the nickname on the new console. However, a title could store the nickname associated with a save game in the save game data, and when loading that save game, add the name to the cache. Thus, after loading a save game from another Xbox, the player would find her preferred nickname in the next nickname list presented.

Case Sensitivity

Case sensitivity is often important in user nicknames. Many nicknames rely on specific capitalization for full meaning or effect, and players are frustrated when their capitalization is not maintained. Capitals are used to delineate words, distinguish clans, and just look "cool." The Xbox nickname cache will honor and store the case sensitivity of the nickname when an application adds it to the cache. If the nickname being added with XSetNickname is already in the cache, the game can specify whether it wants to use the old or new capitalization.
