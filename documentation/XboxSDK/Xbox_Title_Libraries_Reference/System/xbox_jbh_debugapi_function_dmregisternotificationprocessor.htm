DmRegisterNotificationProcessor

Registers notification handlers for debug events that are triggered through calls to DmSendNotificationString.

DMHRAPI DmRegisterNotificationProcessor(
  PDMN_SESSION Session, 
  LPCSTR szType, 
  PDM_EXT_NOTIFY_FUNCTION pfn
  );

Parameters

Session
Pointer to a notification session that was created with a call to DmOpenNotificationSession.
szType
LPCSTR that represents the notification processor.
pfn
Pointer to a notification processor. Notification processors must take the following form: 
DWORD __stdcall my_processor(LPCSTR szNotification);

Return Values

Returns XBDM_NOERR if successful, an error value otherwise.

Remarks

Code on the XDK sends a notification by calling DmSendNotificationString and passing a string in the form, "handler!data data data". The development system can receive these notifications by opening a notification session and installing a notification processor into it with szType of (in this case) "handler". The notifier calls the notification processor that was registered with a szType maching the prefix of the notification string that comes in. The entire string (including the "handler!") is passed to the registered notification handler as the szNotification parameter.

Cease Notifications

To tell the debugging subsystem to quit sending notifications to your handler, you need to close the session in which the handler was registered.

Requirements

Header: Declared in XbDm.h and XboxDbg.h
Import Library: Use XbDm.lib for target system code; and XboxDbg.lib for development system code.
