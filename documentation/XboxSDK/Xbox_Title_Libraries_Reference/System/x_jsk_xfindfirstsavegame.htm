XFindFirstSaveGame

The XFindFirstSaveGame function begins an enumeration of a title's save games on the specified device.

HANDLE XFindFirstSaveGame(
  LPCSTR lpRootPathName,                
  PXGAME_FIND_DATA pFindGameData         
);

Parameters

lpRootPathName
[in] Pointer to a null-terminated string that specifies the root directory of the user data storage region to search. The parameter must contain only a drive letter, colon, and trailing backslash; a subdirectory cannot be specified.
pFindGameData
[out] Pointer to an XGAME_FIND_DATA structure that receives information about the first save game found.

Return Values

If the function succeeds, the return value is a search handle used in a subsequent call to XFindNextSaveGame or XFindClose.

If the function fails, the return value is INVALID_HANDLE_VALUE. To get extended error information, call GetLastError.

Remarks

The storage region specified in lpRootPathName can be the title's user data region on the hard disk (U:\), a mapped user data region for an alternate title (X:\), or the title's user data region of a mounted memory unit (F:\ through M:\).

XFindFirstSaveGame opens a search handle and returns information about the first save game found. The search handle can then be used in XFindNextSaveGame to continue enumeration of the available save games. When the search handle is no longer needed, close it using the XFindClose function.

The lpRootPathName parameter determines the storage device that will be searched. The title's user data region of the specified device will be examined, and the save games located there enumerated. The exception to this general behavior occurs when the alternate title drive (X:\) is specified. In that case, if an alternate title's user data region has been successfully mapped with XMountAlternateTitle, the save games in the mapped region will be enumerated.

Requirements

Header: Declared in Xbox.h, for include use Xtl.h.
Import Library: Use Xapilib.lib
Unicode: The function is ANSI only, but the returned structure contains both Unicode and ANSI members.
