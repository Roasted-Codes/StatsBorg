XCalculateSignatureBeginEx

The XCalculateSignatureBeginEx function begins the process of calculating a signature over a save game or other data set. Unlike XCalculateSignatureBegin, the signature will be calculated for a specified alternate title rather than for the current title.

HANDLE XCalculateSignatureBeginEx(
  DWORD dwFlags,
  DWORD dwAltTitleId
);

Parameters

dwFlags
[in] Value indicating how the signature should be calculated. This parameter can be 0 or the following flag:
Value Meaning
XCALCSIG_FLAG_NON_ROAMABLE Specifies that the signature be calculated over the data such that the data is non-roamable. That is, the data is only valid on the current Xbox console.
dwAltTitleId
[in] Alternate title identifier for which the signature will be calculated. This must be an identifier listed in the title's certificate as a valid alternate title.

Return Values

If the function succeeds, the return value is a signature handle that can be used in subsequent calls to XCalculateSignatureUpdate and XCalculateSignatureEnd.

If the function fails, the return value is INVALID_HANDLE_VALUE. To get extended error information, call GetLastError.

Remarks

XCalculateSignatureBeginEx opens a signature calculation handle in preparation for calculating a signature over a data set (typically the data representing a save game). Data is added to the running hash with calls to XCalculateSignatureUpdate. Multiple calls to add data can be made if desired, and then the hash is completed and signed with a call to XCalculateSignatureEnd. The hash is signed using a per-title key, and if XCALCSIG_FLAG_NON_ROAMABLE was specified in dwFlags when XCalculateSignatureBeginEx was called, also using a key unique to the current Xbox console.

Only data that has not been tampered with and that is signed by the specified title will compute to the same signature. XCalculateSignatureBeginEx allows a title to load and verify save games that were saved by the title's approved alternate titles. XMountAlternateTitle must first be used to mount the alternate title's data region, then save games in that region can be enumerated, loaded, and verified.

If XCALCSIG_FLAG_NON_ROAMABLE is specified, the hash is additionally signed by a key unique to the current Xbox console. The data will compute to a different signature on any other console, thus will only be valid on the initial console that first performs the calculation and writes the data. For example, if a save game was signed and the game and its signature written to an MU, and that MU taken to another Xbox console, and a player attempted to load the save game on the new console, the signature would not match and the game title should fail the load of that save game.

After calling XCalculateSignatureBeginEx, use the returned handle in calls to XCalculateSignatureUpdate to add the data for which the signature will be calculated, and then finally call XCalculateSignatureEnd to complete signature calculation and to free the signature handle and other resources associated with the signature.
