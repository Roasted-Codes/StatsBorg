timeGetTime

The timeGetTime function retrieves the number of milliseconds that have elapsed since the system was started. It is limited to the resolution of the system timer, which is one millisecond (1 ms). This function is identical to GetTickCount.

DWORD timeGetTime(VOID);

Parameters

This function has no parameters.

Return Values

The return value is the number of milliseconds that have elapsed since the system was started.

Remarks

The elapsed time is stored as a DWORD value. Therefore, the time will wrap around to zero if the system is run continuously for 2^32 milliseconds, about 49.71 days. This can cause problems in code that uses the return value of timeGetTime directly in computations, particularly where the value is used to control code execution. You should always use the difference between two timeGetTime return values in computations, as shown in the example below.

If you need a higher resolution timer, use a high-resolution timer.

It is not necessary to use the timeBeginPeriod and timeEndPeriod functions in order to use timeGetTime. The timer resolution is always 1 millisecond. The timeBeginPeriod and timeEndPeriod functions are not implemented in the XTL.

The only difference between timeGetTime and timeGetSystemTime is that timeGetSystemTime uses the MMTIME structure to return the system time. Thus timeGetTime has slightly less overhead than timeGetSystemTime.

Example

The following example demonstrates how to handle timer wrap around:

DWORD dwStart = timeGetTime();

// Stop if this has taken too long.
if( timeGetTime() - dwStart >= TIMELIMIT )
    Cancel();

where TIMELIMIT is the time interval of interest to the application.
