XMemSize

Application-defined memory callback function that determines the size of an allocated memory block. This callback function replaces the memory size function used by the XTL libraries.

SIZE_T WINAPI XMemSize(
    PVOID pAddress,
    DWORD dwAllocAttributes
);

Parameters

pAddress
[in] Specifies the base address of the memory block for which the size is to be determined. This parameter must be the address returned by the XMemAlloc function when the memory was allocated.
dwAllocAttributes
[in] An XALLOC_ATTRIBUTES value that specifies the allocation attributes of the memory block for which the size is to be determined. These attributes should match those that were specified when the memory was allocated.

Return Values

If the function succeeds, the return value is the size, in bytes, of the allocated memory block.

If the function fails, the return value is −1.

Remarks

If a title implements an XMemSize function, all memory size queries performed by the XTL libraries will instead call the title's XMemSize routine. The title must return the memory size. The title can simply use the XPhysicalSize and HeapSize functions to provide the underlying heap support, if desired. Alternatively, the title can implement its own heap routines and thus replace the default heap allocator.

XTL components that need to determine memory size (graphics, sound, networking, and so on) all call XMemSize instead of other memory size routines. The default XMemSize then calls XPhysicalSize or HeapSize as appropriate. A title can provide its own XMemSize function, thus overriding the default, and allowing the title to trace XTL size queries and to replace the heap allocator. The title should provide all of the allocator replacement functions (XMemAlloc, XMemFree, and XMemSize) or none at all. The allocator functions to use (either the default or title-defined versions) are determined at link time, so the title imposes no additional overhead by supplying its own versions.

The title can, if desired, call its own implementation of XMemSize as well, thus routing all title and XTL memory size queries through a single routine.
