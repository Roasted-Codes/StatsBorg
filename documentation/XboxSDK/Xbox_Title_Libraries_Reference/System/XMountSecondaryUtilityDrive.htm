XMountSecondaryUtilityDrive

Mounts the title's secondary utility storage region. A hard disk partition is selected using the current LRU mechanism and assigned to the title for use as temporary storage and caching of game data on the N: drive.

BOOL XMountSecondaryUtilityDrive();

Parameters

None.

Return Values

If the function succeeds, the return value is TRUE.

If the function fails, the return value is FALSE. To get extended error information, call GetLastError.

Remarks

This function is used to mount an extra hard disk partition that can be used for temporary scratch space, caching data from the game DVD, or any other transitory functionality as desired by the title.

After successfully mapping the utility region, the hard disk partition is available to the title through the N:\ drive.

XMountSecondaryUtilityDrive has some differences from the XMountUtilityDrive function:

  • The game cannot specify whether the alternate utility drive is formatted before it is mounted. It will always be formatted before it is mounted.
  • The data stored on the N: drive has no guarantee to be preserved or remounted if still available. The partition used by a title for the Z: drive will be automatically remounted if still available but this does not apply to the N: drive.
  • There are no flags in the XBE header for the N: drive. It must be mounted/unmounted at run time. It will always be formatted before being mounted.
  • The /UDCLUSTER parameter for the XBE Image Builder Tool will apply to both utility partitions. It is not possible to specify a separate cluster size for the N: drive.
  • The N: drive cannot be mounted if the Z: drive is not already mounted. The function will halt if the Z: is not mounted.
  • If the XMountSecondaryUtilityDrive function is called when the N: drive is already mounted, the function will halt.

Requirements

Header: Declared in Xbox.h.

Import Library: Use Xapilib.lib.
