DotProduct3 Sample

The DotProduct3 sample demonstrates an alternative approach to Direct3D bump mapping. This technique is named after the mathematical operation that combines a light vector with a surface normal. The normals for a surface are traditional (x, y, z) vectors stored in RGBA format in a texture map (called a normal map, for this technique).

Overview

The lighting equation for simulating bump mapping involves using the dot product of the surface normal and the lighting vector. The lighting vector is simply passed into the texture factor, and the normals are encoded in a texture map. The blend stages look like this:

  SetTextureStageState( 0, D3DTSS_COLOROP, D3DTOP_DOTPRODUCT3 );
  SetTextureStageState( 0, D3DTSS_COLORARG1, D3DTA_TEXTURE );
  SetTextureStageState( 0, D3DTSS_COLORARG2, D3DTA_TFACTOR );

The only trick is getting the normals stored in the texture. To do this, the components of a vector (XYZW) are each turned from a 32-bit floating value into a signed 8-bit integer and packed into a texture color (RGBA). The source code shows how to do this using a custom-generated normal map as well as one built from a bump-mapping texture image.
