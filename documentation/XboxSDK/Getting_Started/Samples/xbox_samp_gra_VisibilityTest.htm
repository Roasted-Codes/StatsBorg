VisibilityTest Sample

This sample illustrates how to do visibility testing on the XBox

Overview

This sample draws two objects, one of which is possibly occluded by the other. After starting the visibility test on the back object, the program waits for the test to complete before either rendering or not rendering the object.

Path

Source: (XDK Root)\Samples\Xbox\Graphics\VisibilityTest

Remarks

In a real game situation, you would do the following tasks.

  1. Intertwine visibility tests with actual rendering so the visibility test would have time to complete before you check the results.
  2. While you check the results, if a test has not yet concluded, draw additional geometry. If no additional geometry is able to be drawn, proceed as if the object is visible. It is important not to wait for tests to conclude. As much as possible, the graphics pipeline needs to be busy 100 percent of the time.

This sample uses a low-resolution version of the object to do the visibility test, and draws the high-resolution object only if the visibility test suggests we should. Another suggestion is to render the real object and check vis test results the following frame. Then use the logic that if an object was not visible in frame n, it probably won't be visible in frame n+1.
