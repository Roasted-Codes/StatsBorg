Cartoon Sample

The Cartoon sample draws a geometric shape, simulating cartoon lighting by using a 1D texture map. This sample demonstrates nontraditional 3D rendering techniques that simulate 2D animation in real time.

Lighting

The Cartoon sample does not use DirectX lighting; if it were to use traditional 3D lighting, the cartoonish effect would be lost and the mesh would look photo-realistic. Instead, a color (yellow, in this case) is specified for each vertex when the model is created. Next, a 32-by-1 true color texture is created and is filled with a nonuniform, gray-scale gradient. Only four unique shades of gray are used.

In place of a light source, Cartoon has a vector member variable called m_vLight. The first step in rendering the object is to calculate the UV texture coordinates (into the 1D texture map) of each vertex by comparing the direction of the normal of each vertex to the direction of the light vector.

Once the texture coordinates are calculated, the object is rendered normally. This provides the cartoon shading, but you still need to render the black outline.

The Outline

In pass two of the render, increase the scale of the mesh slightly (1.025f) and set its texture to black. If you were to render the scene now, you would see only a black silhouette of the mesh. To draw just the outline, you can take advantage of the fact that the mesh's triangles are defined clockwise.

In this pass, you will draw only the back-facing polygons (the ones that are defined counter-clockwise with respect to the camera). None of the polygons that were rendered in the first pass will be rendered in this pass. They will be removed during back-face culling. And because you haven't cleared the z-buffer, you will see only the parts of the larger, inverted mesh that extend beyond the original mesh.
