GameServerClient Sample

Demonstrates how to connect to a dedicated game server running in a datacenter.

Overview

This client connects to a sample game server running in the PartnerNet datacenter. The server doesn't actually host a real game, but merely acts as a UDP echo service. To run the sample, you need a valid account on your Xbox. You also need to run an output viewer such as Visual Studio .NET or XbWatson.

Path

Source: (XDK Root)\Samples\Xbox\Networking\GameServerClient

Remarks

The first step in connecting to a game service is to include the game server service ID in the list of services passed to the XOnlineLogon during authentication. This ID is assigned by the Xbox Live Administrators and is typically the same as the ID assigned to the title.

After authentication succeeds, the title calls XOnlineGetServiceInfo to obtain the game server Internet address and port. Once this is done, the title can create a socket and communicate with the service using the Winsock API functions.

Because game server development will most likely be done on a development system located outside the datacenter, this sample also demonstrates how to connect to a game server running on a development system. If the USE_LOCAL_MACHINE preprocessor symbol is defined, the sample connects to the machine whose IP address is identified by the global constants strHostAddress and PORT. Please note that when connecting to the development PC, you will need to use the non-secure debug libraries.
