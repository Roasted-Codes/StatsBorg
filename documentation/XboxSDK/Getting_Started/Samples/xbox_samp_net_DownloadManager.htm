DownloadManager Sample

Demonstrates how a title can save and restore a logon state after launching another image. For demonstration purposes, the Xbox content downloader is launched after allowing multiple users to sign on.

Overview

Titles save the current logon state using the XOnlineSaveLogonState function. This saves the current logon state into an XONLINE_LOGON_STATE structure. To restore the saved logon state, use the XOnlineRetrieveLogonState function.

Path

Source: (XDK Root)\Samples\Xbox\Networking\DownloadManager

Remarks

Copy the media tree to the target machine before running this sample. Be sure to rename the DownloadManager.xbe to Default.xbe so that it will be launched when the Xbox Downloader is exited. Also, Downloader.xbe must be stamped with the title ID of the sample (xbecert /TESTID:0xffff010C Downloader.xbe), and copied into the sample location as DownloadManager sample.

When the sample launches the Xbox downloader using XLaunchNewImage, it passes the current logon state as launch data. When the downloader exits, it uses XLaunchNewImage to load default.xbe and passes that launch data back to it. The sample checks for launch data when it start up. If the sample determines that the data is a saved logon state, it attempts to restore the users from that logon state.

The CXBoxSample::Initialize function in DownloadManager.cpp provides details on how to check for launch data. For information on how the users are restored, see RestoreUsersFromLogonState(). Note the additional checks being made in this function for unplugged controllers, and so forth. See CXBoxSample::UpdateStateMainMenu() for details on how the current logon state is obtained and passed to the the Xbox downloader.
