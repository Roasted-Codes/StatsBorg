SwapCallback Sample

The SwapCallback sample demonstrates how to use Swap and VBlank callbacks to determine when the game is close to dropping frames. A game can use this information to scale back rendering quality instead of dropping frames.

The sample is made to be fill bound by drawing a bunch of big, antialiased quads on the screen. The user can control the number of quads, and observe how close the sample is to dropping frames.

If a game does determine that it is close to (or beyond) the point of dropping frames, it can take steps to reduce fidelity in the game to reduce the overall frame time. What steps taken to reduce frame time depend on whether the game is fill bound, vertex bound, CPU bound, or a mix of the three.
