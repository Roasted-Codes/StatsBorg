Stats Sample

The Stats sample demonstrates the XOnlineStat APIs. The sample models a fictitious game that maintains a leaderboard for each of the four levels supported by the game.

Overview

Each leaderboard tracks the following player statistics.

Statistic Description
Kills The number of players "killed" by this player.
Deaths The number of times this player has "died."
Assists The number of assists this player has provided.
Started The number of times this player has started a level.
Completed The number of times this player has completed a level.
Rating A player rating (larger is better) used by the stats service to rank players.
Rank This player's rank for the leaderboard, as computed by the stats service.

When a "game" is completed, the statistics for a particular player are written to the leaderboard associated with the game level. If there are existing statistics for that player on the leaderboard, the new statistics are added to the existing values. In addition, an overall leaderboard is maintained so that the all-time numbers of kills and other statistics can be recorded.

When writing stats, a "rating" (XONLINE_STAT_RATING) is computed by the game title. For demonstration purposes, the sample (in CXBoxSample::CalculateRating) computes the rating as: 100*kills + 10*assists - 5*deaths. It is up to the title to decide how the rating is computed, but the bigger the number, the better the rating. Using this rating, the stats service will rank each player on a leaderboard.

Note    In addition to the currently signed-on user, other "players" are selected from existing user accounts that are stored locally on the machine, as well as from any Memory Units which have been inserted. These additional players are also signed on, because the XOnlineStat APIs require that they be signed on before their statistics can be updated.
