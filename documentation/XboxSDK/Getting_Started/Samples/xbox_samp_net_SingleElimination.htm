SingleElimination Sample

Demonstrates creating a single elimination tournament and playing through it.

Path

Source: (XDK Root)\Samples\Xbox\Networking\SingleElimination

Remarks

Copy the media tree to the target machine before running this sample.

This sample goes through the following steps:

  1. Get all Xbox Live accounts from the machine, including the memory units. There must be at least two accounts to create a tournament.
  2. Enable the user to select a user and create a tournament from that account. A single elimination tournament is created and all of the accounts are signed up for the tournament, up to a maximum of eight.
  3. Enter pre-init status until the registration time ends. The registration time can't be set too short or it may expire before all players have been signed up. Setting the perfect registration time for a sample is complicated because the server clock could be slightly ahead or behind of the client machine.
  4. Once registration time ends, enter active mode. The user can view the tournament and see who plays whom, who gets a bye, and so on. From this screen the user can simulate playing a round and send a 'winner' on to the next round.

The files SampleCompetition.cpp and SampleCompetition.h were generated by the Xbox Live Authoring Submission Tool (XLAST). The steps followed to create these files were:

  1. Run xlast.exe - installed with the XDK.
  2. Click File and then New to create a new competitions package. Set the project name, directory, and title ID appropriately. SampleCompetition was the project name used for this sample.
  3. Right click on the SampleCompetition. From the menu that appears, select New Competition Type, and create a single elimination tournament.

  4. Add the three custom attributes that this sample uses (your game will have different custom attributes). These are:

    map Int
    name String - max. length 200
    icon Binary - max. length 400
  5. Finally, use the build action to generate the code and the .xqs file.
