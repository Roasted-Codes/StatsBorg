CubeMap Sample

The CubeMap sample demonstrates an environment-mapping technique called cube mapping. Environment-mapping is a technique in which the environment surrounding a 3D object is put into a texture map so that the object can have complex lighting effects without expensive lighting calculations.

Overview

Cube mapping is a technique that employs a six-sided texture. In the CubeMap sample, the camera and the teapot are in a cube. Each of the six sides of the cube has one of the six environmental textures statically mapped on it. The teapot uses those same six maps, but the texture coordinates are dynamically generated based on the position of the camera as it orbits the teapot.

Cube mapping is superior to sphere-mapping because sphere maps are inherently view-dependent (sphere maps are constructed with one particular viewpoint in mind). Cube maps also have no geometry distortions, so they can be generated on the fly for each of the six cube map faces.

Cube mapping works with Direct3D texture coordinate generation. By setting D3DTSS_TCI_CAMERASPACEREFLECTIONVECTOR, Direct3D will generate cube map texture coordinates from the reflection vector for a vertex, thereby making this technique easy for environment-mapping effects where the environment is reflected in the object.

Controls

For a complete list of controller mappings, press the Back button on the Xbox controller while this sample is running.
