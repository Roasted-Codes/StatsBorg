PolynomialTextureMaps Sample

This sample implements polynomial texture maps using the Xbox GPU.

Overview

Traditional methods of light mapping using textures and bump maps use a different color for each point on a surface. Polynomial texture mapping, on the other hand, encodes a set of coefficients for each point on a surface. Programs use these coefficients as parameters to a polynomial model that approximates the surface color at any specified point on the surface. The advantage of this technique is that it can be used to model the way the appearance of an object changes as the lighting position changes.

Using PTM, the object being rendered is sampled from a number of different lighting directions, and coefficients are obtained by a least-squares fit of the model to the data. During rendering, the graphics processor is used to evaluate the polynomial model very quickly. Because each point on a surface can vary with the lighting direction, PTMs can capture interesting surface reflectance properties that normal texture maps cannot.

Remarks

This sample uses pre-computed PTMs. However, you can use the XTLs to generate PTMs from a set of sample textures that show an object's lighting under various conditions.
