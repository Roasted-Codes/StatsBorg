Play3DSound Sample

The Play3DSound sample demonstrates how to position DirectSoundBuffers and the Listener in 3D. The gamepad can control source position and velocity, as well as listener position. The velocity of the moving sound source and/or listener is calculated and passed to the buffer for a Doppler effect.

Sounds are created using DirectSound and the helper class in the XBSound.h/XBSound.cpp files. The helper class just provides easy access to the .wav file to get format information and read sample data from the file. 3D DirectSound buffers get sent to the crosstalk mixbins. Therefore, in order to get sound from those mixbins to the speakers, you must have a DSP image loaded that contains at least the crosstalk code. See the DownloadScratch() function for how to do this.
