DebugChannel Sample

This sample provides basic examples of what can be done through the debug channel, including:

  • Executing a function inside the running application—for example, using the RCmdTexture function to select a texture. Arguments can be pulled from parameters passed to the function.
  • Modifying application variables. This is really just a special case of calling a function whose purpose is to modify memory. In this sample, the spin rate and light color settings can be manipulated. A notification function can also be called— for example, setting a flag to denote that the light should be redefined.
  • Sending messages through the debug channel. Messages can be sent only to a console that asked for them, rather than putting them in the debug output.

Overview

The DebugChannel sample demonstrates how to add code to an application so that it can be controlled from the development system via the debug channel.

This program runs on the Xbox and requires the DebugConsole sample (RXCmdC) to be running on the development PC. You'll find the DebugConsole sample in (XDK Root)\Samples\Win32\Misc\DebugConsole. First, launch the DebugChannel sample on your target console, and then connect to it through RXCmdC.

Path

Source: (XDK Root)\SamplesSamples\Xbox\Misc\DebugChannel

Remarks

The DebugConsole sample program sends commands to the DebugChannel sample through the debug channel to the debug monitor on the Xbox machine. The DebugChannel sample running on the Xbox console receives the commands on a dedicated thread through a registered command processor callback function. The callback function stores commands in a buffer. The program polls this buffer once per frame, deciphering and handling the commands each time it does.

It is important to note the synchronization that is performed between the external command processor function RXCmdProcessorProc and the polling function RXCmdHandleRCmds. The external command processor function is called from the debug monitor thread, not the application thread. Any work done in the external command processor notification function should be thread-safe or protected, as in this sample.
