HostMigration Sample

Demonstrates a simple method titles can use to migrate the host of a game session to another console when the original host leaves.

Overview

This implementation adds three new messages and two states to a basic networking application. The functions to look at are:

  • FrameMoveFindNewHost – the "meat" of host migration.
  • FrameMoveResumeGame – resumes the game as the new host (mirrors the StartGame state).
  • SendConnectToMigratedHost – closely mirrors the standard JoinGame message.
  • SendMigrateHostApproved – closely mirrors the standard JoinApproved message.
  • SendPlayerRejoin – closely mirrors the standard PlayerJoined message.
  • ProcessConnectToMigratedHost
  • ProcessMigrateHostApproved
  • ProcessPlayerRejoin

Because connecting to a new host is closely related to connecting to a new game, the messages and state transitions are similar to those for initial connection. The main difference is that the player game state is maintained rather than initialized.

In this sample, rather than the new host keeping track of all of the connections it is waiting for, the client resubmits its state information upon connecting to the new host.

Each Xbox has a matching list of current players. The player in the first element of the list is the host.

The algorithm used for host migration is as follows:

  1. If the host disconnects, remove the host from the player list, and try to connect to the new first player.
  2. If I'm the first player, start hosting. Skip steps 3–9.
  3. Start a timeout timer (2 seconds in this example).
  4. Start a response delay timer (0.3 seconds in this example).
  5. Try to connect to who I think is host.
  6. If the response delay timer has triggered, go to step 4.
  7. If I receive a reply, go to step 9.
  8. If my timeout timer has triggered, remove the player from the player list and go to step 2.
  9. Connection was accepted, start the rejoin handshaking.

    Note    If for some reason the new host is a host, but is full, this sample sends a JoinDenied message so that it can notify the player the game is full.

Note    Step 1 is in ProcessDropouts. Steps 2–8 are implemented in FrameMoveFindNewHost. Step 9 is in ProcessMigrateHostApproved.
